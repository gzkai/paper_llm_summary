## 研究背景与动机

### 核心问题

当前多模态系统仍以"语言为中心"，将视觉、音频等非语言模态作为**外部附件**（bolt-on components）松散耦合到语言模型上。这种 *language-plus-auxiliary* 范式导致：
1. **架构碎片化**：模态特定的分支设计（如 modality-aware MoE、3D RoPE、双向注意力）
2. **次优集成**：连续视觉特征通过投影层接入 LLM，表示空间不统一
3. **理解与生成冲突**：两个目标的优化方向相互竞争

### 核心挑战

将视觉信号压缩到**有限码本**（finite codebook）时不可避免地引入信息损失，导致离散视觉建模存在所谓的"性能天花板"——业界普遍认为离散视觉模型无法与连续模型竞争。

### 论文目标

提出 **DiNA（Discrete Native Autoregression）** 范式，证明：
- 离散 token 可以统一表示文本、视觉、音频
- 离散建模**没有固有性能天花板**，性能瓶颈主要在于训练数据规模
- 理解和生成可以在同一自回归目标下协同，而非相互竞争

## 核心方法

### 框架总览：DiNA（离散原生自回归）

```
原始输入（文本 / 图像 / 音频）
        ↓ 模态特定 Tokenizer（dNaViT / Audio Tokenizer）
离散 Token 序列（统一 ID 空间）
        ↓ LongCat-Flash-Lite A3B（MoE 解码器，模态无关）
        ↓ 下一 Token 预测（单一自回归目标）
输出 Token 序列
        ↓ 模态特定 Detokenizer
文本 / 图像 / 音频输出
```

**关键设计哲学**：将视觉和音频视为语言的**内在扩展**，而非外部附件。所有模态 embedding 随机初始化，与模型联合端到端训练，在共享离散空间中自然对齐。

### 模块一：视觉 Tokenizer — dNaViT

#### 设计动机：语义完备性（Semantic Completeness）

定义：离散表示 $z = \{idx_1, \ldots, idx_n\}$ 满足语义完备性，当且仅当对任意图像相关查询 $\mathcal{Q}$ 和任务 $\mathcal{T}$：

```math
\mathcal{P}(A \mid z, \mathcal{Q}) \approx \mathcal{P}(A \mid I, \mathcal{Q})
```

语义完备性包含两个子属性：
- **判别不变性**：离散化不降低下游判别性能
- **生成充分性**：离散码提供足够的语义描述，支持高保真图像重建

#### 关键组件一：语义对齐编码器（SAE）

现有视觉编码器分三类，各有不足：

| 类别 | 代表 | 优点 | 缺点 |
|------|------|------|------|
| 低级重建模型 | VAE, VQ-VAE | 像素保真度高 | 高级概念推理弱 |
| 自监督语义编码器 | DINOv2, SigLIP | 结构/对比特征好 | 缺乏显式语义基础，生成重建弱 |
| 无编码器原始像素 | EVE, NEO | 简单可扩展 | 像素冗余大 |

论文提出 **SAE（Semantic-and-Aligned Encoder）**：通过大规模视觉-语言训练获得的语言对齐语义编码器，同时具备：
1. **语义丰富性**：捕捉高级概念和细粒度视觉细节（如文字细节）
2. **与语言模型的亲和性**：自然融入统一离散 token 空间

SAE 训练目标（多方面对齐）：

```math
\mathcal{L}_{\text{SAE}} = \mathbb{E}_{(I,\mathcal{Q},A)}\left[-\log P(A \mid \mathbf{z_p}, \mathcal{Q})\right]
```

实践中直接采用现有视觉-语言模型（如 **Qwen2.5-ViT**，28× 空间压缩比）作为 SAE，无需从头训练。

#### 关键组件二：残差向量量化（RVQ）

对 SAE 特征 $\mathbf{z}$ 进行 $L$ 级级联码本量化：

```math
\mathbf{r}_0 = f_{\text{proj}}(\mathbf{z}), \quad
\hat{\mathbf{q}}_l = \text{VQ}(\mathbf{r}_{l-1}), \quad
\mathbf{r}_l = \mathbf{r}_{l-1} - \hat{\mathbf{q}}_l, \quad
\hat{\mathbf{z}} = \sum_{l=1}^{L} \hat{\mathbf{q}}_l
```

码本使用 EMA 更新（非梯度下降），码本条目更新规则：

```math
\mathbf{e}_k \leftarrow \frac{\mathbf{m}_k}{N_k}
```

量化训练目标：

```math
\mathcal{L}_{\text{quant}} = \lambda_c \mathcal{L}_{\text{commit}} + \lambda_s \mathcal{L}_{\text{semantic}}
```

**任意分辨率支持**：采用可变长度序列建模（FlashAttention），最大训练序列长度 8192，最大图像分辨率 1736×1736。

#### 关键发现：内在信息恢复（Intrinsic Information Recovery）

实验发现：**随机初始化的 ViT-Base** 重建效果优于预训练版本（PSNR 30.52 vs 21.86）！

原因分析：残差架构天然保留了低级信号传播的潜在路径：

```math
\mathbf{z}_p = \mathbf{x}_0 + \sum_{l=1}^{L} \mathcal{F}_l(\mathbf{x}_{l-1})
```

恒等映射 $\mathbf{x}_{l-1}$ 确保早期层的细粒度视觉信号不被高级语义抽象覆盖，而是通过网络逐步积累到 $\mathbf{z}_p$ 中，保持与原始图像的高互信息 $\mathcal{I}(I; \mathbf{z}_p)$。

#### De-tokenizer 设计

- **像素解码器**：400M 参数 ViT，从离散码 embedding 重建图像，使用像素损失 + 感知损失 + 对齐损失
- **图像精细化器**：基于 OmniGen2 初始化的 Flow Matching 模型，提升高频细节和感知锐度

解码器训练目标：

```math
\mathcal{L}_{\text{dec}} = \lambda_1 \mathcal{L}_{\text{pixel}} + \lambda_2 \mathcal{L}_{\text{percep}} + \lambda_3 \mathcal{L}_{\text{align}}
```

#### Pre-Buffer 模块

发现离散模型早期训练损失显著高于连续基线，原因是多级 embedding 求和后缺乏重新编码。引入 **Pre-Buffer**（单层 FFN）对多级求和特征重新映射，显著加速收敛。

### 模块二：音频 Tokenizer

**架构**：Whisper 编码器（large-v3 初始化）→ 4× 下采样 → 8 层 RVQ（码本大小：8k/4k/2k/1k/1k/1k/1k/1k）→ 12.5 Hz 离散音频 token

训练目标：

```math
\mathcal{L}_{\text{audio}} = \lambda_1 \mathcal{L}_{\text{recon}} + \lambda_2 \mathcal{L}_{\text{commit}} + \lambda_3 \mathcal{L}_{\text{llm}}
```

三阶段训练：
1. **Decoder Warm-up**：冻结编码器和 LLM，训练解码器做 Mel 频谱重建
2. **语义-声学联合训练**：解冻所有模块（除 LLM 和 Flow Matching），启用 RVQ
3. **Decoder Fine-tuning**：高质量 24kHz 数据，解码器改为 DiT 模块去噪

**训练数据**：约 250 万小时（Web 音频 320 万小时 → 过滤保留 16.2% = 320 万小时 + 合成语音 120 万小时 + 任务特定 40 万小时）

---

### 模块三：语言模型骨干

**基础模型**：LongCat-Flash-Lite A3B（MoE 架构）
- 总参数：68.5B
- 激活参数：平均 3B（范围 2.9B~4.5B，取决于上下文）
- 架构特点：Zero-Expert + Shortcut MoE

**模态无关设计**：所有 token（文本、视觉、音频）通过**单一模态无关路径**处理，无模态特定分支。

---

### 模块四：内部语言引导音频生成

参考 Moshi 方法，在文本引导音频模态中，段级对齐的文本 token 和音频 token 通过 element-wise 求和融合。

两种生成策略（通过随机延迟统一训练范式）：

**并行生成**：每步同时生成文本和音频 token，音频 token 延迟 N 步，适合全双工低延迟场景
- LlamaQuestions: 79.33
- ReasoningQA: 74.95

**串行生成**：先生成文本段，再生成音频段，语义质量更高
- LlamaQuestions: 81.67
- ReasoningQA: 80.30

**统一训练**：对每个对齐的文本-音频段，随机从 1 到文本段长度选择延迟，使两种策略成为统一框架的两个极端情况。

### 模块五：强化学习（RL）

#### 文生图 RL（GRPO）

多级 RVQ 的加权 GRPO 目标：

```math
r_t^{(l)} = \frac{\pi_\theta^{(l)}(a_t^{(l)}|s_t)}{\pi_{\text{old}}^{(l)}(a_t^{(l)}|s_t)}
```

```math
\mathcal{L}_{GRPO} = \mathbb{E}\left[\sum_{l=1}^{L} w_l \cdot \min\left(r_t^{(l)} \cdot \hat{A}_t,\ \mathrm{clip}(r_t^{(l)}, 1-\epsilon, 1+\epsilon) \cdot \hat{A}_t\right)\right]
```

四维奖励模型：
1. **综合能力**：物体计数、颜色准确性、空间位置、属性一致性
2. **OCR 能力**：GOT-OCR 2.0 识别，编辑距离量化文字保真度
3. **文本-图像语义对齐**：VLM 作为奖励模型
4. **图像质量**：HPS v3 + Aesthetic Score + Unified Reward

#### 图像理解 RL（熵爆炸问题及解决方案）

发现问题：随训练进行，策略熵和训练-推理不匹配同时增加，产生噪声/乱码 token，形成正反馈循环。

**解决方案：序列级过滤**

带过滤的 GRPO 目标：

```math
\mathcal{L}_{\text{GRPO-Filtered}} = \mathbb{E}\left[
\underbrace{\mathbb{I}\{H_{\text{seq}} \le \mu_H + n\sigma_H\}}_{\text{熵过滤}} \cdot
\underbrace{\mathbb{I}\{\max_{t'}|\pi_{\text{sampler}} - \pi_{\text{actor}}| \le \delta\}}_{\text{概率差过滤}} \cdot
\min(r_t \cdot \hat{A}_t,\ \mathrm{clip}(\cdot) \cdot \hat{A}_t)
\right]
```

## 关键实验发现

### 1. 离散 vs 连续建模差距可消除

| 实验 | 表示类型 | PT Tokens | MT+SFT Tokens | OCRBench | MMMU |
|------|---------|-----------|----------------|---------|------|
| I | 连续 | 0.1B | 4B | 779 | 49.8 |
| II | 离散 | 0.1B | 4B | 598 | 44.8 |
| VI | 连续 | 5B | 300B | 858 | 58.0 |
| VII | 离散 | 5B | 300B | **844** | **60.0** |

**结论**：数据规模扩展后，离散模型（VII）与连续模型（VI）性能几乎持平（OCRBench 844 vs 858，MMMU 60.0 vs 58.0）。**离散建模没有固有性能天花板**。

### 2. 理解与生成协同而非竞争

| 训练设置 | 理解损失 | 生成损失 |
|---------|---------|---------|
| Pure-Und（100B tokens 理解数据） | 基准 | -- |
| Pure-Gen（100B tokens 生成数据） | -- | 基准 |
| Unified（50B+50B 混合） | 基准 +0.006 | 基准 -0.02 |

**结论**：统一模型生成损失比纯生成模型低 0.02，理解损失仅增加 0.006。**生成不损害理解，理解反而促进生成**。

### 3. 模态无关 MoE 的自发专业化

原本模态无关的 MoE 专家在多模态训练后：
- 部分专家自发形成对特定模态（视觉/音频）的功能专业化
- 路由机制变得更结构化和稳定
- 每个专家处理的平均 token 数从 507.1 增加到 584.6（容量利用率提升）

### 4. Platonic 表示假说验证

t-SNE 可视化（50,000 个样本点）显示：
- **Qwen2.5-VL**（非原生）：视觉和文本 token 形成明显分离的模态簇
- **Qwen3.5**（原生训练）：部分跨模态对齐
- **LongCat-Next**（DiNA）：视觉和文本 token **交织分布**，形成统一语义空间

## 基础设施创新：VHalf 流水线并行

针对多模态模型中异构计算负载（Embedding 层 + 模态特定损失模块 vs LLM 层）导致的流水线气泡问题，提出 **VHalf-based 流水线并行**：

1. **V 形调度**：将第一阶段（Embedding）和最后阶段（模态损失模块）共置于同一物理设备
2. **共享缓冲区**：在 V 形流水线第一阶段构建共享缓冲区，使模态损失模块直接访问 RVQ 多级 token 的隐状态

效果：
- **消除流水线气泡**：异构负载被有效吸收，实现近完美负载均衡
- **消除跨阶段通信**：Embedding 层和模态损失模块共置于同一设备，通过零拷贝内存访问解决前向激活和反向梯度传输

## 核心贡献总结

1. **DiNA 范式**：首个工业级原生多模态离散自回归框架，将文本、视觉、音频统一在共享离散 token 空间，使用单一 NTP 目标

2. **dNaViT**：基于 SAE + RVQ 的离散原生分辨率视觉 Transformer，支持任意分辨率 tokenization/de-tokenization，28× 压缩比；Pre-Buffer 模块解决多级 embedding 对齐问题；首次证明离散视觉建模无固有性能天花板

3. **内在信息恢复发现**：残差连接天然保留低级信号传播路径，使语义编码器具备重建能力，即使未经重建监督

4. **理解-生成统一**：实验证明理解促进生成，两者在 DiNA 框架下协同而非竞争，有效消除"理解-生成冲突"

5. **音频统一建模**：随机延迟统一训练范式，将并行和串行语音生成统一为同一框架的两个极端情况

6. **工业级性能**：A3B 激活参数规模下，超越 Qwen3-Omni-A3B-Instruct（全模态基线），在数学推理、OCR、文生图等多项任务上超越专业模型

3. **数据扩展与表示学习**：探索大规模感知预训练与离散 token 建模的协同设计；当前发现传统预训练范式不能直接转化为一致增益，需联合设计数据、预训练目标和离散化策略
