## 一、核心成就

> *14B Real-Time Long Video Generation Model can be Cheaper, Faster but Keep Stronger than 1.3B*  
> — Helios Team

**Helios** 是首个在单张 NVIDIA H100 GPU 上达到 **19.5 FPS** 的 14B 视频生成模型，支持分钟级长视频生成，质量与强基线模型相当。

### 三大突破（均无需常规技巧）

| 突破维度 | 挑战 | 解决方案 | 不依赖的常规技巧 |
|---------|------|---------|----------------|
| **无漂移长视频** | 自回归误差累积导致颜色/位置/图像退化 | Easy Anti-Drifting | self-forcing、error-banks、keyframe sampling |
| **实时生成** | 14B 模型 3D 注意力 $O(L^2)$ 计算量 | Deep Compression Flow | KV-cache、稀疏注意力、量化、因果掩码 |
| **单卡训练** | 4 个 14B 模型同时驻留 80GB 显存 | 基础设施优化 | 并行框架、分片框架（TP/CP/SP） |

---

## 二、模型架构总览

Helios 是一个 **14B 自回归视频扩散 Transformer**，由三大技术模块构成：

```
输入: [Historical Context X_Hist] + [Noisy Context X_Noisy]
         ↓ Representation Control（任务切换: T2V / I2V / V2V）
         ↓ Multi-Term Memory Patchification（历史上下文压缩）
         ↓ Guidance Attention（引导去噪）
         ↓ Pyramid Unified Predictor Corrector（多尺度采样）
         ↓ Adversarial Hierarchical Distillation（3步生成）
输出: 时序连贯的视频片段
```

---

## 三、技术模块详解

### 3.1 Unified History Injection（无限时长生成）

#### Representation Control

将长视频生成形式化为**视频续写**（Video Continuation）：

- 输入 = 历史上下文 $X_\text{Hist} \in \mathbb{R}^{B \times C \times T_\text{Hist} \times H \times W}$ + 噪声上下文 $X_\text{Noisy} \in \mathbb{R}^{B \times C \times T_\text{Noisy} \times H \times W}$
- 训练和推理时保持 $T_\text{Hist} \gg T_\text{Noisy}$ 固定
- 通过 $X_\text{Hist}$ 的表示自动切换任务模式：

| $X_\text{Hist}$ 的值 | 任务模式 |
|---------------------|---------|
| 全零 | T2V（文本到视频） |
| 仅最后一帧非零 | I2V（图像到视频） |
| 正常视频片段 | V2V（视频到视频） |

> **优势**：避免 Diffusion Forcing + Causal Mask 方案的推理范式偏差，保留双向推理优势。

#### Guidance Attention

历史上下文与噪声上下文统计特性不同，需区别对待：

**自注意力**：引入逐头放大 token `amp`，选择性增强历史信息：
$$X_\text{Self} = \text{Attention}([Q_\text{Noisy}, Q_\text{Hist}],\; [K_\text{Noisy}, K_\text{Hist} \cdot amp],\; [V_\text{Noisy}, V_\text{Hist}])$$

**交叉注意力**：仅对噪声上下文注入文本语义（历史已包含语义，重复注入冗余）：
$$X_\text{Cross} = \text{Attention}(Q_\text{Noisy},\; K_\text{Text},\; V_\text{Text})$$

- 固定 $X_\text{Hist}$ 的时间步为 0，表明其始终为干净帧
- 不使用因果掩码（Causal Mask），保留双向推理能力，可无缝集成 FlashAttention

### 3.2 Easy Anti-Drifting（高质量长视频生成）

#### 三种漂移模式分析

| 漂移类型 | 表现 | 根因 |
|---------|------|------|
| **Position Shift** | 场景重置、循环运动 | 绝对 RoPE 在超训练长度时暴露未见位置编码，RoPE 周期性与多头注意力交互 |
| **Color Shift** | 色彩饱和度随时间偏移 | 历史帧积累的颜色偏差逐渐放大 |
| **Restoration Shift** | 模糊/噪声伪影 | 模型以自身不完美输出为条件，误差累积放大 |

#### 解决方案一：Relative RoPE

无论目标视频长度，将 $X_\text{Hist}$ 的时间索引约束在 $[0, T_\text{Hist})$，$X_\text{Noisy}$ 分配到 $[T_\text{Hist}, T_\text{Hist} + T_\text{Noisy})$。

- 相对索引使模型在任意长度下稳定生成
- 消除 RoPE 周期性与多头注意力交互导致的 **重复运动（Repetitive Motion）**

#### 解决方案二：First-Frame Anchor

训练和推理时始终在 $X_\text{Hist}$ 中保留**第一帧**：

- 第一帧作为全局视觉锚点，约束后续片段的分布偏移
- 有效缓解颜色漂移，增强主体一致性
- 实验观察：漂移很少在生成开始阶段出现，移除此组件导致 720 帧处就出现明显退化

#### 解决方案三：Frame-Aware Corrupt

训练时对历史帧施加随机扰动，模拟推理时的不完美历史输入：

| 扰动类型 | 概率 | 参数范围 |
|---------|------|---------|
| 高斯噪声 | $p_a$ | 强度 $\in [b_\min, b_\max]$ |
| 降采样+上采样（模糊） | $p_b$ | 倍率 $\in [c_\min, c_\max]$ |
| 曝光调整（颜色抖动） | $p_c$ | 幅度 $\in [a_\min, a_\max]$ |
| 保持干净 | $p_d$ | — |

- 扰动**逐帧独立**执行，$T_\text{Hist}$ 帧对应 $T_\text{Hist}$ 次独立扰动决策
- 无需 self-forcing 或 error-banks 即可实现长视频稳定生成

### 3.3 Deep Compression Flow（实时生成）

#### 3.3.1 Multi-Term Memory Patchification（历史上下文压缩）

**核心观察**：近期历史帧对局部运动重要，远期历史帧仅提供粗粒度全局上下文。

将 $X_\text{Hist}$ 分为三段（短期/中期/长期），对每段应用不同压缩比的 3D 卷积核：

$$L_\text{total} = HW \left( \frac{T_1}{p_t^{(1)} p_h^{(1)} p_w^{(1)}} + \frac{T_2}{p_t^{(2)} p_h^{(2)} p_w^{(2)}} + \frac{T_3}{p_t^{(3)} p_h^{(3)} p_w^{(3)}} \right)$$

- 压缩比随时间距离增大：$p_t^{(1)} < p_t^{(2)} < p_t^{(3)}$
- **$L_\text{total}$ 与目标视频长度无关**，固定 token 预算下可保留更长历史
- 历史长度扩展至 18 段时计算/显存保持稳定，而朴素方法在历史长度 6 时就 OOM

#### 3.3.2 Pyramid Unified Predictor Corrector（噪声上下文压缩）

**核心观察**：早期采样步噪声大，决定全局结构；后期采样步精炼细节。

采用**粗到细调度**：早期阶段在低分辨率潜空间采样，逐步过渡到全分辨率。

**训练**：将生成过程划分为 $K=3$ 个阶段，每阶段学习多尺度速度场：
$$x_t^k = (1-\lambda_t)\, x^k + \lambda_t\, \mathrm{Up}(x^{k-1}), \quad v^k = x^k - \mathrm{Up}(x^{k-1})$$
$$\mathcal{L} = \mathbb{E}_{k,\lambda_t,x_t^k,\mathrm{Up}(x^{k-1}),y} \left[ \left\| u_\theta^k(x_t^k, y, \lambda_t, k) - v^k \right\|_2^2 \right]$$

**推理效率**：多尺度采样的总处理 token 数：
$$\left( H \times W + \frac{H}{2} \times \frac{W}{2} + \frac{H}{4} \times \frac{W}{4} + \cdots + \frac{H}{2^{K-1}} \times \frac{W}{2^{K-1}} \right) \times \frac{N}{K}$$

相比单尺度 $H \times W \times N$，计算量显著降低。

**UniPC 适配**：跨分辨率阶段时重置状态缓冲区，避免跨尺度校正伪影。

#### 3.3.3 Adversarial Hierarchical Distillation（步数蒸馏）

基于 DMD 框架，将采样步数从 50 步降至 **3 步**，同时消除 CFG。

**四项关键改进**：

| 改进项 | 解决的问题 | 方法 |
|--------|-----------|------|
| **Pure Teacher Forcing** | 长自回归展开（Self-Forcing）训练成本极高，限制模型只能用 1.3B | 蒸馏阶段使用**真实数据**作为历史上下文，每步仅生成单段，无需长序列展开 |
| **Staged Backward Simulation** | 标准 DMD 管道与 Helios 分阶段采样不兼容 | 将 backward simulation 分解为 $K$ 个阶段，逐阶段估计 $x_0^k$，最终 $x_0 = x_0^K$ |
| **Coarse-to-Fine Learning** | 多阶段梯度路径复杂，训练初期难以收敛 | 分阶段 ODE 初始化 + Beta 分布动态采样时间步（早期集中高噪声，后期均匀分布） |
| **Adversarial Post-Training** | 纯 DMD 继承教师模型偏置，可能产生过饱和/不自然视频 | 引入多头 GAN 判别器，使用非饱和 GAN 目标 + 近似 R1 正则化 |

**完整目标函数**：
$$\mathcal{L}_{G_\theta} = \mathcal{L}_\text{DMD} + w_G \cdot \mathcal{L}_G, \quad w_G = 5\times10^{-2}$$
$$\mathcal{L}_{p_\text{fake}} = \mathcal{L}_\text{Flow} + w_D \cdot \mathcal{L}_D, \quad w_D = 1\times10^{-2}$$

> **注意**：实验表明强化学习（Reward-weighted Regression）反而导致视频严重闪烁，最终未采用。

## 四、推理时无训练技巧

### Adaptive Sampling（自适应采样）

通过 EMA 维护全局 RGB 统计量（均值 $\bar{\mu}_t$、方差 $\bar{\sigma}_t^2$），当当前段统计量偏离超过阈值时，对历史上下文施加 Frame-Aware Corrupt，降低模型对偏置历史的依赖。

### Interactive Interpolation（交互插值）

支持推理时动态修改 prompt，通过线性插值 $M$ 个中间 embedding 平滑过渡：
$$\mathbf{e}^{[j]} = (1-\lambda_j)\mathbf{e}^{(1)} + \lambda_j\mathbf{e}^{(2)}, \quad \lambda_j = \frac{j}{M-1}$$

## 五、训练基础设施优化

### 5.1 单 GPU 训练（无需并行框架）

| 优化项 | 解决的问题 | 方法 |
|--------|-----------|------|
| **磁盘卸载** | VAE 潜变量 + 文本嵌入占用大量显存 | 预计算并卸载到磁盘，GPU 仅保留 DiT |
| **Sharded EMA（ZeRO-3）** | FP32 EMA 副本在每 GPU 上冗余存储 | EMA 参数分片，每卡仅存 $14\times4/Z$ GiB |
| **Asynchronous VRAM Freeing** | Stage-3 需同时驻留多个 14B 模型 | 串行执行，前向完成后立即异步卸载到主机内存（pinned memory + non-blocking） |
| **Cache Grad for GAN** | GAN 训练需保留完整计算图 | 前向时缓存判别器对输入的梯度，立即释放中间激活，解耦反向传播 |
| **Sequence-wise Tiling** | Linear 层激活值 $O(B\ell D)$ 显存瓶颈 | 沿序列维度切分为 $X = \lceil \ell/D \rceil$ 个 tile 逐个计算，显存降至 $O(B\ell D/X)$ |

### 5.2 推理效率优化（Triton 内核）

| 优化项 | 效果 |
|--------|------|
| **Flash Normalization** | LayerNorm/RMSNorm 融合，中间激活 $O(B\ell D) \to O(B\ell)$ |
| **Flash RoPE** | RoPE 融合，中间激活 $O(B\ell HD) \to O(B\ell D)$ |
| **FlashAttention** | 无因果掩码，与高效注意力后端无缝集成 |

### 5.3 三阶段训练配置

| 配置 | Stage-1（Base） | Stage-2（Mid） | Stage-3（Distilled） |
|------|----------------|---------------|---------------------|
| GPU 数量 | 64 H100 | 64 H100 | 128 H100 |
| 全局 Batch Size | 128 | 256 | 128 |
| 训练步数 | ~13K | ~36K | ~6K |
| LoRA Rank | 128 | 256 | 256 |
| 精度 | BFloat16 | BFloat16 | BFloat16 |

## 七、HeliosBench 基准

**问题**：缺乏综合评估实时长视频生成的开源基准。

**HeliosBench 构成**：
- **240 个提示词**
- 覆盖 4 种时长：极短（81 帧）、短（240 帧）、中（720 帧）、长（1440 帧）
- 评估维度：Throughput Score（吞吐量得分）+ Drifting Score（漂移得分）+ 常规视频质量指标

---

## 八、局限性与未来工作

1. **评估指标不足**：现有指标（Aesthetic、Smoothness）在不同方法间差异微小，难以精确区分；需要开发更符合人类感知的指标。

2. **拼接边界闪烁**：与现有自回归模型一样，片段拼接处可能出现闪烁；未来可探索 RL 优化 Smoothness 目标。

3. **分辨率限制**：受资源限制，实验仅在 $384 \times 640$ 分辨率下进行，未探索更高分辨率。

4. **长视频记忆机制**：尚未针对长视频的记忆机制进行深入设计。

## 九、核心贡献总结

| 贡献 | 技术方案 | 效果 |
|------|---------|------|
| **无漂移长视频** | Relative RoPE + First-Frame Anchor + Frame-Aware Corrupt | 无需 self-forcing/error-banks，分钟级视频高质量、强连贯性 |
| **实时生成** | Multi-Term Memory Patchification + Pyramid UniPC + Adversarial Distillation | 单卡 H100 **19.5 FPS**，**128× 加速** |
| **单卡训练** | 磁盘卸载 + Sharded EMA + 异步 VRAM 释放 + Cache Grad + Sequence Tiling | 14B 模型可在单 GPU 训练，batch size 与图像扩散模型相当 |
| **HeliosBench** | 240 提示词，4 种时长 | 填补实时长视频生成评估空白 |
| **长视频 SOTA** | Helios-Distilled 总分 6.94 | 超越所有对比方法（短视频和长视频均 SOTA） |
