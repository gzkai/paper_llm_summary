## 研究背景与动机

### 语言建模的二元性

语言建模本质上包含两类质性不同的子任务：
1. **组合推理（Compositional Reasoning）**：需要深度、动态的计算，依赖 MoE 的条件计算
2. **知识检索（Knowledge Retrieval）**：大量文本（命名实体、惯用表达）是局部、静态、高度程式化的，天然适合通过查找表（Lookup）以 $`\mathcal{O}(1)`$ 复杂度完成

### 现有框架的核心缺陷

标准 Transformer 缺乏原生的知识查找原语（Knowledge Lookup Primitive），因此 LLM 被迫**用计算来模拟记忆检索**。例如，识别 `"Diana, Princess of Wales"` 这一实体需要消耗多个 Attention 层和 FFN 层来逐步组合特征（PatchScope 分析显示需要 6 层才能完成），本质上是在运行时重建一个静态查找表，浪费了宝贵的序列深度。

### 解决方案：条件记忆（Conditional Memory）

本文倡导引入与条件计算（MoE）互补的新稀疏轴——**条件记忆**：
- **条件计算（MoE）**：稀疏激活参数，处理动态逻辑
- **条件记忆（Engram）**：稀疏查找操作，检索静态嵌入

## 核心方法：Engram 架构

### 总体设计

Engram 是一个条件记忆模块，通过**结构性地将静态模式存储与动态计算分离**来增强 Transformer backbone。给定输入序列 $`X = (x_1, \dots, x_T)`$ 和第 $`\ell`$ 层隐藏状态 $`\mathbf{H}^{(\ell)}`$，模块分两个功能阶段处理每个位置 $`t`$：**检索（Retrieval）** 和 **融合（Fusion）**。

最终通过残差连接集成到 backbone：

```math
\mathbf{H}^{(\ell)} \leftarrow \mathbf{H}^{(\ell)} + \mathbf{Y}
```

---

### 阶段一：基于哈希 N-gram 的稀疏检索

#### 1. Tokenizer 压缩（Tokenizer Compression）

标准 subword tokenizer 优先无损重建，常为语义等价词分配不同 ID（如 `Apple` vs `apple`）。为最大化语义密度，预计算一个满射函数：

```math
\mathcal{P}: V \to V', \quad x'_t = \mathcal{P}(x_t)
```

基于 NFKC 归一化、小写化等规则将原始 token ID 映射为规范 ID，**对 128k tokenizer 实现 23% 的有效词汇量压缩**。

压缩后的 suffix N-gram 为：$`g_{t,n} = (x'_{t-n+1}, \dots, x'_t)`$

#### 2. 多头哈希（Multi-Head Hashing）

直接参数化所有可能 N-gram 的组合空间不可行，采用哈希方法。为缓解碰撞，对每个 N-gram 阶 $`n`$ 使用 $`K`$ 个独立哈希头：

```math
z_{t,n,k} \triangleq \phi_{n,k}(g_{t,n}), \quad \mathbf{e}_{t,n,k} = \mathbf{E}_{n,k}[z_{t,n,k}]
```

其中 $`\phi_{n,k}`$ 为轻量级乘法-XOR 哈希函数，$`\mathbf{E}_{n,k}`$ 为大小为质数 $`M_{n,k}`$ 的嵌入表。

拼接所有检索嵌入得到最终记忆向量：

```math
\mathbf{e}_t \triangleq \mathop{\Vert}_{n=2}^{N} \mathop{\Vert}_{k=1}^{K} \mathbf{e}_{t,n,k} \in \mathbb{R}^{d_{\text{mem}}}
```

---

### 阶段二：上下文感知门控（Context-aware Gating）

检索到的嵌入 $`\mathbf{e}_t`$ 是上下文无关的先验，存在哈希碰撞和多义性噪声。为此引入受 Attention 启发的上下文感知门控：

- 当前隐藏状态 $`\mathbf{h}_t`$ 作为动态 **Query**（已通过前置 Attention 层聚合全局上下文）
- 检索记忆 $`\mathbf{e}_t`$ 作为 **Key** 和 **Value** 的来源

```math
\mathbf{k}_t = \mathbf{W}_K \mathbf{e}_t, \quad \mathbf{v}_t = \mathbf{W}_V \mathbf{e}_t
```

计算标量门控值（添加 RMSNorm 保证梯度稳定性）：

```math
\alpha_t = \sigma\left( \frac{\text{RMSNorm}(\mathbf{h}_t)^\top \text{RMSNorm}(\mathbf{k}_t)}{\sqrt{d}} \right)
```

门控输出：$`\tilde{\mathbf{v}}_t = \alpha_t \cdot \mathbf{v}_t`$

**语义对齐特性**：若检索记忆与当前上下文矛盾，$`\alpha_t \to 0`$，自动抑制噪声。

最后通过轻量级深度因果卷积（kernel size=4，dilation=最大 N-gram 阶数，SiLU 激活）扩展感受野：

```math
\mathbf{Y} = \text{SiLU}\left( \text{Conv1D}( \text{RMSNorm}(\tilde{\mathbf{V}}) ) \right) + \tilde{\mathbf{V}}
```

---

### 与多分支架构的集成

本文采用 **流形约束超连接（mHC，M=4 分支）** 作为 backbone，Engram 通过参数共享策略适配多分支：

- **共享**：单一稀疏嵌入表 + 单一 Value 投影矩阵 $`\mathbf{W}_V`$（跨所有分支共享）
- **独立**：M 个分支各有独立的 Key 投影矩阵 $`\{\mathbf{W}_K^{(m)}\}_{m=1}^M`$

第 $`m`$ 个分支的门控：

```math
\alpha_t^{(m)} = \sigma\left( \frac{\text{RMSNorm}(\mathbf{h}_t^{(m)})^\top \text{RMSNorm}(\mathbf{W}_K^{(m)} \mathbf{e}_t)}{\sqrt{d}} \right)
```

线性投影可融合为单一 FP8 矩阵乘法，最大化 GPU 计算利用率。

---

### 系统效率：计算与存储解耦

Engram 的确定性检索（索引仅依赖输入 token 序列，不依赖运行时隐藏状态）带来独特系统优势：

| 阶段 | 策略 |
|------|------|
| **训练** | 嵌入表跨 GPU 分片，All-to-All 通信原语收集/分发激活行 |
| **推理** | 嵌入表卸载到 Host DRAM，异步预取（Prefetch）通过 PCIe 传输，与前置 Transformer 块的计算重叠，掩盖通信延迟 |

**多级缓存层次**：基于 N-gram 的 Zipfian 分布，高频嵌入缓存于 GPU HBM，长尾稀有模式驻留在 NVMe SSD。

**实测结果**：将 100B 参数嵌入表完全卸载到 Host DRAM，推理吞吐量损失仅 **<3%**（4B 模型：9031→8858 tok/s；8B 模型：6315→6140 tok/s）。

## 稀疏分配定律与 Scaling Law

### 稀疏分配问题（Sparsity Allocation）

**核心问题**：在固定总参数量和训练计算量（Iso-parameter + Iso-FLOPs）下，如何在 MoE 专家和 Engram 嵌入之间分配稀疏容量？

**参数定义**：
- $`P_{\text{tot}}`$：总可训练参数量
- $`P_{\text{act}}`$：每 token 激活参数量（决定训练 FLOPs）
- $`P_{\text{sparse}} \triangleq P_{\text{tot}} - P_{\text{act}}`$：非激活参数（"免费"的参数预算）

**分配比例** $`\rho \in [0,1]`$：

```math
P_{\text{MoE}}^{(\text{sparse})} = \rho\, P_{\text{sparse}}, \qquad P_{\text{Engram}} = (1-\rho)\, P_{\text{sparse}}
```

- $`\rho=1`$：纯 MoE 模型
- $`\rho<1`$：减少路由专家数量，将释放的参数分配给 Engram

### U 形 Scaling Law

在两个计算量级（$`C=2\times10^{20}`$ 和 $`C=6\times10^{20}`$ FLOPs）下实验，保持稀疏比例 $`P_{\text{tot}}/P_{\text{act}} \approx 10`$。

**关键发现**：验证损失与分配比例 $`\rho`$ 呈现一致的 **U 形关系**：

| 极端情况 | 问题 |
|---------|------|
| $`\rho \to 100\%`$（纯 MoE） | 缺乏静态模式的专用记忆，被迫通过深度和计算低效重建 |
| $`\rho \to 0\%`$（纯 Engram） | 丧失条件计算能力，动态上下文相关推理受损，记忆无法替代计算 |

**最优分配**：$`\rho \approx 75\%\text{--}80\%`$（分配约 20--25% 的稀疏预算给 Engram），在两个计算量级下稳定一致。在 10B 量级下，最优点验证损失从 1.7248（$`\rho=100\%`$）降至 1.7109（$`\Delta=0.0139`$）。

### 无限记忆 Scaling

在固定 MoE backbone（3B 参数，568M 激活）上，将 Engram 槽位数从 $`2.58\times10^5`$ 扩展到 $`1.0\times10^7`$（新增约 130 亿参数），验证损失呈 **严格幂律（对数线性）** 下降——更大记忆持续带来收益，无需额外计算。

## 长上下文实验

### 设置

基于 DeepSeek-V3 的上下文扩展策略，使用 YaRN（scale=10，α=1，β=32，f=0.707）进行 32768-token 上下文扩展训练（5000 步，30B tokens 高质量长文本数据）。

评估基准：**LongPPL**（书籍/论文/代码/长CoT）和 **RULER**（NIAH、VT、CWE、FWE、QA）。

### 关键结果（32k 上下文）

| 模型 | Multi-Query NIAH | Variable Tracking | FWE |
|------|-----------------|-------------------|-----|
| MoE-27B (50k steps) | 84.2 | 77.0 | 73.0 |
| Engram-27B (41k, ~82% FLOPs) | 89.5 | 83.2 | **99.6** |
| Engram-27B (46k, Iso-Loss) | **97.0** | 87.2 | 98.6 |
| Engram-27B (50k, Iso-FLOPs) | **97.0** | **89.0** | 99.3 |

**三层对比**：
1. **Iso-Loss**（46k vs MoE 50k）：控制基础能力后，Multi-Query NIAH：97.0 vs 84.2（+12.8），VT：87.2 vs 77.0（+10.2）
2. **Iso-FLOPs**（50k vs 50k）：全面领先
3. **极端设置**（Engram 41k vs MoE 50k）：仅用 82% 的预训练计算，LongPPL 持平，RULER 超越

## 机制分析

### 1. Engram 等效于增加模型深度

**假设**：Engram 通过释放 backbone 早期层的静态重建任务，等效地增加了模型的有效深度。

**验证工具一：LogitLens**
- 将各中间层隐藏状态投影到最终 LM Head，计算与最终输出分布的 KL 散度
- 结果：Engram 变体在早期层的 KL 散度**系统性更低**，曲线下降更陡峭，说明模型更快完成特征组合，更早达到高置信度预测

**验证工具二：CKA（中心化核对齐）**
- 计算 Engram 与 MoE 各层表示的相似度矩阵，引入软对齐指数 $`a_j`$：

```math
a_j = \frac{\sum_{i \in \mathcal{I}_j} S_{i,j} \cdot i}{\sum_{i \in \mathcal{I}_j} S_{i,j}}, \quad \mathcal{I}_j = \mathop{\text{argtop}k}_{i}(S_{i,j})
```

- 结果：相似度矩阵出现明显的**对角线上移**（$`a_j > j`$），例如 Engram-27B 第 5 层的表示与 MoE baseline 第 12 层最相似，**证实 Engram 在更浅层达到了更深的表示**

### 2. 消融研究

| 消融项 | 影响程度 |
|--------|---------|
| 去除多分支融合 | **最大回归** |
| 去除上下文感知门控 | **最大回归** |
| 去除 Tokenizer 压缩 | **最大回归** |
| 去除深度卷积 | 小幅下降 |
| 使用 4-gram（替代 2/3-gram） | 略微次优（固定预算下稀释容量） |

**最优注入位置**：Layer 2（单层最优），实际采用 Layer 2 + Layer 6 的双层设计（兼顾早期干预与丰富上下文门控）。

### 3. 敏感性分析（推理时屏蔽 Engram）

| 任务类型 | 保留性能比例 | 解读 |
|---------|------------|------|
| 事实知识（TriviaQA 等） | **29--44%** | Engram 是参数知识的主要存储库 |
| 阅读理解（C3 等） | **81--93%** | 上下文依赖任务主要依赖 backbone Attention |

### 4. 门控可视化

门控标量 $`\alpha_t`$ 在以下情况强烈激活（红色高亮）：
- **英文**：多 token 命名实体（"Alexander the Great"、"the Milky Way"）、惯用表达（"By the way"、"Princess of Wales"）
- **中文**：成语和历史实体（"四大发明"、"张仲景"）

证实 Engram 成功识别并处理了程式化语言依赖关系。

## 核心贡献总结

1. **条件记忆范式**：提出"条件记忆"作为条件计算（MoE）的互补稀疏轴，从根本上解决 LLM 用计算模拟记忆检索的低效问题

2. **Engram 模块**：现代化经典 N-gram 嵌入，集成 Tokenizer 压缩 + 多头哈希 + 上下文感知门控 + 多分支集成，实现 $`\mathcal{O}(1)`$ 常数时间查找

3. **U 形稀疏分配定律**：发现 MoE 与 Engram 的最优参数分配比约为 75--80%:20--25%，在多个计算量级下稳定，为混合架构设计提供可复现的原则

4. **基础设施感知效率**：确定性寻址使 100B 参数表可卸载到 Host DRAM，推理吞吐损失 <3%，有效绕过 GPU 显存限制

5. **机制理解**：通过 LogitLens + CKA 分析，揭示 Engram 等效于增加模型有效深度，并通过释放 Attention 容量显著提升长上下文能力
