## 研究背景与动机

### 问题定义

生成式检索（GR）通过自回归解码物品语义标识符（SID）来完成推荐候选生成。现有 GR 方法（如 TIGER、FORGE）均基于**下一个 token 预测（NTP）**范式，存在两个根本性缺陷：

**缺陷一：无法建模用户偏好的偏序结构**

NTP 在 token 级别将所有交互信号视为孤立的逐点样本，缺乏物品级别的显式建模，无法捕捉同一会话内物品之间的偏好层次。例如：

> 用户在同一 session 中：购买了外套和电脑 > 点击了扫帚 > 曝光但未点击薯片

NTP 无法区分这四类信号的偏好强度差异。

**缺陷二：候选物品与用户序列之间缺乏深层交互**

GR 在推理时通过单个隐状态与整个词表做内积来预测下一个 token，本质上是一种双塔近似，只能捕捉浅层交互，忽视了每个候选物品与完整历史序列之间的丰富关系。

---

## 核心方法：RankGR

**RankGR**（**Rank**-enhanced **G**enerative **R**etrieval）将检索过程分解为两个互补阶段：**初始评估阶段（IAP）** 和 **精化评分阶段（RSP）**，两者在同一 GR 模型下联合优化。

### 1. 问题形式化

设用户集合 $` \mathcal{U}=\{u_1,\ldots,u_{N_u}\} `$，物品集合 $` \mathcal{I}=\{i_1,\ldots,i_{N_i}\} `$。

每个物品 $` i `$ 通过 RQ-VAE 编码为 $` m `$ 个语义码字组成的 SID $` \{c^i_1, \ldots, c^i_m\} `$，第 $` l `$ 层码字的编码过程为：

```math
c^i_l = \arg\min_{c \in [1,m]} \left\| \mathcal{H}^{i}_l - q_l^c \right\|
```

其中 $` \mathcal{H}^{i}_l `$ 为第 $` l `$ 层的残差特征，$` \mathcal{H}^{i}_1 = \mathcal{H}^{i} `$ 为原始多模态特征。

标准 NTP 训练目标：

```math
\mathcal{L}_{\text{NTP}} = \frac{1}{N_{u}} \sum_{j=1}^{N_{u}} \sum_{t=1}^{T} \sum_{k=1}^{m} - \log P_{\{\theta, \tau\}} ( c_{k}^{i(t)} \mid x_{ \lt t}^{j}, c_{\lt k}^{i(t)} )
```

---

### 2. 初始评估阶段（IAP）

#### 物品级得分建模

GR 模型对第 $` t `$ 个物品的每个码字位置生成隐状态：

```math
h_{l}^{i(t)} = \mathcal{M}_{\text{GR}}(x_{\lt t}^{j}, c_{\lt l}^{i(t)}), \quad \forall l \in [1, m]
```

每层码字的概率通过内积+softmax计算：

```math
p_l(c^{i(t)}_l) = \text{softmax}({h^{i(t)}_l}^\top \text{emb}(c^{i(t)}_l))
```

物品 $` i(t) `$ 的最终 IAP 得分为各层 log 概率之和：

```math
\pi(i(t)) = \sum_{l=1}^{m} \log p_l(c^{i(t)}_l)
```

#### 列表式直接偏好优化（LDPO）

LDPO 将用户反馈划分为四个偏好层级：
- $` \mathcal{G}_4 `$：购买（最强偏好）
- $` \mathcal{G}_3 `$：点击
- $` \mathcal{G}_2 `$：曝光未点击
- $` \mathcal{G}_1 `$：伪曝光（排序模块选中但未实际展示）

**训练序列构造**：将用户历史与来自四个层级的目标物品拼接：

```math
x^j_{\text{train}}=[\underbrace{c^{i(1)}_1, \dots, c^{i(T)}_m}_{\text{用户历史}},\underbrace{c^{p_1}_1, \dots, c^{p_K}_m}_{\text{目标物品（多层级）}}]
```

**注意力掩码**：防止不同层级的目标物品相互干扰：

```math
M_{t_it_j} = 
\begin{cases} 
0, & \text{if } t_j \le t_i \;\land\; ( t_j \in x^j \lor \exists k, \{t_i, t_j\} \subseteq p_k ) \\
-\infty, & \text{otherwise}
\end{cases}
```

**LDPO 损失**：要求高层级物品得分严格高于低层级物品，采用 softmax 对比形式：

```math
\mathcal{L}_{\text{LDPO}} = -\sum_{j=1}^{3} \sum_{k \in \mathcal{G}_{j+1}} \log \left( \frac{e^{\beta \pi(k)}}{e^{\beta \pi(k)} + \sum_{i \in \mathcal{G}_1 \cup \cdots \cup \mathcal{G}_j} e^{\beta \pi(i)}} \right)
```

**IAP 总损失**（NTP + LDPO 联合优化）：

```math
\mathcal{L}_{\text{IAP}} = \mathcal{L}_{\text{NTP}} + \alpha \cdot \mathcal{L}_{\text{LDPO}}
```

超参数 $` \alpha `$ 控制偏好对齐的强度，实验表明 $` \alpha=1 `$ 时效果最佳，过大会损害基础自回归生成目标。

---

### 3. 精化评分阶段（RSP）

IAP 仍依赖内积近似，RSP 通过轻量级评分模块实现候选物品与输入序列的**深层交互**。

**候选保留**：在每一层 $` l `$ 保留 IAP 得分 Top-$` \lambda_l `$ 的码字：

```math
\mathcal{C}_{l}^{i(T)} = \text{Top-K}\left( \text{Softmax}(h_l^{i(T)}, \tau), K=\lambda_l \right)
```

**历史状态聚合**：将所有历史隐状态汇总为键值对：

```math
\mathbf{H} = \left\{ h_2^{i(0)}, \dots, h_1^{i(T-1)}, \dots, h_{l-1}^{i(T)}, h_l^{i(T)} \right\}
```

**Rank Head（目标注意力）**：以候选码字嵌入为 Query，历史隐状态为 Key/Value，计算深层交互：

```math
\tilde{h}_{l,\tilde{c}_l}^{i(T)} = \mathrm{Attention}(Q = \text{emb}(\tilde{c}_l), K = \mathbf{H}, V = \mathbf{H})
```

**MLP 打分**：

```math
s_{l,\tilde{c}_l}^T = \sigma(\mathrm{MLP}(\tilde{h}_{l,\tilde{c}_l}^{i(T)})) \in (0,1)
```

**BCE 训练损失**：

```math
\mathcal{L}_{\text{BCE}} = -\frac{1}{N_u} \sum_{j=1}^{N_u} \sum_{l=1}^{m} \sum_{\tilde{c}_l \in \mathcal{C}_{l}^{i(T)}} y_{l,\tilde{c}_l}^T\log(s_{l,\tilde{c}_l}^T) + (1-y_{l,\tilde{c}_l}^T)\log(1-s_{l,\tilde{c}_l}^T)
```

**推理时物品总得分**（各层 RSP 得分乘积）：

```math
s^T(\tilde{c}_1, \dots, \tilde{c}_m) = \sum_{l=1}^{m} \log s_{l,\tilde{c}_l}^T
```

**总训练损失**：

```math
\mathcal{L} = \mathcal{L}_{\text{BCE}} + \mathcal{L}_{\text{IAP}}
```

### 4. 生产部署架构

**异步预计算架构**（解决 LLM 推理延迟问题）：

| 组件 | 功能 |
|------|------|
| **Early Triggering** | 在用户请求生命周期开始时立即触发异步请求，创建 >100ms 的并行窗口 |
| **Prompt Construction** | 异步构建上下文感知 prompt，不阻塞主请求线程 |
| **LLM Inference** | 使用 RTP-LLM 框架优化，负载均衡 + KV Cache 复用 + 算子融合，推理延迟降至 **60ms** |
| **Redis Caching** | IAP+RSP 生成的候选物品存入 Redis，主流程到达检索引擎时直接读取缓存，实现伪实时检索 |

**流式训练与模型更新**：
- 实时交互数据通过 Message Queue 进入流式训练模块
- **每小时**将增量更新的权重同步回 LLM 推理平台
- 实现闭环在线学习，适应用户偏好的动态变化

## 实验设置

| 项目 | 内容 |
|------|------|
| **骨干模型** | Qwen2.5-0.5B-Instruct |
| **SID 生成** | 基于 FORGE 框架，2 层码字（2-level SID） |

| **学习率** | 线性调度，基础 lr = $` 5 \times 10^{-5} `$ |

**数据集**：

| 数据集 | 用户数 | 物品数 | 交互数 | 稀疏度 |
|--------|--------|--------|--------|--------|
| Amazon (Clothing) | 252.5 万 | 71.5 万 | 1253 万 | 99.98% |
| Taobao | 2 亿 | 10 亿 | 0.9 万亿 | 99.99% |

**评估指标**：HR@K（Hit Rate），Taobao 数据集分 Click 和 PV（Page View）两个维度报告。

|------|-------|-------|--------|--------|

| **RankGR** | **2.27%** | **3.13%** | **4.68%** | **9.96%** |

> RankGR 在小 K 值指标上提升最显著，说明 RSP 有效将相关物品推到更高排名。

### 消融实验（Taobao Click HR@20）

| 变体 | HR@20 | 相对下降 |
|------|-------|---------|
| **RankGR（完整）** | **12.28%** | — |
| w/o IAP（仅 RSP） | 10.04% | -18.28% |
| w/o RSP（仅 IAP） | 10.99% | -10.44% |
| w/o Both（基础 GR） | 9.54% | -22.34% |

**关键发现**：
- IAP 贡献大于 RSP（-18.28% vs -10.44%），说明偏序偏好建模是核心
- RSP 在小 K 值指标（HR@20）上贡献相对更大，印证其对高排名物品的精化作用
- 两者互补，缺一性能均大幅下降

### 超参数分析

- **$` \alpha `$ 最优值为 1**：$` \alpha `$ 从 0.5 增至 1 时 Click 和 PV 均提升；超过 1 后过度强调偏好对齐，损害基础自回归目标
- **$` \lambda_2 `$ 需足够大**：$` \lambda_2=60 `$ 时性能大幅下降，说明 IAP 过早剪枝会丢失相关物品；$` \lambda_2=1400 `$ 时效果最佳

### 在线 A/B 测试（淘宝"猜你喜欢"，1月20日–27日，5%流量）

| 指标 | Base | RankGR |
|------|------|--------|
| PVR（曝光覆盖率） | 36.79% | **49.88%** |
| IPV（商品页面浏览量） | — | **+1.08%** |
| Transaction Count | — | **+0.57%** |

> RankGR 在所有部署检索方法中贡献了约 50% 的曝光，同时提升了用户参与度和商业指标。

## 核心贡献总结

1. **LDPO（列表式直接偏好优化）**：首次将列表级偏好对齐引入生成式检索，通过四层偏好层级（购买>点击>曝光>伪曝光）的对比损失，无需额外奖励模型即可实现端到端偏序建模

2. **RSP（精化评分阶段）**：通过轻量级 Rank Head（目标注意力 + MLP）实现候选码字与完整历史序列的深层交互，突破内积近似的局限

3. **两阶段联合优化框架**：IAP 负责粗粒度偏好对齐，RSP 负责细粒度候选精化，两者共享同一 GR 骨干模型联合训练，兼顾效率与效果

4. **生产级部署方案**：异步预计算架构（60ms LLM 推理 + Redis 缓存）+ 流式训练（每小时模型更新），支持近万 QPS 的实时推荐服务

## 与相关工作的关键区别

| 对比维度 | RankGR | TIGER/FORGE | OneRec-Think/GR2 |
|---------|--------|-------------|-----------------|
| 偏好建模粒度 | 列表级（4 层偏好层级） | token 级 NTP | Pairwise DPO |
| 候选精化 | RSP 目标注意力（深层交互） | 无 | 无 |
| 奖励模型 | 不需要（直接利用用户反馈） | 不需要 | 需要（GR2 用 RL） |
| 部署优化 | 异步预计算 + Redis 缓存 + 流式训练 | 未详述 | 未详述 |
| 工业验证规模 | 淘宝首页，0.9 万亿次交互 | 学术数据集为主 | Meta/快手内部 |

## 局限性与未来工作

- 目前 SID 生成依赖 RQ-VAE，量化损失对不同模态特征的影响尚未深入分析
- LDPO 中四个偏好层级的划分基于经验，更细粒度的偏好信号（如停留时长、加购等）尚未利用
- RSP 的轻量级设计在极大候选空间下的扩展性有待进一步验证
