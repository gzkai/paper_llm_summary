## 研究背景与动机

### 传统多阶段级联架构（MCA）的局限

电商搜索系统传统上采用**多阶段级联架构（Multi-stage Cascading Architecture, MCA）**：

```
召回（~10⁹ 候选）→ 预排序（~10⁴）→ 精排（~10²）→ 最终结果
```

MCA 虽然在计算效率和业务转化之间取得平衡，但存在两大根本性缺陷：

| 缺陷 | 描述 |
|------|------|
| **碎片化计算（Fragmented Compute）** | 大量服务资源消耗在通信和存储上，而非数值计算；MFU 仅 3.26% |
| **目标冲突（Objective Collision）** | 召回/预排序目标是"尽量召回相关物品"，精排目标是"精准排序"，多层漏斗过滤导致真正相关的物品可能在早期阶段被过滤，后续再精准的模型也无法挽救 |

### 电商搜索的三大独特挑战

与视频推荐（OneRec）不同，电商搜索面临额外挑战：

1. **物品信息冗余噪声多**：商家为增加曝光会添加大量不相关关键词，语义顺序弱，编码器难以提取核心信息
2. **强相关性约束**：搜索 Query 通常只有 2-3 个短关键词，任何属性不匹配都会导致严重相关性问题；传统 SID 会丢失核心属性信息
3. **用户潜在搜索意图推断**：用户输入简短 Query 时，需要结合行为序列推断真实搜索意图

## 核心方法：OneSearch

OneSearch 是**首个工业部署的端到端生成式电商搜索框架**，包含三大核心创新模块：

### 整体架构

```
输入：[用户ID | 查询 Query | 查询SID | 短行为序列 | 长行为序列(隐式) | 用户画像]
         ↓
   Encoder（BART-based）
         ↓
   Decoder（自回归生成）
         ↓
输出：物品 SID 列表（Beam Search，size=512）
         ↓
   奖励模型重排（可选）
         ↓
最终推荐物品列表
```

---

### 创新一：KHQE — 关键词增强层级量化编码

#### 1.1 对齐协同与语义表示

从真实用户搜索日志中提取高质量 query2query、item2item、query2item 对，使用蒸馏 BGE 模型生成内容 Embedding $e_q$ 和 $e_i$，设计四类对齐任务：

```math
\mathcal{L}_{\text{align}} = \lambda_1 \cdot \mathcal{L_{q2q}} + \lambda_2 \cdot \mathcal{L_{i2i}} + \lambda_3 \cdot \mathcal{L_{q2i}} + \lambda_4 \cdot \mathcal{L_{rank}} + \lambda_5 \cdot \mathcal{L_{rel}}
```

- $`\mathcal{L_{q2q}}`$, $`\mathcal{L_{i2i}}`$：协同相似对的对比损失
- $`\mathcal{L_{q2i}}`$：确保 BGE 反映真实业务特性
- $`\mathcal{L_{rank}}`$：区分不同行为层级（展示/点击/下单）的 query-item 对
- $`\mathcal{L_{rel}}`$：用 LLM 对相似度阈值附近的对打相关性分，让 BGE 拟合

#### 1.2 核心关键词增强

**问题**：物品文本含大量冗余词，语义顺序弱，编码器难以提取核心属性。

**解决方案**：
- 用 NER 识别 **18 类结构化属性**（实体/修饰词/品牌/材质/风格/功能/位置/受众/颜色/营销/季节/图案/场景/规格/价格/型号/主播/系列）
- 挖掘过去 1 年点击 query-item 对作为标注数据，按 PV 降序筛选高频关键词
- 用 Qwen-VL 为每个物品识别对应关键词；Query 用 Aho-Corasick 自动机实时匹配
- 最终表示为关键词向量与内容向量的平均：

```math
e_{q}^{o} = \frac{1}{2} \left(e_{q} + \frac{1}{m}\sum_{i=1}^{m}e_{k}^{i}\right), \quad e_{i}^{o} = \frac{1}{2} \left(e_{i} + \frac{1}{n}\sum_{j=1}^{n}e_{k}^{j}\right)
```

**效果**：以 4096-1024-512 配置为例，关键词增强使 Level 2 码本利用率（CUR）提升 24.84%，Level 3 提升 26.15%，独立编码率（ICR）提升 6.86%。

#### 1.3 RQ-OPQ 层级量化 Tokenizer

**问题**：RQ-KMeans 优先编码共享特征，丢失每个物品的独特属性（残差信息）。

**解决方案**：RQ-KMeans + OPQ 联合 Tokenizer：

- **RQ-KMeans**（3层）：处理层级语义，码本配置 (4096, 1024, 512)
  - 第 3 层使用 balanced k-means（仅对第 3 层，避免全层均衡导致的层级聚类崩溃）
- **OPQ**（2层）：对 RQ-KMeans 最后一层的**残差 Embedding** 量化，保留独特属性，码本配置 (256, 256)
- 最终 SID 长度为 5 位：`(4096, 1024, 512 | 256, 256)`

| 方法 | ICR | Recall@10 | MRR@10 |
|------|-----|-----------|--------|
| OnlineMCA | - | 0.3440 | 0.1323 |
| RQ-VAE | 38.83% | 0.2171 | 0.0689 |
| RQ-Kmeans | 68.08% | 0.2844 | 0.1038 |
| **RQ-OPQ** | **91.91%** | **0.3369** | **0.1194** |

### 创新二：Mu-Seq — 多视角用户行为序列注入

从三个视角建模用户行为：

#### 2.1 行为序列构造的用户 ID（Behavior Sequence-Constructed User IDs）

**问题**：Tiger 的随机哈希 User ID 无法表达用户个性化（不同用户可能被分配同一 ID）。

**解决方案**：用加权衰减的点击行为序列构造 User ID：

```math
SID_{short} = \left\lceil \sum_{i=1}^{m} \lambda_i \cdot SID_{s_i} \right\rceil, \quad \lambda_i = \frac{\exp(\sqrt{i})}{\sum_{i}^{m}\exp(\sqrt{i})}
```

```math
SID_{long} = \left\lceil \sum_{j=1}^{n} \mu_j \cdot SID_{l_j} \right\rceil, \quad \mu_j = \frac{\exp(\sqrt{j})}{\sum_{j}^{n}\exp(\sqrt{j})}
```

User ID 总长度为 10（$`SID_{short}`$ + $`SID_{long}`$ 拼接）。冷启动用户使用基于 Query-物品共现统计的默认行为序列。

#### 2.2 显式短行为序列（Explicit Short Behavior Sequence）

直接将用户最近的搜索 Query 序列 $`Seq_{query}`$ 和点击物品序列 $`Seq_{short}`$ 的 SID 拼接到 Prompt 中，让模型显式学习近期偏好。

同时采用**滑动窗口数据增强**：对 $`Seq_{short} = \{s_1, ..., s_m\}`$ 生成 $m$ 条训练样本，帮助模型学习用户兴趣变化，并处理历史较少的新用户。

#### 2.3 隐式长行为序列（Implicit Long Behavior Sequence）

**问题**：长行为序列（点击/下单/搜索相关单元，长度可达 $`10^3`$）无法直接拼接到 Prompt。

**解决方案**：聚合 RQ 聚类中心向量，用 QFormer 压缩：

```math
\mathbf{M}_{click} = \left\{\sum_{i=1}^{m} \mathbf{Item}_{emb}^{L_1}, \sum_{i=1}^{m} \mathbf{Item}_{emb}^{L_2}, \sum_{i=1}^{m} \mathbf{Item}_{emb}^{L_3} \right\}
```

三类行为序列（点击 $`\mathbf{M}_{click}`$、下单 $`\mathbf{M}_{order}`$、搜索相关单元 $`\mathbf{M}_{RSU}`$）通过 QFormer 聚合为 $`\mathbf{Q} \in \mathbb{R}^{N_M \times 768}`$，作为隐式长行为表示输入模型。

### 创新三：PARS — 偏好感知奖励系统

#### 3.1 三阶段监督微调（Multi-stage SFT）

| 阶段 | 目标 | 任务 |
|------|------|------|
| **Stage 1：语义内容对齐** | SID ↔ 文本互转 | query/item文本→SID；SID→文本；文本→类目 |
| **Stage 2：共现同步** | 学习 Query-物品协同语义 | query↔item互预测；query_SID↔item_SID互预测 |
| **Stage 3：用户个性化建模** | 引入完整用户信息 | [uid + q + SID_q + Seq_q + Seq_short + Seq_long_emb] → item_SID |

#### 3.2 自适应奖励系统（Adaptive Reward System）

**用户行为分级**（6 级）及基础奖励权重：

| 级别 | 行为 | 基础权重 |
|------|------|---------|
| 1 | 搜索场景购买 | 2.0 |
| 2 | 推荐场景同类购买 | 1.5 |
| 3 | 点击 | 1.0 |
| 4 | 曝光未点击 | 0.5 |
| 5 | 同类未曝光 | 0.2 |
| 6 | 其他类别随机物品 | 0.0 |

**偏差校正**：原始 CTR/CVR 存在统计偏差（新品 CTR 虚高），使用对数平滑校正：

```math
Cnt_{T} = \log((Cnt_{pos}+10) \cdot (Cnt_{clk}+10) \cdot (Cnt_{order}+10))
```

```math
Ctr_i = \frac{\log(Cnt_{clk} + 10)}{Cnt_{T}}, \quad Cvr_i = \frac{\log(Cnt_{order} + 10)}{\log(Cnt_{clk} + 10)}
```

**加权奖励分**（调和平均）：

```math
r(q, i) = 2\lambda \cdot \frac{Ctr_i \cdot Cvr_i}{Ctr_i + Cvr_i}
```

**偏好差异权重**（差距越小，越鼓励区分细微差异）：

```math
rw_{\Delta} = \frac{1.0}{r(q, i_{pos}) - r(q, i_{neg})}
```

#### 3.3 奖励模型（三塔 SIM 架构）

基于 SIM（Search-based Interest Model）的三塔架构，分别学习 CTR、CVR、CTCVR：

```math
Rscore = \lambda_1 \cdot CTR + \lambda_2 \cdot CVR + \lambda_3 \cdot CTCVR + 10 \cdot \lambda_4 \cdot S_{Rel}
```

其中相关性分 $`S_{Rel}`$ 放大 10 倍，确保生成物品满足相关性约束。

#### 3.4 混合排序框架（Hybrid Ranking Framework）

**两阶段 RL 训练**：

**第一阶段**（奖励模型引导）：收集真实搜索日志 → 用奖励模型对 SFT 模型输出重排 → 选取排名发生变化的样本 → List-wise DPO 训练：

```math
\mathcal{L} = -\mathbb{E} \left[ \log \sigma \left( \log \sum_{i_l \in \mathcal{I}_l} \exp \left( rw_{\Delta} \max \left(0, \hat{r}_\theta(x_u, i_w) - \hat{r}_\theta(x_u, i_l) - \delta \right) \right) \right) + \alpha \log\pi_\theta\left(i_w | x_u \right) \right]
```

其中隐式奖励：

```math
\hat{r}_\theta(x_u, i_{w/l}) = \beta \log \frac{\pi_\theta(i_{w/l}|x_u)}{\pi_{\text{ref}}(i_{w/l}|x_u)}
```

**第二阶段**（纯用户交互反馈）：用真实用户交互数据（前 3 级为正样本，后 3 级为负样本）继续训练，尽量接近流式更新，突破奖励模型的性能上限。

**更新频率**：多阶段 SFT 每周一次；奖励模型 RL 每天一次（购物节期间效果显著）；用户交互偏好对齐尽量接近流式。

## 实验结果

### 离线性能对比

**数据集**：快手商城搜索平台 2025 年 5-8 月用户交互数据，约 10 亿 PV，训练 90 天，测试最后 1 天。

**评估**：30k 点击对 + 30k 下单对，HR@350 和 MRR@350（Top 350 物品）

| 方法 | Order HR@350 | Order MRR@350 | Click HR@350 | Click MRR@350 |
|------|-------------|--------------|-------------|--------------|
| OnlineMCA | 51.74% | 19.26% | 64.40% | 16.89% |
| MCA w/o ranking | 75.75% | 4.19% | 80.23% | 3.00% |
| RQ-Kmeans (4096-1024-512) | 59.58% | 14.29% | 62.49% | 11.82% |
| +keywords | 62.38% | 14.30% | 66.14% | 12.10% |
| +l3 balanced | 63.16% | 13.59% | 68.26% | 11.67% |
| +Adaptive RS | 64.33% | 16.11% | 68.94% | 13.80% |
| RQ-OPQ (2/256) | 65.05% | 15.33% | 68.88% | 12.90% |
| **RQ-OPQ + Adaptive RS（OneSearch）** | **66.46%** | **18.38%** | **71.06%** | **16.33%** |

**关键发现**：
- OneSearch 召回率（HR@350）比 OnlineMCA 高 **14.72%**（order），MRR 接近（18.38% vs 19.26%）
- MCA w/o ranking 的高 HR 低 MRR 验证了目标冲突问题：预排序只召回不排序，精排只排序不召回

### 消融实验：多视角行为序列注入

| 变体 | Order HR | Order MRR | Click HR | Click MRR |
|------|---------|----------|---------|----------|
| **OneSearch（完整）** | **66.46%** | **18.38%** | **71.06%** | **16.33%** |
| w/o User SIDs（→ Hashing ID） | -0.94% | -0.37% | -1.72% | -0.36% |
| w/o $`Seq_{short}`$ | -3.43% | -1.53% | -4.15% | -1.32% |
| w/o $`Seq_{long}^{emb}`$ | -2.26% | -1.01% | -3.00% | -1.05% |
| w/o 滑动窗口增强 | -1.95% | -0.81% | -1.80% | -0.70% |

**关键发现**：短行为序列贡献最大（HR 平均 -3.79%，MRR 平均 -1.43%）；长行为序列次之；行为序列构造的 User ID 优于随机哈希 ID。

### 消融实验：OPQ 编码配置

| 配置 | Order HR@10 | Order MRR@10 | Click HR@10 | Click MRR@10 |
|------|------------|-------------|------------|-------------|
| **RQ-OPQ (2/256)** | **28.42%** | **14.15%** | **33.69%** | **11.94%** |
| RQ-OPQ (4/256) | -2.36% | -1.77% | -2.52% | -1.56% |
| RQ-OPQ (4*2/256) | -10.20% | -5.57% | -11.77% | -3.84% |
| RQ-OPQ (4*4/256) | -24.18% | -11.83% | -27.11% | -9.61% |

最优配置为 2 层 OPQ（256-256），更多层导致性能急剧下降（类似全层 balanced k-means 的层级聚类崩溃问题）。

### 在线 A/B 测试

在快手商城搜索平台进行严格 A/B 测试：

| 方法 | Item CTR | PV CTR | PV CVR | Buyer | Order |
|------|---------|--------|--------|-------|-------|
| MCA w/o ranking | -9.97% | -20.33% | -11.55% | -28.78% | -39.14% |
| $`OneSearch^1`$（无 RQ-OPQ + 长序列） | -1.10%* | -2.06%* | +0.39%* | +1.27%* | -2.22%* |

| 对象 | 暖物品/用户 | 冷物品/用户 | 平均 |

冷启动场景增益均高于暖场景，验证了生成式框架的语义推断能力。

## 核心贡献总结

1. **KHQE（关键词增强层级量化编码）**：
   - 五任务对齐训练（q2q/i2i/q2i/rank/rel）融合协同与语义信号
   - NER 提取 18 类核心关键词增强 Embedding，提升 ICR 6.86%
   - RQ-OPQ 联合 Tokenizer（3层RQ + 2层OPQ），ICR 达 91.91%，远超 RQ-VAE（38.83%）

2. **Mu-Seq（多视角用户行为序列注入）**：
   - 行为序列构造 User ID（加权衰减，长度10）
   - 显式短序列（Prompt 拼接 + 滑动窗口增强）
   - 隐式长序列（RQ 聚类中心聚合 + QFormer 压缩）

3. **PARS（偏好感知奖励系统）**：
   - 三阶段 SFT（语义对齐 → 共现同步 → 用户个性化）
   - 6 级行为分级 + 对数平滑偏差校正的自适应奖励
   - 三塔 SIM 奖励模型（CTR/CVR/CTCVR + 相关性）
   - 混合排序：List-wise DPO（奖励模型引导）+ 纯用户交互偏好对齐

4. **首个工业部署的端到端生成式电商搜索框架**：
   - Item CTR +1.67%，Order +3.22%，Buyer +2.40%
   - MFU 3.26% → 27.32%（+700%），OPEX 节省 75.40%

## 与快手 GR 家族的关联

| 系统 | 场景 | 核心特点 |
|------|------|---------|
| OneRec V1/V2 | 视频推荐 Feed | RQ-KMeans + 编码器-解码器/Lazy Decoder-Only |
| OneLive | 直播推荐 | 动态 Tokenizer + 时间感知注意力 |
| OneLoc | 本地生活 | 地理感知 SID + 邻居感知 Prompt |
| OneSug | 查询建议 | Prefix2Query + 奖励加权排序 |
| **OneSearch** | **电商搜索** | **KHQE + Mu-Seq + PARS，首个 E2E 搜索 GR** |

OneSearch 是快手 GR 家族向**电商搜索**领域的首次延伸，解决了搜索特有的强相关性约束和物品噪声问题。
