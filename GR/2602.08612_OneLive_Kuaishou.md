## 研究背景与动机

### 工业推荐系统的两大痛点

**① 级联架构的目标错位**

传统推荐系统采用"检索 → 粗排 → 精排"级联流程，各阶段目标不一致：检索追求覆盖率，精排追求准确率。上游过早过滤的优质内容无法被下游恢复，形成不可逆的信息瓶颈。

**② 碎片化计算图导致 MFU 极低**

传统推荐模型由稀疏 Embedding 查找、序列建模、特征交叉等异构模块拼接，计算图高度不规则，内存访问主导计算，**MFU（模型浮点利用率）仅约 2.56%**（在线排序模型实测）。

### 直播推荐的四大独特挑战

直播场景无法直接套用现有 GR（生成式推荐）方法，原因如下：

| 挑战 | 说明 |
|------|------|
| **动态内容** | 主播实时切换才艺（唱歌、跳舞、聊天），内容持续漂移，静态 SID 失效 |
| **有限生命周期** | 直播只在播出期间可被推荐，必须在"黄金曝光窗口"内完成分发 |
| **实时响应约束** | 高并发、低延迟，要求模型在保持精度的同时具备高吞吐 |
| **多目标异构偏好** | 用户对点击、长看、送礼、关注等行为的偏好高度异构 |

## 核心方法：OneLive 框架

OneLive 由三大模块组成：**动态 Tokenizer**、**高效生成架构**、**多目标强化学习对齐**。

---

### 1. 动态 Tokenizer

#### 1.1 语义与协同动态对齐（IA Embedding）

**两阶段范式**：

**阶段一：动态内容理解**
- 采用 MLLM 对直播内容进行多模态理解，以 **30 秒滑动窗口**实时更新内容 Embedding
- 通过知识蒸馏训练轻量级 MLLM，持续摄入实时片段生成动态嵌入

**阶段二：实时协同后对齐**
- 在动态内容 Embedding 基础上，通过双塔架构融入即时用户交互反馈（评论、送礼、点击）

作者塔中，将静态属性 $` x^{AId} \in \mathbb{R}^d `$、实时内容嵌入 $` x^{MLLM}_{30s} `$ 和长期平均嵌入 $` x^{MLLM}_{pooling} `$ 通过门控机制融合：

```math
x^{MLLM} = \text{MLP}(x^{MLLM}_{30s} \oplus x^{MLLM}_{pooling})
```

```math
x^{Author} = \lambda x^{AId} + (1 - \lambda) x^{MLLM}
```

其中 $` \lambda `$ 为可学习权重，平衡异构输入。

用户塔与用户-作者对齐：

```math
x^{User} = \text{MLP}(x^{UId})
```

```math
\mathcal{L}_{Alignment} := \text{In-Batch-Softmax}(x^{User}, x^{Author})
```

最终输出 **IA Embedding** $` x^{IA} := x^{Author} `$，同时包含语义信息与协同信号。

#### 1.2 残差量化（Res-Kmeans）

采用 **Res-Kmeans** 替代 RQ-VAE，对 IA Embedding 进行逐层残差量化：

```math
q^{l} = \arg\min_k \lVert c_k^l - x^l \rVert
```

其中 $` c^l_k \in \mathcal{C}^{l} = \text{Kmeans}(x^{l}, N_{l}) `$ 为第 $` l `$ 层码本，$` x^{l+1} = x^{l} - c^{l}_{q^l} `$ 为残差，迭代 $` T `$ 次得到 SID $` \{q^0, q^1, \dots, q^T\} `$。

**量化质量对比**（约 400 万主播）：

| 类型 | 方法 | 码本大小 | $` L_0 \times L_1 `$ UR | SID CR | Author CR |
|------|------|---------|------------------------|--------|-----------|
| MLLM | Res-Kmeans | (512,512,512) | 53.83% | 28.10% | 63.83% |
| MLLM | Res-Kmeans | (8192,8192,8192) | 2.34% | 3.56% | 9.47% |
| IA | RQ-VAE | (8192,8192,8192) | 1.16% | 8.54% | 22.78% |
| **IA** | **Res-Kmeans** | **(8192,8192,8192)** | **4.50%** | **0.66%** | **1.76%** |

**结论**：IA Embedding + Res-Kmeans + 大码本（8192）组合码本利用率 100%、碰撞率最低（SID CR 仅 0.66%），最终选用 $` T=3 `$ 层、$` N_l=8192 `$ 的配置。

### 2. 生成式推荐架构

#### 2.1 Lazy Decoder-Only 基础架构

直播场景用户行为序列多且长（有效观看 $` S^{Eff} `$、送礼 $` S^{Gift} `$、短视频 $` S^{Video} `$ 等），总长度远超目标 SID（如 $` 3 \times 500 \gg 3 `$）。传统 Encoder-Decoder 架构将大量计算浪费在编码冗余历史上。

OneLive 采用 **Lazy Decoder-Only** 架构，将历史序列投影为 KV 上下文：

```math
x_i = \text{Concat}(x_i^{\text{AId}}, x_i^{\text{IA}}, \dots) \in \mathbb{R}^d
```

```math
E = \text{MLP}(\{x_0, x_1, \dots, x_{|S|}\}) \in \mathbb{R}^{|S| \times d}
```

Decoder 由 $` L `$ 层 Transformer Block 组成，每层包含交叉注意力、自注意力和 FFN：

```math
h^l = h^{l-1} + \text{CrossAttn}(q=\text{RMSNorm}(h^{l-1}), k=E, v=E)
```

```math
h^l = h^{l} + \text{SelfAttn}(\text{RMSNorm}(h^{l}))
```

```math
h^l = h^{l} + \text{FFN}(\text{RMSNorm}(h^{l}))
```

#### 2.2 时间感知门控注意力（Time-Aware Gated Attention）

从三个维度建模时序动态：

**① 历史序列时间感知**：为每个历史物品注入多粒度时间偏置（小时/天/周）：

```math
x_{i\_ta} = x_i + \text{MLP}\left(\text{Concat}(x_i^{\text{Hour}}, x_i^{\text{Day}}, x_i^{\text{Week}})\right)
```

**② 生成锚点时间感知**：为 [BOS] token 注入查询时刻的时间特征，实现时间感知的条件生成：

```math
x_{[\text{BOS}]\_ta} = x_{[\text{BOS}]} + \text{MLP}\left(\text{Concat}(x_q^{\text{Hour}}, x_q^{\text{Day}}, x_q^{\text{Week}})\right)
```

**③ 注意力门控时间感知**：在交叉注意力和自注意力中引入可学习门控分数，自适应调制注意力权重：

```math
\text{Score}(X) = \sigma(XW_{\theta})
```

```math
O = \left(\text{MultiHeadAttn}(XW_Q, X'W_K, X'W_V) \odot \text{Score}(X)\right)W_O
```

时间感知门控注意力使在线主播的有效率提升 **14.29%**。

#### 2.3 Sequential Multi-Token Prediction（Sequential MTP）

**动机**：标准自回归推理中，$` q^1 `$ 和 $` q^2 `$ 是 $` q^0 `$ 的残差分量，预测任务相对简单，但需要依次扩大 beam size 进行推理，计算开销随 beam size 急剧增大。

**方案**：设计三个串联 Decoder 共享 SID Embedding 层：
- **主 Decoder**：保留完整 $` L `$ 层 Transformer Block，预测 $` q^0 `$
- **轻量 Decoder（×2）**：单层 Block，参数共享主 Decoder 第一层（除 FFN），预测 $` q^1, q^2 `$

轻量 Decoder 的输入为前一模块隐状态与当前 SID Embedding 的拼接：

```math
h_i^{k} = \text{MLP}(\text{Concat}(\{h_i^{' k-1}, \text{Emb}(q^{i})\}))
```

```math
h_{i:2}^{' k} = \text{MTP}_{k}(h_{i:2}^{k})
```

总训练损失为多个 MTP 模块损失的加权融合：

```math
\mathcal{L}_{MTP} := w_0\mathcal{L}_{\text{MTP}_{main}} + w_1\mathcal{L}_{\text{MTP}_{1}} + w_2\mathcal{L}_{\text{MTP}_{2}}
```

**推理加速效果**（3 层模型）：

| 变体 | QPS 提升 | 平均延迟 | P99 延迟 |
|------|---------|---------|---------|
| Lazy Dec-Only | — | 5.05ms | 8.48ms |
| + MTP | **+31.8%** | **2.25ms** | **3.62ms** |

> MFU 从在线排序模型的 **2.56%** 提升至 **22.78%**（L20 GPU）

#### 2.4 QK Norm 训练稳定性优化

深层模型训练时，QK logits $` QK^T `$ 随训练进行持续增大，导致 Softmax 趋近 one-hot 分布，梯度消失，训练崩溃。

引入 **QK Norm**，在计算注意力前对 Q 和 K 分别施加 RMSNorm：

```math
\text{Attn}(Q,K,V) = \text{Softmax}\left(\frac{\text{RMSNorm}(Q) \cdot \text{RMSNorm}(K)^T}{\sqrt{d}}\right)V
```

QK Norm 有效抑制 logit 爆炸，消除长时间训练中的 loss 尖峰现象。

### 3. 多目标强化学习对齐

#### 3.1 奖励模型（Pantheon）

采用多目标集成排序模型 **Pantheon** 作为奖励模型，通过加权 BCE 损失联合优化 CTR、LVTR（长看率）、GTR（送礼率）等多目标：

```math
Score = \text{Pantheon}(\text{User}, \text{Author})
```

```math
\mathcal{L}_{Pantheon} := \sum_{XTR \in \{CTR, LVTR, GTR, \dots\}} w^{XTR}\mathcal{L}^{XTR}_{Pantheon}
```

直接使用 $` r := Score `$ 作为每个候选主播的奖励，无需手工设计固定权重，自适应用户偏好异构性。

#### 3.2 优化策略：DPO vs. GRPO

对每个 query $` q `$，参考模型生成响应集合 $` \{o_1, \dots, o_G\} `$，奖励模型打分后归一化得到优势分数 $` A_i = \frac{r_i - \text{mean}(r)}{\text{std}(r)} `$。

**DPO**：选最高优势为正样本、最低优势为负样本：

```math
\mathcal{L}_{\mathrm{DPO}} := -\sigma\left(\lambda\log\frac{\pi_{\theta}(o_{pos}|q)}{\pi_{\theta ref}(o_{pos}|q)} - \lambda\log\frac{\pi_{\theta}(o_{neg}|q)}{\pi_{\theta ref}(o_{neg}|q)}\right)
```

**GRPO**：利用组内所有响应的相对优势进行策略优化：

```math
\mathcal{L}_{\mathrm{GRPO}} := \frac{1}{G} \sum^G_{i=1}\min\left(\frac{\pi_{\theta}(o_{i}|q)}{\pi_{\theta ref}(o_{i}|q)}A_i,\ \text{clip}\left(\frac{\pi_{\theta}(o_{i}|q)}{\pi_{\theta ref}(o_{i}|q)}, 1-\epsilon, 1+\epsilon\right)A_i\right)
```

**最终总损失**：

```math
\mathcal{L}_{OneLive} := \mathcal{L}_{MTP} + w\mathcal{L}_{RL}
```

RL 训练中每轮仅采样 **1% 的 query** 运行策略优化，保证训练稳定性。

## 核心贡献总结

1. **Dynamic Tokenizer**：首次将 MLLM 动态内容理解（30s 滑动窗口）与实时协同后对齐（双塔 + In-Batch 对比）结合，生成兼具语义与协同信号的 IA Embedding，配合 Res-Kmeans 大码本（8192）实现近零碰撞量化

2. **Time-Aware Gated Attention**：从历史序列、生成锚点、注意力门控三个维度建模时序动态，有效应对直播有限生命周期约束，在线有效率提升 14.29%

3. **Sequential MTP**：三模块串联共享 KV Cache，推理 QPS 提升 31.8%，延迟降低 55.45%，MFU 从 2.56% 提升至 22.78%；**QK Norm** 解决深层模型训练中的 logit 爆炸问题

4. **多目标 RL 对齐**：以 Pantheon 集成排序模型为奖励，GRPO 策略优化统一多目标偏好对齐，无需手工权重调参，自适应用户异构偏好

5. **工业级验证**：快手 App + 快手极速版双平台部署，服务 **4 亿日活**，在线 A/B 测试持续近一个月，所有核心业务指标显著提升

## 局限性与未来工作

- MLLM 动态 Embedding 的 30s 窗口粒度与在线延迟之间的平衡仍有优化空间
- 参数规模实验呈现边际收益递减，更大规模模型的部署成本需进一步探索
- 未来将继续探索更高效的直播生成式推荐方法
