## 研究背景与动机

### 生成式推荐（GR）的工业进展

OneRec 等工作已将生成式推荐（GR）成功落地工业场景，通过残差量化将 item 编码为语义 ID（SID），并用统一 Transformer 骨干进行自回归推荐，替代传统"召回-粗排-精排"级联流水线。然而，这类方法无法直接迁移到**直播推荐**场景。

### 直播推荐的四大独特挑战

| 挑战 | 描述 | 对 GR 的影响 |
|------|------|-------------|
| **动态内容（Dynamic Content）** | 主播实时切换活动（聊天/唱歌/跳舞），内容持续变化 | 静态 SID 无法捕捉直播内容漂移 |
| **有限生命周期（Limited Lifecycle）** | 直播从开始到结束通常在短时间内完成，候选池实时变化 | 必须在黄金曝光窗口内完成推荐，保证候选时效性 |
| **实时响应约束（Real-Time Constraint）** | 高并发、严苛延迟要求 | 对推理效率和吞吐量要求极高 |
| **多目标（Multiple Objectives）** | 用户反馈包括点击、长观看、分享、关注、礼物等 | 需融合异构多目标信号进行个性化对齐 |

## 核心方法：OneLive

OneLive 由三大核心模块组成：**动态分词器**、**高效生成架构**、**多目标强化学习对齐**。

```
OneLive 整体架构
├── 1. 动态分词器（Dynamic Tokenizer）
│   ├── 动态内容理解：MLLM（30s 滑动窗口）→ 实时内容嵌入
│   ├── 实时协作后对齐：双塔架构（用户塔 + 作者塔）→ IA Embedding
│   └── 残差量化：Res-Kmeans（T=3层，N_l=8192）→ SID {q⁰, q¹, q²}
│
├── 2. 生成式推荐器（Generative Recommender）
│   ├── 基础架构：Lazy Decoder-Only（跨注意力 + 自注意力 + FFN）
│   ├── 时序动态建模：时间感知门控注意力（TA Gated-Attn）
│   ├── 推理加速：Sequential MTP（多 token 预测）
│   └── 训练稳定：QK Norm（RMSNorm 作用于 Q/K）
│
└── 3. 强化学习对齐（Reinforcement Learning）
    ├── 奖励模型：Pantheon（多目标集成排序模型）
    └── 优化策略：GRPO（Group Relative Policy Optimization）
```

### 模块一：动态分词器（Dynamic Tokenizer）

#### 1.1 语义与协作动态对齐

**两阶段范式**：

**① 动态内容理解（Dynamic Content Understanding）**：
- 采用 MLLM 以 **30 秒滑动窗口**持续摄取直播实时片段，生成动态内容嵌入
- 用大型 LLM 构建数据集，通过知识蒸馏微调轻量 MLLM

**② 实时协作后对齐（Real-time Collaborative Post-alignment）**：
- 双塔架构：作者塔融合静态属性与动态内容特征，用户塔处理用户属性
- 作者塔门控融合公式：
$$x^{MLLM} = \text{MLP}(x^{MLLM}_{30s} \oplus x^{MLLM}_{pooling})$$
$$x^{Author} = \lambda x^{AId} + (1-\lambda) x^{MLLM}$$
- 基于交互样本流进行用户-作者对齐（In-Batch-Softmax），得到 **IA Embedding**

#### 1.2 残差量化

采用 **Res-Kmeans**（而非 RQ-VAE）进行层次化量化：
$$q^l = \arg\min_k \lVert c_k^l - x^l \rVert, \quad x^{l+1} = x^l - c^l_{q^l}$$

**量化方案对比实验**（~400 万作者嵌入）：

| 类型 | 方法 | 码本大小 | $L_0$ UR | $L_0 \times L_1$ UR | SID CR | Author CR |
|------|------|---------|----------|---------------------|--------|-----------|
| MLLM | Res-Kmeans | (512,512,512) | 100% | 53.83% | 28.10% | 63.83% |
| MLLM | Res-Kmeans | (8192,8192,8192) | 88.99% | 2.34% | 3.56% | 9.47% |
| IA | RQ-VAE | (512,512,512) | 100% | 74.41% | 15.76% | 39.76% |
| IA | RQ-VAE | (8192,8192,8192) | 100% | 1.16% | 8.54% | 22.78% |
| IA | Res-Kmeans | (512,512,512) | 100% | 93.98% | 9.72% | 23.03% |
| **IA** | **Res-Kmeans** | **(8192,8192,8192)** | **100%** | **4.50%** | **0.66%** | **1.76%** |

> **结论**：IA + Res-Kmeans + 大码本（8192）组合达到最低碰撞率（SID CR 仅 0.66%），最终选用 $T=3$ 层、$N_l=8192$ 的 Res-Kmeans。

### 模块二：生成式推荐器（Generative Recommender）

#### 2.1 基础 Decoder-Only 架构

**Lazy Decoder-Only**（受 OneRec-v2 启发）：
- 将用户多路历史序列（有效观看 $S^{Eff}$、礼物 $S^{Gift}$、短视频 $S^{Video}$ 等）投影为 KV 上下文 $E \in \mathbb{R}^{|S| \times d}$
- Decoder 由 $L$ 层 Transformer 块组成，每层包含：跨注意力（Cross-Attn）、自注意力（Self-Attn）、FFN
- 训练时 $h^0 = \text{Emb}(\{[\text{BOS}], q^0, q^1\})$（Teacher Forcing）

**相比 Encoder-Decoder 的优势**：将更多计算分配给解码过程，增强生成能力；避免对冗长用户序列的显式编码开销。

#### 2.2 时序动态建模（Temporal Dynamics Conditioning）

三层次时序感知：

**① 历史序列时序感知**：为每个历史 item 注入多粒度时间偏置（小时/天/周）：
$$x_{i\_ta} = x_i + \text{MLP}(\text{Concat}(x_i^{Hour}, x_i^{Day}, x_i^{Week}))$$

**② 生成锚点时序感知**：用查询时刻的时间特征增强 `[BOS]` token：
$$x_{[BOS]\_ta} = x_{[BOS]} + \text{MLP}(\text{Concat}(x_q^{Hour}, x_q^{Day}, x_q^{Week}))$$

**③ 注意力门控时序感知（TA Gated-Attn）**：在跨注意力和自注意力中引入可学习门控分数：
$$\text{Score}(X) = \sigma(X W_\theta)$$
$$O = (\text{MultiHeadAttn}(XW_Q, X'W_K, X'W_V) \odot \text{Score}(X)) W_O$$

#### 2.3 Sequential MTP（序列多 Token 预测）

**动机**：标准自回归推理中，$q^1$ 和 $q^2$ 的推理批次远大于 $q^0$，计算开销随 beam size 快速膨胀；且 SID1/SID2 是 SID0 的残差，预测任务相对简单。

**设计**：三个串联解码器，共享 SID 嵌入层：
- **主解码器（Main-Decoder）**：完整 $L$ 层 Transformer，负责预测 $q^0$
- **轻量解码器（Decoder-Lite × 2）**：单层 Transformer 块，共享主解码器第一层参数（除 FFN），负责预测 $q^1$ 和 $q^2$
- 输入为前一模块隐状态与前序 SID 嵌入的拼接：
$$h_i^k = \text{MLP}(\text{Concat}(\{h_i^{'k-1}, \text{Emb}(q^i)\}))$$
- 联合损失：$\mathcal{L}_{MTP} := w_0 \mathcal{L}_{MTP_{main}} + w_1 \mathcal{L}_{MTP_1} + w_2 \mathcal{L}_{MTP_2}$

**推理加速效果**（3 层 Lazy Dec-Only）：

| 方法 | QPS 提升 | 平均延迟 | P99 延迟 |
|------|---------|---------|---------|
| Lazy Dec-Only | - | 5.05 ms | 8.48 ms |
| + MTP | **+31.8%** | **2.25 ms** | **3.62 ms** |

（6 层版本 QPS 提升更高达 +62.0%，延迟从 6.82 ms 降至 2.97 ms）

#### 2.4 QK Norm（训练稳定化）

**问题**：随模型加深，QK logits 出现数值爆炸，bf16 混合精度下误差累积导致 Softmax 趋近 one-hot，梯度消失，训练崩溃。

**解法**：在计算注意力 QK logits 前对 Q、K 施加 RMSNorm：
$$\text{Attn}(Q,K,V) = \text{Softmax}\left(\frac{\text{RMSNorm}(Q)\text{RMSNorm}(K)^T}{\sqrt{d}}\right)V$$

效果：彻底消除长期训练中的 loss 偶发尖峰，保证稳定收敛。

### 模块三：强化学习多目标对齐

#### 3.1 奖励模型

使用快手内部统一多目标集成排序模型 **Pantheon** 作为奖励模型，输出单一综合排序分：
$$Score = \text{Pantheon}(\text{User}, \text{Author})$$
$$\mathcal{L}_{Pantheon} := \sum_{XTR \in \{CTR, LVTR, GTR, \ldots\}} w^{XTR} \mathcal{L}^{XTR}_{Pantheon}$$

直接以 $r := Score$ 作为每个候选作者的奖励信号。

#### 3.2 优化策略

对比了 DPO 和 GRPO 两种策略：

- **DPO**：选最高/最低优势样本作为正/负对，基于对比损失优化
- **GRPO**（最终采用）：利用组内所有响应的相对优势进行梯度优化，反馈信号更丰富、更鲁棒

$$\mathcal{L}_{GRPO} := \frac{1}{G}\sum_{i=1}^G \min\left(\frac{\pi_\theta(o_i|q)}{\pi_{\theta ref}(o_i|q)} A_i, \text{clip}(\cdot, 1-\epsilon, 1+\epsilon) A_i\right)$$

**训练策略**：在离线流式训练中融入 RL，每轮仅对 1% 的 query 执行 RL 采样，保证训练稳定性。

$$\mathcal{L}_{OneLive} := \mathcal{L}_{MTP} + w\mathcal{L}_{RL}$$

## 实验设置

### 数据集

- **离线**：快手 App 大规模工业直播平台日志，近两个月数据
  - 4 亿用户、300 万主播、数十亿交互记录
  - 最后一天数据用于测试，其余用于训练
  - 覆盖长观看（LongView）和点击（Click）记录

### 评估指标

| 指标 | 说明 |
|------|------|
| **ACC@all** | 训练阶段多级 SID 命中的平均准确率 |
| **HR@K** | Beam Search Top-K 中目标主播的命中率 |
| **MRR@K** | Top-K 预测中正确标签的平均倒数排名 |
| **Reward（LVTR/CTR/WTR/GTR）** | 在线排序模型对生成结果的多目标打分 |

### 基线方法

- **传统模型**：SASRec、KuaiFormer（长序列压缩）、GNN（异构图）
- **生成式模型**：TIGER（基础 SID 生成）、OneRec（工业落地版）

|------|------|----------------|-----------------|--------------|---------------|

## 核心贡献总结

1. **动态分词器**：首次针对直播场景提出实时内容感知的动态分词方案，通过 MLLM 30s 滑动窗口 + 双塔协作后对齐，生成既含语义又含协作信号的 IA Embedding，并以 Res-Kmeans 量化为低碰撞率 SID。

2. **时间感知门控注意力（TA Gated-Attn）**：三层次时序建模（序列时序 + 生成锚点时序 + 门控注意力），显式建模直播有限生命周期约束，将在线主播有效率提升 14.29%。

3. **Sequential MTP**：受 DeepSeek-V3 启发，设计三级串联解码器（主解码器 + 2 个轻量解码器），通过参数共享复用 KV Cache，推理 QPS 最高提升 62%，延迟降低 55.45%，MFU 从 2.56% 提升至 22.78%。

4. **QK Norm**：引入 RMSNorm 约束 Q/K 量级，彻底消除深层 Transformer 在 bf16 混合精度训练中的 logit 爆炸和 loss 尖峰问题。

5. **统一多目标对齐框架**：以 Pantheon 集成排序模型为奖励，采用 GRPO 进行在线策略优化，在多目标（CTR/LVTR/GTR/WTR）上全面优于 DPO，并在大规模在线 A/B 测试中验证了显著的业务收益。

6. **首个大规模直播 GR 落地**：OneLive 是首个成功部署于大规模真实直播平台的统一生成式推荐框架，日活服务 4 亿用户，300 万主播。
