## 研究背景与动机

### 工业推荐系统的 GSU/ESU 范式

大规模工业推荐系统中，用户历史交互序列可超过 10 万条，无法全量输入模型。业界普遍采用两阶段策略：

- **GSU（General Search Unit，通用检索单元）**：用简单方法从用户全量历史中检索 Top-K 相关子序列
- **ESU（Exact Search Unit，精确检索单元）**：对 GSU 检索到的子序列进行端到端精细化建模，预测 CTR/CVR 等目标

代表方法：TWIN、TWIN V2 等均依赖 **ID Embedding** 支持 GSU/ESU 计算。

### ID-based 方法的三大缺陷

| 缺陷 | 描述 |
|------|------|
| **低信息密度（Low Information Units）** | 通过用户-物品交互日志学习粗粒度语义，长尾问题严重，细粒度语义表达能力弱 |
| **知识孤岛（Knowledge Isolation）** | 协同信号分散在亿级物品 Embedding 中，物品下线后知识完全丢失 |
| **弱泛化能力（Weak Generalization Ability）** | 强依赖流式训练，一旦停止更新线上性能急剧下降 |

### LLM 的互补优势

| 优势 | 描述 |
|------|------|
| **密集信息单元** | 仅用 <20,000 token 词表表达人类世界知识 |
| **知识集成** | 更深更大的 Transformer 参数存储知识，准确理解语义 |
| **高泛化能力** | 一次训练，长期稳定推理，对任意 token 序列均可准确理解 |

### 直接使用预训练 LLM Embedding 的两大问题

1. **表示不匹配（Representation Unmatch）**：预训练 LLM 目标（图像描述、问答）与 RecSys 目标（点击预测）不一致。例如牙膏和药膏因外形相似可能被 LLM 认为相关，但使用场景完全不同。
2. **表示无法学习（Representation Unlearning）**：冻结的预训练 Embedding 无法通过梯度端到端更新，难以适配动态推荐任务。

### QARM V2 的核心思路

> **目标**：让 LLM 生成的物品 Embedding 在 GSU 侧反映适当的业务语义，同时在 ESU 侧提供可端到端学习的离散特征。

QARM V2 在前作 QARM 基础上提出两大改进：
1. **推理物品对齐机制（Reasoning Item Alignment）**：用推理 LLM 过滤噪声物品对，生成业务对齐的 LLM Embedding，增强 GSU 检索
2. **Res-KmeansFSQ 量化编码机制**：混合残差 K-means 与有限标量量化（FSQ），生成低冲突率的多级语义 ID（SID），增强 ESU 端到端优化

## 核心方法：QARM V2

### 整体架构

```
QARM V2 整体框架
│
├── 推理物品对齐（Reasoning Item Alignment）→ 生成业务对齐 LLM Embedding
│   ├── 数据管道：从 Item2Item/User2Item 检索模型导出物品对
│   ├── 推理过滤：用 Qwen3 推理模型过滤噪声物品对
│   ├── 问答数据：用 Gemini/Qwen2.5-VL-72B 生成物品理解 QA 对
│   └── 三段式 LLM 微调：对比学习 + NTP 生成任务同时优化
│
├── Res-KmeansFSQ 量化编码 → 生成多级语义 ID（SID）
│   ├── 第 1 层：K-means 量化（粗粒度类目/用途）
│   ├── 第 2 层：残差 K-means 量化（中粒度）
│   └── 第 3 层：FSQ 量化（细粒度，规则化，低冲突）
│
├── GSU 增强：用 LLM Embedding 做内积相似度检索，召回 Top-K 历史子序列
│
└── ESU 增强：用三级 SID 作为可学习离散特征，端到端 Target-Attention 精排
```

### 模块一：推理物品对齐（Reasoning Item Alignment）

#### 1.1 推理监督数据管道

**种子数据来源**：
- **利用型 Item2Item 统计检索模型**（Swing）：导出稳定的高相似度物品对（大多数用户同时点击了商品A和商品B）
- **探索型 User2Item 双塔检索模型**：导出用户级动态物品对（当前用户点击了商品A和商品B）

**问题**：上述物品对存在噪声：
- Swing 方法易受热门曝光偏差影响（如"酱油+洗衣液"对，无底层逻辑）
- 双塔模型导出的物品对部分在多模态内容和底层逻辑上完全无关

**推理过滤**：部署最新推理 LLM（Qwen3）对物品对进行相关性判断：

```
推理 Prompt（物品对过滤）：
Item-1 信息：#标题, #属性, ...
Item-2 信息：#标题, #属性, ...
请分析两个商品之间的潜在关联，如语义关联（相似概念或用途）、
互补购买关系（常同时购买或配合使用）。
示例：
(1) (钓鱼竿, 遮阳伞)：高度相关的使用场景
(2) (钓鱼竿, 户外夹克)：共同的户外活动场景
如果商品对关联性强，回复：<answer>Yes</answer>；否则回复：<answer>No</answer>。
```

- 对 Swing 物品对使用 **Qwen3-0.6B**（推理成本低）
- 对 User2Item 物品对使用 **Qwen3-8B**（更复杂的关联判断）
- **过滤效果**：拒绝 10%+ 的 Item2Item 样本，拒绝 70%+ 的 User2Item 物品对

**物品理解 QA 数据**：使用 Gemini、Qwen2.5-VL-72B 等大模型，基于物品的标题、属性、图像、OCR、ASR 等多模态信息生成 10 条多样化问答对，用于深度物品理解。

#### 1.2 Decoder-Only GPT 嵌入生成器（三段式微调）

**核心挑战**：当前 SOTA LLM 均为 Decoder-Only GPT 范式（预测下一个 token），而 RecSys 需要的是压缩嵌入表示，两者目标冲突。

**三段式设计**：将输入 token 序列分为三段，通过修改 Attention Mask 实现：

| 段 | 名称 | 内容 | Attention 范围 |
|----|------|------|---------------|
| 第 1 段 | **Input Segment** | 物品标题、图像、OCR 等 | 自注意 |
| 第 2 段 | **Compression Segment** | 多个特殊 token `<EMB>` | 只能 attend 到 Input Segment |
| 第 3 段 | **QA Segment** | 问答 token | 只能 attend 到前两段 |

**训练目标**：
- **对比学习任务**（Compression Segment）：物品对正负样本对比，对齐业务语义
- **NTP 生成任务**（QA Segment）：保留 LLM 预训练的生成能力，确保细粒度信息提取

**工程技巧**：
- 使用 **Gradient Cache** 技术支持大 batch size 训练
- **Warm-start 退火**：训练初期 QA Segment 可 attend 所有 Input/Query Segment，逐步将 Input Segment 注意力权重降为零，保证训练稳定性

### 模块二：Res-KmeansFSQ 量化编码

#### 2.1 问题分析

前作 QARM 使用三层残差 K-means（Res-Kmeans，每层 4096 个聚类中心）生成 SID，但存在严重的**码本冲突**问题：
- 购物场景中 30%+ 的 SID 对应多个物品候选
- 原因：物品分布呈**长尾分布**，高维空间中分布不均匀（稠密区和稀疏区并存）
- K-means 只最小化类内平均距离，不考虑类间距离，导致码本中心集中在稠密区，稀疏区物品共享同一 SID

#### 2.2 核心观察

> 前两层 Res-Kmeans 码已足以识别物品的主要类目和用途；**第三层应反映物品特有特征**，而非继续拟合真实分布。

#### 2.3 Res-KmeansFSQ 混合量化

**设计思路**：
- **前两层**：保留 Res-Kmeans，捕获粗粒度物品类目和用途模式（数据自适应）
- **第三层**：替换为 FSQ（有限标量量化），提供均匀分布的码，降低冲突率（规则化，与数据分布无关）

**详细步骤**：

**Step 1：收集物品嵌入集合**
$$\mathbf{M} \in \mathbb{R}^{N \times d}, \quad N > 10,000,000$$

**Step 2：两层残差 K-means 码本训练**
$$\mathbf{C}^1 = \texttt{Kmeans}(\mathbf{M}, K), \quad \mathbf{M}^1 = \mathbf{M} - \texttt{NearestRep}(\mathbf{M}, \mathbf{C}^1)$$
$$\mathbf{C}^2 = \texttt{Kmeans}(\mathbf{M}^1, K)$$
其中 $K$ 为聚类数（如 8192），$\texttt{NearestRep}$ 为最近邻中心表示。

**Step 3：第三层 FSQ 训练**
$$\mathbf{M}^2 = \mathbf{M}^1 - \texttt{NearestRep}(\mathbf{M}^1, \mathbf{C}^2)$$
$$\mathbf{Z} = \texttt{FSQ}(\mathbf{M}^2) = \lfloor L \cdot \sigma(\mathbf{M}^2 \mathbf{W}) \rceil$$

其中：
- $\mathbf{W} \in \mathbb{R}^{d \times 13}$：可学习投影矩阵
- $\sigma(\cdot)$：Sigmoid 函数，将值归一化到 $[0,1]$
- $L$：每维量化级别数（如 $L=2$）
- $\lfloor \cdot \rceil$：四舍五入取整
- $\mathbf{Z} \in \mathbb{R}^n$：FSQ 编码

**FSQ vs K-means 的本质区别**：
- K-means：分布相关（distribution-dependent），边缘点被拉向稠密区
- FSQ：分布无关（distribution-independent），预定义均匀量化网格，每个点保持原位

### 模块三：GSU/ESU 增强工作流

#### 3.1 GSU 增强

利用 LLM Embedding $\mathbf{m}$ 做语义感知检索：

$$\{\mathbf{m}_1, \mathbf{m}_2, \dots, \mathbf{m}_k\} = \texttt{Top-K-GSU}(\{\mathbf{m}_1, \mathbf{m}_2, \dots, \mathbf{m}_T\}, \mathbf{m}, k)$$

通过内积相似度 $\langle \mathbf{m}_i, \mathbf{m} \rangle$ 从用户全量历史序列中检索 Top-K 相关子序列。

#### 3.2 ESU 增强

每个物品由原始 ItemID + 三级 SID $\{c^1, c^2, c^3\}$ 表示，通过 Lookup Table 映射为可学习 Embedding，进行 Target-Attention：

$$\texttt{ESU} = \texttt{TargetAttention}\left((\mathbf{I}, \mathbf{c}^1, \mathbf{c}^2, \mathbf{c}^3), \{(\mathbf{I}_1, \mathbf{c}^1_1, \mathbf{c}^2_1, \mathbf{c}^3_1), \dots, (\mathbf{I}_k, \mathbf{c}^1_k, \mathbf{c}^2_k, \mathbf{c}^3_k)\}\right)$$

$$\hat{y}^{\texttt{ctr}}, \hat{y}^{\texttt{cvr}}, \dots \leftarrow \texttt{MoE}(\texttt{ESU}, \texttt{Others})$$

**端到端训练损失**（多任务二元交叉熵）：

$$\mathcal{L} = -\sum_{\texttt{xtr} \in \{\texttt{ctr}, \texttt{cvr}, \dots\}} \left(y^{\texttt{xtr}} \log \hat{y}^{\texttt{xtr}} + (1 - y^{\texttt{xtr}}) \log(1 - \hat{y}^{\texttt{xtr}})\right)$$

## 实验设置

### 数据集

| 数据集 | 类型 | 规模 |
|--------|------|------|
| **Amazon Book** | 公开学术数据集 | 评分≥4 为正样本，序列长度≥20 |
| **Kuaishou Advertising** | 工业数据集 | CTCVR 预测 |
| **Kuaishou Shopping\#1/2/3** | 工业数据集 | CTR/CVR/CTCVR 预测 |
| **Kuaishou Live-streaming\#1/2** | 工业数据集 | Click/Watch Time/Gift/Follow 等 |

### 基线方法

- **DIN**：使用用户最近交互物品
- **SIM-hard**：用相同标签搜索物品
- **SIM-soft**：用可学习 ID Embedding 检索 Top-K 相关物品
- **QARM（前作）**：基础版本，使用 Res-Kmeans（无推理过滤、无 FSQ）

### 评估指标

- **AUC**：全局 ROC 曲线下面积
- **UAUC**：用户级 AUC（平均每用户 AUC）
- **GAUC**：分组 AUC（按用户分组的加权 AUC）
- **HR@K**：Hit Rate，Top-K 检索命中率
- **GMV/Order/Exposure/Revenue**：在线业务指标

## 核心贡献总结

1. **推理物品对齐机制（Reasoning Item Alignment）**：
   - 首次将推理 LLM 引入物品对过滤，拒绝曝光偏差噪声和无逻辑关联的物品对
   - 构建物品理解 QA 数据，增强 LLM 对复杂短视频/直播内容的理解
   - 三段式微调范式，统一对比学习与 NTP 生成任务，将 Decoder-Only GPT 转化为稳定的嵌入生成器

2. **Res-KmeansFSQ 量化编码**：
   - 识别出 Res-Kmeans 码冲突的根本原因（长尾分布 + K-means 只优化类内距离）
   - 混合策略：前两层 Res-Kmeans 捕获粗粒度类目结构，第三层 FSQ 提供均匀分布的细粒度码
   - 将冲突率从 77.92% 降至 32.39%，HR@1 从 80.3% 提升至 95.2%

3. **统一 GSU/ESU 增强框架**：
   - GSU：业务对齐 LLM Embedding 替代 ID Embedding，语义感知检索
   - ESU：多级 SID 作为可学习离散特征，端到端优化，解决 Representation Unlearning 问题

4. **大规模工业验证**：
   - 覆盖广告、购物、直播三大业务场景
   - 在线 A/B 测试验证显著业务收益（广告收入 +4.87%，购物 GMV 最高 +5.61%）
   - 支持快手 4 亿日活用户
