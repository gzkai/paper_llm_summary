## 研究背景与动机

OneRec-V1 已在工业界取得显著成功，但存在两个核心瓶颈：

### 瓶颈一：编码器-解码器架构的计算分配极度低效

OneRec-V1 采用编码器-解码器框架，编码器处理用户历史交互序列，解码器通过 Cross-Attention 利用编码器输出。由于上下文序列远长于目标物品序列，计算资源被严重倾斜：

| 架构 | 上下文长度 N=512 时目标解码占比 | N=3000 时占比 |
|------|-------------------------------|--------------|
| 编码器-解码器（0.5B:0.5B） | **2.34%** | 0.41% |
| 朴素解码器-Only（1B） | 2.85% | 0.49% |
| **Lazy 解码器-Only（1B）** | **≈100%** | **≈100%** |

97.66% 的 FLOPs 被上下文编码消耗，而推荐决策实际发生的解码阶段仅占 2.34%，严重限制了模型规模扩展。

### 瓶颈二：仅依赖奖励模型的强化学习存在固有缺陷

1. **采样效率低**：奖励模型方法需要在线生成和评分，只能覆盖极小比例的用户（约 1%）；
2. **奖励 Hacking**：策略可能学会利用奖励模型中的代理信号偏差，而非真正改善用户体验；
3. **机遇**：随着 OneRec 服务 25% 总流量，真实用户反馈（如视频观看时长）已可直接获取，为 On-Policy 优化创造条件。

## 核心方法：OneRec-V2 两大创新

### 创新一：Lazy Decoder-Only 架构

#### 1.1 数据组织方式的演进

推荐系统训练样本组织面临两难困境：

- **用户中心式**：每个样本包含完整用户历史 → 存在**时间数据泄漏**（训练时可看到未来交互）和**冗余计算**（重叠序列重复处理）
- **朴素曝光中心式**：按曝光时间顺序组织 → 子序列重复训练导致**过拟合**，快手每日 Token 量超 50 万亿，**计算爆炸**

**OneRec 的解法**：按时间顺序组织曝光数据，但**仅对最新曝光物品计算训练损失**（"懒惰"）。

#### 1.2 架构设计

```
输入：用户历史上下文（静态特征 + 短期序列 + 长期序列）
                    ↓
             Context Processor
      （将异构特征映射为统一维度，生成 L_kv 组 K/V 对）
                    ↓
           Lazy Decoder Blocks
    ┌──────────────────────────────────────┐
    │  Cross-Attn（无 K/V 投影，KV共享）    │
    │  Causal Self-Attn                    │
    │  FFN / MoE                           │
    └──────────────────────────────────────┘
                    ↓
            预测目标物品语义 ID [s¹, s², s³]
```

**关键设计细节**：

**① Context Processor**

将用户静态特征、短期行为序列、长期行为序列拼接为统一的 Context 张量，维度为：

```math
d_{\text{context}} = S_{\text{kv}} \cdot L_{\text{kv}} \cdot G_{\text{kv}} \cdot d_{\text{head}}
```

沿特征维度切分为 $L_{\text{kv}}$ 组 K/V 对，每组经 RMSNorm 归一化：

```math
\mathbf{k}_l = \text{RMSNorm}_{k,l}(\mathbf{C}_{l \cdot S_{\text{kv}}})
```

```math
\mathbf{v}_l = \begin{cases}
\text{RMSNorm}_{v,l}(\mathbf{C}_{l \cdot S_{\text{kv}} + 1}), & S_{\text{kv}} = 2 \text{（分离 K/V）} \\
\mathbf{k}_l, & S_{\text{kv}} = 1 \text{（共享 K/V）}
\end{cases}
```

**② Lazy Cross-Attention：KV 共享**

多个解码器 Block 共享同一组 K/V 对，层索引映射关系为：

```math
l_{\text{kv}} = \left\lfloor \frac{l \cdot L_{\text{kv}}}{N_{\text{layer}}} \right\rfloor
```

默认设置 $`L_{\text{kv}}=1`$，$`S_{\text{kv}}=1`$（K/V 完全共享），**无需 K/V 投影层**，大幅节省计算。

**③ Lazy Cross-Attention：Grouped Query Attention（GQA）**

Query 保持 $`H_q`$ 个头，K/V 仅使用 $`G_{\text{kv}}`$ 个头组（$`G_{\text{kv}} < H_q`$），显著降低内存占用和访存瓶颈。

**④ Lazy Decoder Block 结构**

```math
\mathbf{h}_{\text{cross}}^{(l)} = \mathbf{h}^{(l-1)} + \text{CrossAttn}\left(\text{RMSNorm}(\mathbf{h}^{(l-1)}), \mathbf{k}_{l_{\text{kv}}}, \mathbf{v}_{l_{\text{kv}}}\right)
```

```math
\mathbf{h}_{\text{self}}^{(l)} = \mathbf{h}_{\text{cross}}^{(l)} + \text{SelfAttn}\left(\text{RMSNorm}(\mathbf{h}_{\text{cross}}^{(l)})\right)
```

```math
\mathbf{h}^{(l)} = \mathbf{h}_{\text{self}}^{(l)} + \text{FFN}^{(l)}\left(\text{RMSNorm}(\mathbf{h}_{\text{self}}^{(l)})\right)
```

深层 FFN 替换为 MoE（53 个路由专家 + 1 个共享专家，top-3 路由，DeepSeek-V3 无损失负载均衡）。

**⑤ 训练目标**

```math
\mathcal{L}_{\text{Gen}} = -\frac{1}{3} \sum_{i=1}^{3} \log p(s^{i} \mid \text{BOS}, s^{\lt i}, \text{Context})
```

（与 V1 的区别：V2 取 3 个 token 的平均，V1 取求和）

#### 1.3 计算效率对比

| 架构 | 参数量 | GFLOPs | 激活内存 | 收敛损失 |
|------|--------|--------|---------|---------|
| Enc:Dec=1:1 | 1B | 296.36 | 17.63B | 3.28 |
| Enc:Dec=1:2 | 1B | 204.21 | 12.20B | 3.26 |
| Naive Dec-Only | 1B | 634.83 | 31.53B | * |
| **Lazy Dec-Only** | **1B** | **18.89** | **1.24B** | **3.27** |

Lazy Decoder 在相同参数规模下，FLOPs 仅为 Enc-Dec 的 **6.4%**，激活内存仅为 **7.0%**，但收敛损失相当。

### 创新二：基于真实用户交互的偏好对齐

#### 2.1 Duration-Aware Reward Shaping（时长感知奖励塑形）

**问题**：原始观看时长存在视频时长偏差——越长的视频天然积累更多观看时长，与内容质量无关。

**解法**：将视频按时长分桶，计算目标视频在用户历史同时长视频中的分位数作为奖励。

**分桶策略**（对数均匀分桶）：

```math
\mathcal{F}(d) = \lfloor \log_{\beta}(d + \epsilon) \rfloor
```

其中 $`\beta`$ 为桶粒度控制参数，$`\epsilon`$ 为数值稳定常数。

**分位数计算**：

```math
q_i = \frac{|\{p_j \in P_{u, b} \mid p_j \le p_i\}|}{|P_{u, b}|}
```

其中 $`P_{u,b}`$ 为用户 $`u`$ 在时长桶 $`b`$ 内的历史观看时长集合。

**优势值定义**（无需归一化，直接赋值）：

```math
A_i =
\begin{cases}
+1, & q_i > \tau_B \text{ 且 } neg_i = 0 \quad \text{（top 25\% 正样本）}\\
-1, & neg_i = 1 \quad \text{（显式负反馈，如"不喜欢"）}\\
0,  & \text{其他}
\end{cases}
```

#### 2.2 GBPO（Gradient-Bounded Policy Optimization，梯度有界策略优化）

**背景**：OneRec 服务 25% 总流量后，大量 On-Policy 样本（由 OneRec 自身生成的曝光样本）可用。这些样本的 $`\pi_{old} = \text{sg}(\pi_\theta)`$，策略比率恒为 1。然而，传统裁剪方法（GRPO/PPO）认为比率=1 的样本"稳定"，不做截断，但对负样本仍可能产生梯度爆炸。

**问题分析**：

对于策略比率=1 的负样本（$`A_i < 0`$）：

```math
\frac{\partial \mathcal{J}_{ECPO}^i(\theta)}{\partial \theta} = - A_i\cdot\frac{1}{\pi_\theta} \frac{\partial \pi_\theta}{\partial \theta}
```

当 $`\pi_\theta`$ 很小时，梯度极大，导致爆炸。而 BCE 损失对负样本的梯度为：

```math
\frac{\partial \mathcal{L}_{BCE}}{\partial \theta}\bigg|_{y=0} = \frac{1}{1-p_\theta}\frac{\partial p_\theta}{\partial \theta}
```

当 $`p_\theta`$ 很小时，梯度也很小（稳定），与 RL 梯度的行为**相反**。

**GBPO 方案**：用 BCE 损失的梯度动态约束 RL 梯度上界：

```math
\mathcal{J}_{GBPO}(\theta) = -\mathbb{E}\left[\frac{1}{G}\sum_{i=1}^G\frac{\pi_{\theta}(o_i|u)}{\pi_{\theta_{old}}'(o_i|u)}\cdot A_i\right]
```

```math
\pi_{\theta_{old}}'(o_i|u)=
\begin{cases}
\max(\pi_{\theta_{old}}, \text{sg}(\pi_{\theta})), & A_i\geq0 \\[6pt]
\max(\pi_{\theta_{old}}, 1-\text{sg}(\pi_{\theta})), & A_i< 0
\end{cases}
```

**GBPO 的两大优势**：
- **全样本利用**：不丢弃任何样本的梯度，鼓励更多样化探索；
- **梯度有界稳定化**：用 BCE 损失的梯度上界约束 RL 梯度，对负样本尤其稳定。

#### 2.3 与现有 RL 方法的对比

| 方法 | 正样本处理 | 负样本处理 | 样本利用率 | 梯度稳定性 |
|------|-----------|-----------|-----------|-----------|
| GRPO/PPO | ratio 裁剪 | ratio 裁剪 | 部分丢弃 | 中等 |
| DAPO | 放宽上界裁剪 | 无上界 | 较高 | 负样本不稳定 |
| Dual-Clip | ratio 裁剪 | 额外上界截断 | 较低 | 较稳定 |
| ECPO（V1） | min-clip | 提前裁剪 $`\pi_{old}`$ | 较高 | 较稳定 |
| **GBPO（V2）** | **动态下界** | **动态上界（BCE 约束）** | **100%** | **最稳定** |

## 核心贡献总结

1. **Lazy Decoder-Only 架构**：通过"懒惰"的计算策略（仅对目标物品 Token 计算损失），将 FLOPs 利用率从 2.34% 提升至 ≈100%，使 8B 参数模型在等效计算预算下成为可能，并首次在推荐系统中验证了与 Chinchilla 一致的 Scaling Law（$\alpha=0.489$）。

2. **Duration-Aware Reward Shaping**：通过对数分桶 + 分位数排名，消除视频时长对观看时长奖励的天然偏差，建立短期观看行为与长期满意度（App Stay Time）的可靠关联。

3. **GBPO（梯度有界策略优化）**：用 BCE 损失的梯度动态约束 RL 梯度上界，解决负样本梯度爆炸问题，实现 100% 样本利用率与高度稳定的训练。

4. **On-Policy 自迭代优化**：利用 OneRec 自身生成的曝光样本进行强化学习，实现自我提升闭环，几乎所有指标从负转正。

5. **推理效率突破**：L20 GPU 上延迟 36ms，MFU 62%，远超 OneRec-V1 时期的 11.2%（H100）。

## 局限性与未来工作

1. **Scaling 潜力**：0.1B→8B 参数 Scaling Law 拟合良好，预示进一步扩大规模仍有收益空间；
2. **奖励系统**：当前方案通过规则建立短期-长期收益关联，未来目标是让模型直接优化长期价值，实现真正的长期价值自我强化；
3. **生态平衡**：关闭缓存实验揭示冷启动视频曝光减少 44.7%，马太效应和内容生态集中化问题需要专门解决。
