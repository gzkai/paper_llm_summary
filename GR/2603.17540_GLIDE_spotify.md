## 研究背景与动机

### 问题定义

播客消费通常以"习惯性收听"为主——用户反复收听已订阅的节目，而对新节目的探索意愿有限。现有推荐系统过度依赖历史交互模式，缺乏对动态听众意图和语义内容的理解能力，难以支持有效的发现型推荐。

### 核心挑战

1. **目录 Grounding**：如何将 LLM 的生成能力与动态更新的大规模播客目录对接（冷启动、目录变化）
2. **用户个性化**：如何在不膨胀 prompt 长度的前提下注入长期用户偏好
3. **生产部署约束**：LLM 推理延迟高，需在严格的延迟和成本预算内运行

### 非习惯性收听的定义

对用户 $`u`$ 和节目 $`s`$，设 $`T_{u,s}`$ 为过去 28 天对节目 $`s`$ 的总收听时长：

- **习惯性节目**：$`T_{u,s} \ge 10`$ 分钟
- **非习惯性节目**：$`T_{u,s} < 10`$ 分钟，进一步细分为：
  - **非习惯但熟悉**：曾有过收听记录，但未形成当前习惯
  - **非习惯且陌生**：从未收听过的全新节目

## 核心方法：GLIDE

**GLIDE**（**G**rounded **L**LM for **I**nterest **D**iscovery r**E**commendations）将播客推荐形式化为基于语义ID的指令跟随任务，使用 Llama 3.2 1B 作为骨干模型。

### 1. 问题形式化

给定用户 $`u`$ 的交互历史 $`\mathcal{H}_u = \{(e_1,t_1),\ldots,(e_n,t_n)\}`$，目标是生成 $`k`$ 个候选 episode，形式化为条件生成：

```math
p(\text{SID}(e) \mid \text{prompt}(u, \mathcal{H}_u, \text{instruction}))
```

模型接受四类输入：
1. **近期收听历史**：时间序列化的 SID 序列
2. **轻量用户上下文**：自然语言 token（地区、兴趣话题等）
3. **长期用户偏好**：通过 soft prompt 注入的稠密协同过滤用户向量
4. **任务指令**：指定推荐目标的自然语言指令（如"熟悉"或"陌生"）

---

### 2. 语义 ID（SID）构建

#### 残差 K-Means 量化（R-KMeans）

episode 内容嵌入 $`x_e \in \mathbb{R}^d`$ 通过残差量化器编码为长度为 $`M`$ 的离散 token 序列：

```math
r_0 = x_e
```

```math
c_m = \arg\min_{j \in [1..K]} \left\lVert r_{m-1} - \mu_{m,j} \right\rVert_2
```

```math
r_m = r_{m-1} - \mu_{m,c_m}
```

最终 SID 为元组 $`(c_1, \ldots, c_M)`$，作为离散 token 序列处理。

**实现细节**：
- $`K = 256`$，$`M = 4`$，共向 LLM 词表新增 $`M \times K = 1024`$ 个 SID token
- 每个 episode 恰好用 4 个 token 表示，例如：$`\langle\texttt{SID}_1{=}13\rangle\,\langle\texttt{SID}_2{=}65\rangle\,\langle\texttt{SID}_3{=}188\rangle\,\langle\texttt{SID}_4{=}7\rangle`$
- 多层量化诱导出粗粒度到细粒度的语义结构（如 *News* → *Sports-News*）

**选择 R-KMeans 而非 RQ-VAE / R-LFQ 的理由**：
- R-KMeans HR@30 比 RQ-VAE 高 +9.52%，比 R-LFQ 高 +4.76%
- 桶内余弦相似度更高（0.856 vs 0.657），碰撞时语义一致性更好
- 无需端到端训练，无 codebook collapse 问题，工程实现更简单稳定

#### 为何使用内容嵌入而非协同过滤嵌入构建 SID

| 维度 | 内容嵌入 SID | CF 嵌入 SID |
|------|------------|------------|
| 冷启动 | 立即可用 | 需要足够交互量 |
| 稳定性 | 稳定（仅 metadata 变化时更新） | 随用户行为漂移，训练-服务不一致 |

---

### 3. Soft Prompt 用户个性化

将生产个性化模型的用户嵌入 $`v_u`$ 通过两层 MLP 投影到 LLM 隐层维度：

```math
\tilde{V}_u = \mathrm{Proj}(v_u) \in \mathbb{R}^{d_{\text{LLM}}}
```

将 $`\tilde{V}_u`$ 作为单个 soft prompt token 插入到用户上下文开头，紧跟系统指令之后，使 LLM 能在与自然语言 token 和 SID 相同的注意力机制下处理高维用户状态，同时保持紧凑的上下文窗口。

---

### 4. 两阶段训练流程

**阶段一：语义 Grounding（解耦目录对齐）**

通过双向翻译目标将新增 SID token 与自然语言对齐，防止 SID 被视为任意符号：
- SID 序列 → 文本描述（title/summary/topic labels）
- 文本描述 → SID 序列

训练策略：
1. 冻结 Transformer 骨干，仅训练新增 SID token 嵌入
2. 冻结全部参数，使用 LoRA 适配 Transformer 层

> 注意：此阶段**不引入** soft prompt 用户向量，保持阶段纯净。

**阶段二：指令微调（任务特定个性化）**

在语义 grounding 模型基础上进行指令微调，此时联合训练 $`\mathrm{Proj}`$ 投影层与 LLM 参数，学习协同过滤空间到 LLM 内部表示的映射。

---

### 5. 可控多任务学习

Prompt 结构（图 Figure 2）：

```
[System Instruction] 定义角色（推荐器）和输出格式（SID）
[User Context]       <Soft Prompt Vector> ⊕ <Textual Metadata>
[Interaction History] <Recent Episode SIDs 序列>
[Task Instruction]   "Recommend a {{ familiarity_mode }} episode aligned with user interests."
```

通过控制 token 区分两种发现目标：
- `familiar`：目标 episode 属于用户曾收听过但未习惯化的节目
- `unfamiliar`：目标 episode 属于用户从未收听过的节目

推理时仅需切换控制 token，无需重新训练即可在"纯发现"和"熟悉发现"之间切换。

**训练数据去偏策略**：
- **跨界面采样**：从多个应用界面抽取训练样本，减少单一推荐界面的反馈循环效应
- **探索上采样**：提高随机化/探索位置产生的交互的采样权重（信号更少受先验曝光污染）
- **热门度截断**：限制单个 episode 的最大训练样本数，防止热门内容主导损失

---

### 6. 生产部署

#### 碰撞解决

多个 episode 映射到同一 SID 序列时，使用基于热门度的确定性 tie-breaker，每日更新，无需额外模型推理。

#### Beam Search 优化

使用 30 beams 生成 30 个候选，通过优化请求编排层和服务配置（而非修改推理引擎），将吞吐量提升 **8×**，支持 beam width 从 14 扩展到 30，同时满足生产延迟目标。

## 实验设置

| 项目 | 内容 |
|------|------|
| **骨干模型** | Llama 3.2 1B |
| **SID 配置** | R-KMeans，K=256，M=4，共 1024 SID tokens |
| **解码策略** | Beam Search，30 beams |
| **评估指标** | Recall@30、HitRate@30、NDCG@30（按 familiar/unfamiliar 分别报告） |
| **评估框架** | 传统检索指标 + 人工评估（Spotify 内部员工）+ LLM Judge |
| **在线实验** | 21 天随机用户级 A/B 测试，英语市场活跃用户，约 2000 万次曝光 |

## 实验结果

### 离线消融：条件信号贡献

相对于 SID-only 基线的提升：

| 方法 | Recall@30 | NDCG@30 |
|------|-----------|---------|
| LLM（SID only）基线 | — | — |
| LLM（SID + Text） | +25.0% | +28.2% |
| **GLIDE（完整）** | **+29.9%** | **+31.2%** |

**非习惯但熟悉**子集：

| 方法 | Recall@30 | NDCG@30 |
|------|-----------|---------|
| LLM（SID only）基线 | — | — |
| LLM（SID + Text） | +25.0% | +30.5% |
| **GLIDE** | **+28.7%** | **+28.7%** |

**非习惯且陌生**子集（发现任务核心）：

| 方法 | Recall@30 | NDCG@30 |
|------|-----------|---------|
| LLM（SID only）基线 | — | — |
| LLM（SID + Text） | +18.1% | +14.7% |
| **GLIDE** | **+29.5%** | **+35.4%** |

> 关键发现：用户 soft prompt 在"陌生"内容发现上贡献最大（+35.4% vs +14.7%），说明协同过滤信号对于交互信号最弱的场景最有价值。

### 离线消融：SID 量化方法对比

| 量化方法 | HR@30 | 桶内余弦相似度 |
|---------|-------|--------------|
| RQ-VAE（基线） | — | 0.657 |
| R-LFQ | +4.76% | 0.713 |
| **R-KMeans** | **+9.52%** | **0.856** |

### 离线消融：多任务学习效果

相对于单任务基线（无控制 token）：
- 使用 `unfamiliar` 控制 token：非习惯陌生内容 Recall@30 提升 **+11.8%**
- 使用 `familiar` 控制 token：非习惯熟悉内容 Recall@30 提升 **+4.9%**

### 离线消融：语义 Grounding 效果

有 vs 无 Semantic Grounding 阶段（均包含 soft prompt）：
- Recall@5 提升 **+8.34%**

### 离线消融：解码策略对比

| 解码策略 | 作用范围 | Recall@30 变化 |
|---------|---------|--------------|
| **Beam Search** | 完整序列（4 tokens） | — |
| Sampling | 完整序列（4 tokens） | **-27.07%** |
| Sampling | Prefix Ceiling（3 tokens） | -26.17% |
| Sampling | Prefix Ceiling（2 tokens） | -12.53% |

> 关键发现：前 2 个 token（粗粒度语义区域）采样即可接近 beam search，但第 3-4 个 token（细粒度 SID 定位）必须依赖 beam search，否则大幅退化。

### LLM Judge 评估

LLM Judge 评分与 Recall@30 正相关，同时能发现纯检索指标遗漏的质量差异：在某次评估中，LLM Judge 和人工评估员一致偏好某个模型变体，而离线 Recall 指标却偏好另一个——进一步分析发现后者存在更强的热门度偏差。

### 在线 A/B 测试（21 天，~2000 万曝光）

GLIDE 候选作为额外候选来源加入现有候选池，经下游排序后展示：

| 指标 | 提升 | 显著性 |
|------|------|-------|
| 非习惯性收听流量/用户 | **+5.4%** | $`\alpha < 0.01`$ |
| 新节目非习惯性收听流量 | **+14.3%** | $`\alpha < 0.01`$ |
| 整体参与度/用户满意度 | 无回归 | — |

GLIDE 候选占治疗组推荐的 ~34%，满足生产延迟和成本约束。

## 核心贡献总结

1. **指令跟随推荐形式化**：将播客推荐定义为 episode 级别的指令跟随任务，支持会话感知和可控发现

2. **实用 LLM 适配方案**：将 SID + soft prompt 用户个性化集成到单一可提示推荐架构，两阶段训练（语义 Grounding + 指令微调）有效解耦目录对齐与任务特定学习

3. **综合评估策略**：结合传统检索指标、人工评估和 LLM Judge 的三层评估框架，揭示了单一指标的局限性（热门度偏差导致的 Recall 虚高）

4. **生产规模验证**：21 天 A/B 测试证明非习惯性发现 +5.4%、新节目发现 +14.3%，满足实际延迟和成本约束

## 与相关工作的关键区别

| 对比维度 | GLIDE | TIGER/SASRec | 其他 LLM 推荐 |
|---------|-------|-------------|-------------|
| 目录 Grounding | R-KMeans SID，内容嵌入 | RQ-VAE SID | 通常无结构化 Grounding |
| 用户个性化 | Soft Prompt（CF 嵌入投影） | 无 | 文本序列化（上下文膨胀） |
| 可控性 | 控制 token 切换发现目标 | 无 | 有限 |
| 规模 | 生产部署，数千万用户 | 学术数据集 | 多为学术验证 |
| 冷启动 | 内容嵌入 SID 立即可用 | CF SID 需要交互量 | 依场景而定 |

---

## 局限性与未来工作

- 目前 LLM 如何利用预训练世界知识（如话题理解）进行推荐的机制尚未深入分析
- 语言条件控制目前仅用于 familiar/unfamiliar 两个维度，可扩展到更多目标和界面
- 发现导向推荐的长期效果评估方法仍需改进（21 天 A/B 测试可能无法捕捉长期口味演化）
