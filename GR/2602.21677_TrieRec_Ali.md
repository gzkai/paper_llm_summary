## 研究背景与动机

### 生成式推荐（Generative Recommendation, GR）范式

GR 将推荐问题转化为自回归生成任务，包含两个阶段：

1. **Item Tokenization（物品分词）**：通过 RQ-VAE 等层次化量化方法，将每个 item 映射为离散 token 序列 $\mathbf{c}_i = [c_{i,1}, c_{i,2}, \ldots, c_{i,L}]$（语义 ID）。
2. **Autoregressive Generation（自回归生成）**：将用户历史交互展开为 token 流 $\mathbf{X} = [c_{1,1}, \ldots, c_{T,L}]$，训练 Transformer 自回归预测下一个 item 的 token 序列。

层次化量化自然形成 **前缀树（Trie）**：每个 item 对应一条从根到叶的唯一路径，共享前缀的 item 语义更相近。

### 现有方法的核心缺陷：忽视 Trie 结构

现有 GR 方法（TIGER、LETTER、CoST 等）在自回归建模阶段将层次化 token 序列**展平为一维流**，依赖标准 Transformer 的线性位置编码，丢失了两类关键结构信号：

1. **绝对结构位置（Absolute Structural Position）**：token 在 Trie 中的深度和路径决定其语义，同一 token ID 在不同 item 路径中含义不同，但现有方法为其分配相同嵌入，造成**语义歧义**。

2. **相对拓扑关系（Relational Topology）**：
   - 同一路径上的 token 属于同一 item，具有强语义依赖
   - 共享更近最低公共祖先（LCA）的 token 对来自更相似的 item
   - 一维距离无法区分"item 内层次深度"与"item 间时序顺序"，造成**映射碰撞**

## 核心方法：TrieRec

TrieRec 通过两种专门的位置编码将 Trie 的结构归纳偏置注入 Transformer：

```
历史交互 token 流 X
    ↓ TAE（Trie-aware Absolute Positional Encoding）
    ↓ 注入绝对结构位置（路径上下文）
    ↓ TRE（Topology-aware Relative Positional Encoding）
    ↓ 注入 pairwise 拓扑关系（LCA 距离）
    ↓ 自回归生成下一 item token 序列
```

### 模块一：TAE（Trie-aware Absolute Positional Encoding）

**目标**：为每个 token $c_{i,l}$ 生成感知其在 Trie 中绝对位置的编码，包含祖先和后代的上下文信息。

**三步流程**：

① **拼接（Concatenation）**：收集 item $i$ 路径上所有层的 token 嵌入，拼接为扁平向量：

$$\mathbf{x}_i^{con} = [\mathbf{e}_{i,1}^\top; \mathbf{e}_{i,2}^\top; \ldots; \mathbf{e}_{i,L}^\top]^\top \in \mathbb{R}^{L \cdot d}$$

② **聚合（Aggregation）**：通过两层 MLP（含 GELU 激活和层归一化）进行跨 token、跨维度的非线性交互：

$$\hat{\mathbf{x}}_i = \text{MLP}(\mathbf{x}_i^{con}) \in \mathbb{R}^{L \cdot d}$$

③ **投影（Projection）**：使用层感知投影矩阵为每层生成专属位置编码：

$$\mathbf{p}_{i,l} = \mathbf{W}_{l} \hat{\mathbf{x}}_i + \mathbf{b}_{l}, \quad l=1, \ldots, L$$

**加入原始嵌入**：

$$\hat{\mathbf{e}}_{i,l} = \mathbf{e}_{i,l} + \mathbf{p}_{i,l}$$

**设计亮点**：
- MLP 是位置敏感的（非排列不变），可捕获层级关系
- 跨维度交互使不同维度特征相互校准（优于 DiscRec 的线性加权求和）
- 层感知投影建模不同层 token 的差异语义特性
- 仅在第一层 Transformer 施加 TAE，计算开销极小

### 模块二：TRE（Topology-aware Relative Positional Encoding）

**目标**：为每对 token $(u, v)$ 注入基于 Trie 拓扑的结构关系偏置，修正自注意力分数。

**三维拓扑坐标**：对序列中每对 token $(u, v)$，计算：
- $d^{LCA}_{u,v}$：其最低公共祖先（LCA）的树深度（共享前缀长度，反映语义相关程度）
- $\delta_u$：$u$ 到 LCA 的距离
- $\delta_v$：$v$ 到 LCA 的距离

**修正注意力分数**：

$$\tilde{A}_{u,v} = \text{Softmax}_v \left( \frac{(\mathbf{e}_u \mathbf{W}^Q)(\mathbf{e}_v \mathbf{W}^K)^\top}{\sqrt{d}} + \textcolor{red}{B(d^{LCA}_{u,v}, \delta_{u}, \delta_{v})} \right)$$

其中 $B(d^{LCA}_{u,v}, \delta_{u}, \delta_{v})$ 是以三元组为索引的可学习标量偏置（有限离散状态空间，直接穷举映射，无需桶化）。

**设计亮点**：
- 参数共享：$B$ 在所有 Transformer 层间共享，只需计算一次
- 计算高效：LCA 查找可预计算并缓存，运行时仅需 $O(1)$ 查表
- 三维坐标唯一标识 Trie 中的相对位置，解决一维距离的映射碰撞问题

### 方法优势总结

| 特性 | 描述 |
|------|------|
| **模型无关（Model-agnostic）** | 可无缝插入任意基于 Transformer 的 GR 骨干（只需修改嵌入和注意力分数） |
| **高效（Efficient）** | TAE 额外开销 $O(TL^2d^2)$，TRE 额外开销 $O(hT^2L^2\log L)$，均远小于基础 Transformer 的 $O(khT^2L^2d)$ |
| **无超参数（Hyperparameter-free）** | 不引入需要调优的新超参数 |

## 实验设置

### 数据集

| 数据集 | 用户数 | item 数 | 交互数 |
|--------|--------|---------|--------|
| Amazon Beauty | 22,332 | 12,086 | 198,215 |
| Amazon Toys | 19,124 | 11,758 | 165,247 |
| Amazon Food | 14,631 | 8,686 | 150,809 |
| MovieLens-1M | 6,040 | 3,416 | 999,611 |

- 5-core 过滤，留一法（leave-one-out）划分，保留最近 20 条交互

### 评估指标

- **Recall@K** 和 **NDCG@K**（K ∈ {5, 10}）
- 生成式方法使用约束束搜索（beam size=10）

### 基线方法

**传统序列推荐**：Caser、GRU4Rec、SASRec、BERT4Rec

**生成式推荐**：
- HSTU（ICML'2024）：高效注意力机制
- TIGER（NeurIPS'2023）：RQ-VAE + T5 的经典 GR 方法
- LETTER（CIKM'2024）：引入协同信号和多样性正则化的 RQ-VAE
- CoST（RecSys'2024）：用对比损失替换重建损失的 RQ-VAE
- CofiRec（ArXiv'2025）：粗到细分词 + 物品级位置编码

**位置编码对比方法**：
- HiT（ICPC'2023）：代码表示领域的树感知位置编码
- HiRoPE（ACL'2024）：基于 2D RoPE 的树感知位置编码
- DiscRec（ArXiv'2025）：物品内 token 显式信息共享
- LayerPE：CofiRec 中的层级位置编码组件（单独提取对比）

**Trie 感知损失方法**：
- SEATER：通过对比损失注入 Trie 结构信息的唯一现有工作

### 实现细节

- 骨干：4 层 T5，模型维度 128，6 个注意力头，MLP 隐层 1024，dropout 0.1
- RQ-VAE：3 个语义 token + 1 个碰撞 token，每个码本 256 个嵌入（维度 32），$\beta=0.25$
- 文本特征：Sentence-T5-base 提取
- 优化器：AdamW，学习率 $\{1e^{-3}, 2e^{-3}, 3e^{-3}, 4e^{-3}, 5e^{-3}\}$，weight decay $\{0.1, 0.2, 0.3, 0.4\}$

## 实验结果

### 主实验（RQ1）：与各基线对比

在 3 个 GR 骨干（TIGER、CoST、LETTER）× 4 个数据集上，TrieRec 均取得最优性能：

**TIGER 骨干的提升（相对 vanilla TIGER）**：

| 数据集 | N@5 | N@10 | R@5 | R@10 |
|--------|-----|------|-----|------|
| Beauty | +5.35% | +3.80% | +2.72% | +0.56% |
| Toys | +10.84% | +6.22% | +5.79% | +0.13% |
| Food | +9.20% | +8.49% | +7.51% | +6.44% |
| ML-1M | **+17.91%** | **+15.46%** | **+16.06%** | **+12.61%** |

**LETTER 骨干的提升（相对 vanilla LETTER）**：

| 数据集 | N@5 | N@10 | R@5 | R@10 |
|--------|-----|------|-----|------|
| Beauty | +11.59% | +8.67% | +7.03% | +3.79% |
| Toys | +16.14% | +10.93% | +8.80% | +2.56% |
| Food | +9.38% | +5.63% | +6.65% | +4.11% |
| ML-1M | +17.33% | +13.93% | +13.58% | +9.31% |

**跨三个骨干平均提升：8.83%（NDCG@10）**

**关键发现**：
- HiT 和 HiRoPE（来自代码生成领域的树感知编码）在推荐场景效果有限，甚至不如 vanilla 骨干
- LayerPE 和 DiscRec 的粗粒度结构感知不如 TrieRec 的精细 Trie 拓扑重建
- SEATER（Trie 感知对比损失）效果显著弱于 TrieRec，说明通过辅助损失注入 Trie 信息不如直接编码进 Transformer

**TrieRec vs. SEATER（LETTER 骨干）**：

| 数据集 | SEATER N@5 | TrieRec N@5 | SEATER N@10 | TrieRec N@10 |
|--------|-----------|------------|------------|-------------|
| Beauty | 0.0334 | **0.0385** | 0.0413 | **0.0464** |
| Toys | 0.0351 | **0.0403** | 0.0432 | **0.0477** |
| Food | 0.0342 | **0.0373** | 0.0440 | **0.0469** |
| ML-1M | 0.0726 | **0.0914** | 0.0948 | **0.1129** |

### 消融实验（RQ2）

以 LETTER 为骨干，系统验证各组件贡献（4 个数据集上的完整结果）：

| 变体 | 说明 | 效果 |
|------|------|------|
| **TrieRec（完整）** | 完整模型 | **最优** |
| w/o TAE | 去掉 TAE 模块 | 显著下降，路径上下文缺失 |
| w/o TRE | 去掉 TRE 模块 | 显著下降，拓扑关系缺失 |
| TAE-Blocked | MLP 仅允许同维度交互，禁止跨维度 | 性能下降，验证跨维度交互的必要性 |
| TAE-Prefix | 用前缀和替换 MLP 聚合 | **最差变体**，甚至不如 vanilla LETTER，说明简单累加不能捕获层次关系 |
| TRE-w/o LCA | 去掉 LCA 深度维度 | 下降，LCA 提供全局层次定位 |
| TRE-w/o δ | 去掉相对距离维度 | 下降，路径距离提供局部精确定位 |

> **结论**：TAE 和 TRE 互补，共同构成多层次防御；跨维度 MLP 交互和完整三维 LCA 坐标均不可缺少。

### 效率分析（RQ3）

与 vanilla LETTER 相比，TrieRec 额外计算开销 **< 10%**，训练时间和推理延迟均在同一数量级，性价比极高。

## 核心贡献总结

1. **识别关键问题**：首次系统指出现有 GR 方法忽视 Trie 结构的问题，并从绝对结构位置和相对拓扑关系两个维度分析其危害。

2. **TAE（Trie-aware Absolute Positional Encoding）**：通过拼接-MLP-投影架构，为每个 token 生成感知其在 Trie 中绝对位置的编码，捕获路径内的跨 token、跨维度非线性依赖。

3. **TRE（Topology-aware Relative Positional Encoding）**：将 token 对的 LCA 深度和路径距离编码为可学习的注意力偏置，将 Trie 拓扑关系直接注入自注意力机制。

4. **模型无关、高效、无超参数**：TrieRec 可即插即用于任意 GR 骨干，额外开销 < 10%，无需调整新超参数。

5. **实验验证**：在 4 个数据集 × 3 个骨干上平均提升 **8.83%**，显著优于所有基线，包括专门设计的树感知位置编码方法和 Trie 感知损失方法。
