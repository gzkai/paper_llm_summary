## 研究背景与动机

### 问题定义：HintQ 场景

**HintQ**（Hint Query）是淘宝首页的一个**预搜索查询推荐**功能，在用户尚未输入任何关键词之前，根据用户的历史行为序列和画像属性，主动推荐个性化的搜索查询词。

形式化定义：给定用户 $`u`$ 的人口统计画像 $`\mathcal{P}_u`$（年龄、性别、地区等）和时序行为序列 $`\mathcal{S}_u = \{(t_i, a_i, c_i)\}_{i=1}^L`$（$`t_i`$ 为距今天数，$`a_i \in \{\text{search}, \text{click}\}`$，$`c_i`$ 为内容），目标是生成排序好的 $`K`$ 条推荐查询词列表 $`\mathcal{Q} = [q_1, q_2, \dots, q_K]`$，同时优化相关性与业务指标（CTR、GMV）。

### 现有方法的不足

1. **传统方法**（协同过滤、DNN 匹配）：语义浅层、冷启动表现差、惊喜度低，依赖 ID-based 匹配和共点击启发式规则
2. **工业 LLM 部署范式**：通常离线构建静态的 context-to-query 映射，无法捕捉用户多面兴趣，且下游 ranker 优化短期 CTR 与语义质量存在 misalignment
3. **端到端生成方案**：在线 beam search 计算开销大、实时延迟预算紧张

## 核心方法：AIGQ 框架

AIGQ 包含三大核心创新：**训练范式**、**策略优化**、**部署架构**。

### 1. 用户中心的上下文工程

#### 行为内容表示

- 搜索词直接使用文本
- 商品点击行为：微调 Qwen3-32B 将商品多模态元数据（标题、类目、属性、标签等）转换为简洁文本描述

#### Prompt 压缩（两种策略）

**（1）特征结构化**：将小词表的离散字段（时间戳、行为类型）编码为特殊 token：

```math
\text{``1|1 day ago|search|iPhone17''} \Rightarrow \text{<1><1\_day\_ago><search> iPhone17}
```

新增特殊 token 的 embedding 初始化为其组成子词 embedding 的均值（如 `<1_day_ago>` = mean("1", "day", "ago")）。

**（2）指令裁剪**：SFT 已内化任务目标，采用最简指令格式，去除冗余说明文本。

### 2. 统一样本构建框架（IL-SFT 的数据基础）

两阶段流水线：

#### 阶段一：Session-Aware 行为聚合

- **AIGQ-Direct**：以单次 HintQ 曝光为 session 单元（至少有一次点击才算有效 session），高行为保真度但上下文较窄
- **AIGQ-Think**：以天级 session 定义：

```math
\mathcal{S}_u^{\text{AIGQ-Think}} = \{ h \in \mathcal{H}_u \mid t_{\text{first}} \leq h.\text{time} \leq t_{\text{exit}} \}
```

聚合跨域行为，发现多样兴趣。

#### 阶段二：兴趣引导的标签重排序

- **AIGQ-Direct** 标签列表：

```math
\mathcal{Q}_u^{\text{AIGQ-Direct}} = \texttt{Concat}(\mathcal{Q}_u^{\text{HintQ-Click}}, \mathcal{Q}_u^{\text{Rank-TopK}})
```

- **AIGQ-Think** 标签列表（聚合更多信号）：

```math
\mathcal{Q}_u^{\text{AIGQ-Think}} = \texttt{Concat}\big( \mathcal{Q}_u^{\text{HintQ-Click}},\, \mathcal{Q}_u^{\text{Global-Search}},\, \mathcal{Q}_u^{\text{Rank-TopK}},\, \mathcal{Q}_u^{\text{LLM-Gen}} \big)
```

可靠性从高到低排列，并使用强 LLM 过滤与用户历史弱相关的候选。

### 3. Interest-Aware List SFT（IL-SFT）

与传统 NTP 不同，IL-SFT 将目标视为**有序的兴趣驱动列表**，而非无结构 token 序列：

```math
\mathcal{L}_{\text{SFT}} = -\mathbb{E}_{(\mathbf{x}, \mathbf{z}) \sim \mathcal{D}} \left[ \sum_{t=1}^{T} \log P(z_{t} \mid z_{\lt t}, \mathbf{x}; \theta) \right]
```

其中 $`\mathbf{x} = [\mathcal{P}, \mathcal{S}]`$ 为用户上下文。

#### AIGQ-Direct

直接生成 Top-K 查询词的平铺列表，训练集 $`\mathcal{D}_{direct} = \{ (\mathbf{x}^{(n)}, \mathbf{y}^{(n)}) \}_{n=1}^N`$。

#### AIGQ-Think（CoT 推理增强）

两阶段认知过程：
1. 归纳多兴趣点（trigger 行为集合 $`\{ \mathcal{T}_k \}_{k=1}^K`$）
2. 为每个兴趣点生成查询词列表 $`\mathcal{Q}_k`$

输出结构：$`\mathbf{y} = \{ (\mathcal{T}_1 \rightarrow \mathcal{Q}_1), \dots, (\mathcal{T}_K \rightarrow \mathcal{Q}_K) \}`$

使用 Qwen3-32B 作为教师模型生成推理链 $`\mathbf{r}_k`$，并构建蒸馏数据集：

```math
\mathcal{D}_{think} = \left\{ \big( \mathbf{x}^{(n)},\, \mathbf{r}^{(n)},\, \mathbf{y}^{(n)} \big) \right\}_{n=1}^N
```

### 4. Interest-aware List GRPO（IL-GRPO）

标准 GRPO 对整个序列赋予单一标量奖励，导致高质量查询词因邻居较差而被错误惩罚（credit assignment 问题）。IL-GRPO 引入**双层奖励机制**。

#### 双层奖励设计

```math
\mathcal{R}_{\text{query}}(q_{i,j}) = \sum_{m} w_m r_m(q_{i,j}), \quad \mathcal{R}_{\text{seq}}(o_i) = \mathcal{R}_{\text{fmt}}(o_i) + \mathcal{R}_{\text{rep}}(o_i)
```

- **查询级奖励** $`\mathcal{R}_{\text{query}}`$ 集成三方面：
  - 用户偏好对齐（线上 CTR 模型预测概率）
  - 语义保真度（ROUGE-L）
  - 查询一致性（长度约束）
- **序列级奖励** $`\mathcal{R}_{\text{seq}}`$：格式合规性 + 去重惩罚

优势估计（对每个 token $`t`$ 属于第 $`j`$ 个查询位置）：

```math
A_{i,t} = \underbrace{\frac{\mathcal{R}_{\text{query}}(q_{i,j}) - \mu_j^Q}{\sigma_j^Q}}_{\text{Query Advantage}} + \lambda \underbrace{\frac{\mathcal{R}_{\text{seq}}(o_i) - \mu^S}{\sigma^S}}_{\text{Sequence Advantage}}
```

其中 $`\mu_j^Q, \sigma_j^Q`$ 是第 $`j`$ 个查询槽的位置特定统计量，$`\mu^S, \sigma^S`$ 是序列级统计量。

#### 持续 CTR 奖励适配

CTR 模型每日更新，奖励公式（线性缩放 + 截断）：

```math
r_{\text{CTR}}(q_j) = \text{Clip}\left( \gamma \cdot \hat{p}_{\phi_T}(q_j \mid u), \, 0, \, \beta \right)
```

日迭代更新机制：

```math
\phi_{T+1} \leftarrow \text{Train}(\phi_T, \mathcal{D}_T), \quad \pi_{\theta_{T+1}} \leftarrow \text{RL}(\pi_{\theta_T}, r_{\text{CTR}}^{\phi_{T+1}})
```

#### 解耦的 Trigger 奖励优化（AIGQ-Think 专用）

Trigger 侧使用专属的结构一致性目标（ROUGE-L），与查询侧奖励严格分离：

```math
r_{\text{trigger}}(\mathcal{T}_k) = \text{ROUGE-L}_F\big( \mathcal{T}_k^{\text{gen}}, \mathcal{T}_k^{\text{ref}} \big)
```

Trigger token 的优势估计仅来自 $`r_{\text{trigger}}`$，查询 token 使用复合优势，防止梯度干扰。

#### 熵引导的训练稳定化

1. **动态熵正则化**：损失中加入 $`\beta(s)\mathcal{H}(\pi_\theta)`$，$`\beta(s)`$ 随训练步数线性衰减
2. **熵自适应裁剪**：

```math
\epsilon(e_t) =
\begin{cases}
\epsilon_{\text{high}}, & \text{if } e_t \geq \tau \\
\epsilon_{\text{low}},  & \text{if } e_t < \tau
\end{cases}
```

$`\tau`$ 为当前 batch 中 token 熵的第 80 百分位数，不确定 token 给予更大更新空间。

3. **On-policy 更新**：每步新采样 rollout
4. **难度过滤**：聚焦中等难度样本（奖励阈值 $`[0.1, 0.95]`$）

### 5. 混合离线-在线部署架构

#### AIGQ-u2q 近线召回

- **AIGQ-Direct** 异步推理（单次请求需数秒，远超在线延迟预算）
- 检测到有意义行为信号时触发推理，结果写入用户级 KV 缓存（保留最近 $`M`$ 次）
- 在线服务时直接读取缓存，作为召回源

#### AIGQ-x2q 实时召回

解决近线缓存的"时效性缺口"（最新点击可能未被捕获）：

- **AIGQ-Think** 离线每日批量生成 CoT 输出 $`\{\mathcal{T}_k \rightarrow \mathcal{Q}_k\}`$
- 构建 trigger-to-query 索引：每个 trigger $`x_i`$ 映射到关联查询集 $`\mathcal{Q}_k`$
- 综合打分：

```math
S_{\mathrm{final}}(x, q) = \alpha \cdot S_{\mathrm{rel}}(x, q) + (1 - \alpha) \cdot S_{\mathrm{eff}}(q)
```

$`S_{\mathrm{rel}}`$ 为推理关联度，$`S_{\mathrm{eff}}`$ 聚合 PV、预测 CTR 等业务信号，$`\alpha`$ 可调。

## 实验结果

### 数据集与设置

- **数据集**：淘宝线上 e-commerce 日志（2025年10月~11月），训练集约 100 万 PV（1-30天），测试集（31-32天）
- **评估指标**：Query HR@30、Cate HR@30、Semantic Similarity（Sem. Sim.）、Unique Leaf Categories（Unique Cates）
- **基线**：EBR（双塔 DNN，线上最强非生成召回）、零样本 LLM（Qwen3-30B-A3B、Gemini 3 Pro、GPT-5.1）、标准 SFT/GRPO 变体

## 实现细节

- **基础模型**：Qwen3-30B-A3B（30B 稀疏 MoE，8 个路由专家）
- **部署平台**：阿里内部 PPU 810E（约为 NVIDIA A100 算力的 60%）
- **词表扩展**：新增 68 个特殊 token（行为索引 1-50、时间偏移、行为类型）
- **SFT 训练**：2 epochs，Adam 优化器，batch size 1024，lr $`2 \times 10^{-3}`$，Megatron 框架
- **IL-GRPO 训练**：ROLL 框架，rollout batch 512，lr $`1 \times 10^{-6}`$，每个样本采样 8 个响应（temperature=1.0, top-k=100），奖励截断 $`\pm 10`$，优势截断 $`\pm 2.0`$

## 核心贡献总结

1. **首个端到端生成式 HintQ 推荐框架**：提出 AIGQ-Direct（轻量化近线个性化生成）和 AIGQ-Think（CoT 推理增强多兴趣生成）两个变体

2. **IL-SFT**：兴趣感知列表级 SFT，通过 session-aware 行为聚合 + 兴趣引导标签重排，将有序列表作为监督目标，超越传统 NTP

3. **IL-GRPO**：新型策略梯度算法，在查询粒度估计优势，双层奖励（本地查询级 + 全局序列级）+ 持续 CTR 奖励适配 + 解耦 trigger 奖励 + 熵引导稳定化，有效解决列表生成的 credit assignment 问题

4. **混合部署架构**：AIGQ-u2q 近线召回（AIGQ-Direct 异步推理+缓存）+ AIGQ-x2q 实时召回（AIGQ-Think CoT 输出构建 trigger-to-query 索引），在严苛延迟约束下实现深度个性化
