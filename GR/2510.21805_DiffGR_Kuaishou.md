## 研究背景与动机

### 生成式推荐（GR）框架

生成式推荐将推荐任务转化为序列生成：
1. **Tokenization**：用预训练语言模型将物品内容编码为稠密向量，再通过向量量化（VQ）离散化为 $`n`$ 位语义 ID（SID），每位从码本中选一个 Token
2. **Generation**：用自回归模型（ARM，类 GPT）逐位预测目标物品的 SID

### SID 的两个结构特性

**特性一：物品内一致性（Intra-item Consistency）**

$`n`$ 位 SID 共同唯一指定一个物品（如 "Dior Rouge 999 Velvet"），各位之间具有内在关联。

**特性二：位间异质性（Inter-digit Heterogeneity）**

不同位编码不同语义粒度（如 Category / Brand / Type / Size），预测难度差异显著，存在"易位"和"难位"之分。

### ARM 的两大缺陷

| 缺陷 | 描述 |
|------|------|
| **信息壁垒** | 因果注意力强制从左到右依赖，每位只能看到前缀，无法利用双向物品内语义和跨位互验，监督信号退化为单一因果路径 |
| **监督失衡** | 下一 Token 目标对所有位分配等权重，"难位"训练不足、"易位"过训练，导致 SID 生成不准确 |

**具体例子**（Figure 1）：第二位（Brand）是难位，第三位（Ball Type）是易位。ARM 在预测第二位时只能看到第一位前缀，预测出错（131 而非 134），错误沿因果链传播。DiffGRM 先填高置信度的第三位（56），再以第一位 + 第三位为上下文修正第二位（134）。

## 核心方法：DiffGRM

**DiffGRM**（**Diff**usion-based **G**enerative **R**ecommendation **M**odel）用**掩码离散扩散模型（MDM）**替换自回归解码器，实现双向上下文 + 任意顺序并行生成。

### 整体架构

```
用户历史序列（SID 序列）
         ↓
   Transformer 编码器 → H_u（用户历史表示）
         ↓
   MD-Decoder（掩码扩散解码器）
   - 双向（非因果）自注意力
   - Cross-Attention（KV 来自编码器 H_u）
   - 并行预测所有被掩码的 SID 位
         ↓
   目标物品 SID (s⁰, s¹, ..., sⁿ⁻¹)
```

---

### 创新一：并行语义编码（PSE，Parallel Semantic Encoding）

**问题**：现有 GR 系统普遍使用残差量化（RQ-VAE / RQ-KMeans），后层 Token 依赖前层残差，存在从左到右的顺序偏置，与 MDM 的并行预测冲突，且加剧位间异质性。

**解决方案**：采用 OPQ（Optimized Product Quantization）并行分区量化：

1. 学习正交旋转矩阵 $`\mathbf{R}_o`$，最小化量化失真：$`\tilde{\mathbf{h}}_i = \mathbf{R}_o \mathbf{h}_i`$
2. 将旋转后的向量 **均匀切分** 为 $`n`$ 个子向量：$`\tilde{\mathbf{h}}_i = \mathbf{v}_i^1 \oplus \cdots \oplus \mathbf{v}_i^n`$
3. 每个子向量**独立**分配到对应码本：

```math
s_i^k = \arg\min_j \|\mathbf{v}_i^k - \mathbf{c}_j^{(k)}\|_2^2
```

**效果**：各位赋值解耦，消除残差顺序依赖，支持真正的并行预测，并均衡各位信息量。

---

### 创新二：在线一致噪声（OCN，On-policy Coherent Noising）

**问题**：MDM 的随机掩码导致监督稀疏且不均衡。理论分析：$`n`$ 层码本下 MDM 有 $`n \cdot 2^{n-1}`$ 个监督信号，全覆盖需至少 $`2^n - 1`$ 种掩码配置，而 ARM 只需 $`n`$ 个信号且 1 个样本即可覆盖。

**解决方案**：以当前模型为策略 $`\pi_\theta`$，优先对**难位**施加噪声，构造一组嵌套视图（Views）。

**算法流程**：

1. **难度评估**：对完全掩码输入运行一次 MD-Decoder，得到每位的置信度和难度分：

```math
p_{\max}^{(k)} = \max_{v} p_\theta^{(R,k)}(v), \quad \delta^{(k)} = 1 - p_{\max}^{(k)}
```

$`\delta^{(k)}`$ 越大 → 该位越难（置信度越低）

2. **难度排序**：按 $`\delta^{(k)}`$ 降序得到排列 $`\sigma`$（从难到易）

3. **构造嵌套视图**：给定递增调度 $`1 \le m_1 < \cdots < m_R \le n`$，视图 $`r`$ 掩码最难的 $`m_r`$ 位：

```math
\mathcal{M}^{(r)} = \{\sigma(1), \ldots, \sigma(m_r)\}
```

```math
\mathbf{y}^{(r,k)} = \begin{cases} \mathbf{E}_{\text{mask}}[k], & k \in \mathcal{M}^{(r)} \\ \mathbf{E}_{\text{sid}}^{(k)}[s^k], & k \notin \mathcal{M}^{(r)} \end{cases}
```

4. **多视图训练损失**：

```math
\mathcal{L} = \frac{1}{R} \sum_{r=1}^{R} \mathcal{L}^{(r)}, \quad \mathcal{L}^{(r)} = \frac{1}{|\mathcal{M}^{(r)}|} \sum_{k \in \mathcal{M}^{(r)}} \left(-\sum_{v=0}^{M-1} \tilde{q}_v^{(k)} \log p_\theta^{(r,k)}(v \mid \mathbf{y}^{(r)}, \mathbf{H}_u)\right)
```

**效果**：
- 嵌套掩码集避免组合爆炸，稳定优化
- 梯度集中于相同难位，可见上下文随视图递增
- 编码器输出 $`\mathbf{H}_u`$ 跨视图复用，计算高效

---

### 创新三：置信度引导并行去噪（CPD，Confidence-guided Parallel Denoising）

**问题**：推荐系统需要返回多样的 Top-K 候选集，而 MDM 常用的贪心填充只能产生单个 Top-1 输出；ARM 的从左到右 Beam Search 无法充分利用 MDM 的并行生成能力。

**解决方案**：基于 MD-Decoder 的全局并行 Beam Search，优先填充高置信度的位。

**算法**（Beam Width $`B_{\text{act}}`$）：

1. **初始化**：从完全掩码输入出发，选取 Top-$`B_{\text{act}}`$ 个最高置信度的（位, 码字）对：

```math
\mathcal{B}_R = 	ext*{Top}_{B_{\text{act}}} \left\{ \log p_\theta(y_k = c \mid \mathbf{y}^{(R)}, \mathbf{H}_u) \right\}_{k,c}
```

2. **逐步去噪**：在每个反向步骤 $`t_r`$，对每个活跃分支中仍被掩码的位打分：

```math
s_{r-1}(b, k, c) = \text{score}_r(b) + \log p_\theta(y_k = c \mid \mathbf{y}_b^{(r)}, \mathbf{H}_u)
```

3. **剪枝保留**：保留 Top-$`B_{\text{act}}`$ 个最高分的候选，更新掩码集：

```math
\mathcal{B}_{r-1} = 	ext*{Top}_{B_{\text{act}}} \{s_{r-1}(b, k, c)\}
```

4. 重复至 $`t_1 = 0`$，去重后保留 Top-K

**效果**：不依赖固定从左到右顺序，高置信度位优先填充，低置信度位在更丰富的上下文下再预测，产生更准确且多样的 Top-K 候选。

### 复杂度分析

| 模块 | ARM | DiffGRM |
|------|-----|---------|
| 编码器 | $`O(N^2 d_m)`$ | $`O(N^2 d_m)`$ |
| 解码器（训练） | $`O(n^2 d_m + nNd_m)`$ | $`R \cdot O(n^2 d_m + nNd_m)`$ |
| 解码器（推理） | $`O(B_{\text{act}} \cdot n \cdot N d_m)`$ | $`O(B_{\text{act}} \cdot n^2 \cdot N d_m)`$ |

- **训练**：工业场景历史序列长（$`N`$ 大），以编码器为主，ARM 和 DiffGRM 总体训练复杂度相当
- **推理**：DiffGRM 多一个 $n$ 因子，但 $n$ 通常很小（3–4），且额外评分在 GPU 上并行执行（利用缓存 KV），实际墙钟延迟接近 ARM

## 实验结果

### 数据集

| 数据集 | 用户数 | 物品数 | 交互数 | 平均序列长度 |
|--------|--------|--------|--------|------------|
| Sports and Outdoors | 35,598 | 18,357 | 260,739 | 8.32 |
| Beauty | 22,363 | 12,101 | 176,139 | 8.87 |
| Toys and Games | 19,412 | 11,924 | 148,185 | 8.63 |

评估方式：留最后一项（leave-last-out），指标为 Recall@K 和 NDCG@K（K ∈ {5, 10}）

### 整体性能对比（RQ1）

**vs 最优基线（Sports: ActionPiece/RPG，Beauty/Toys: RPG）**：

| 数据集 | 指标 | 次优基线 | **DiffGRM** | 提升 |
|--------|------|---------|------------|------|
| Sports | NDCG@10 | 0.0264 | **0.0305** | **+15.53%** |
| Sports | Recall@10 | 0.0500 | **0.0550** | **+10.00%** |
| Beauty | NDCG@10 | 0.0464 | **0.0502** | **+8.19%** |
| Beauty | Recall@10 | 0.0809 | **0.0876** | **+8.28%** |
| Toys | NDCG@10 | 0.0490 | **0.0524** | **+6.94%** |
| Toys | NDCG@5 | 0.0401 | **0.0455** | **+13.47%** |

DiffGRM 在 12 个指标中排名第一的有 11 个（Toys Recall@10 略低于 RPG 的 0.0869，为 0.0834）。

### 消融实验（RQ3）

**NDCG@10 对比**：

| 变体 | Sports | Beauty | Toys |
|------|--------|--------|------|
| (1.1) PSE → RQ-KMeans | 0.0200 | 0.0343 | 0.0305 |
| (1.2) PSE → Random | 0.0138 | 0.0300 | 0.0206 |
| (2.1) w/o OCN（随机掩码） | 0.0250 | 0.0368 | 0.0385 |
| (2.2) w/o On-policy（仅一致噪声） | 0.0263 | 0.0455 | 0.0430 |
| (3.1) w/o CPD（随机固定顺序 Beam Search） | 0.0273 | 0.0496 | 0.0499 |
| **DiffGRM（完整）** | **0.0305** | **0.0502** | **0.0524** |

**关键结论**：
- PSE 必须：RQ-KMeans 引入顺序依赖，与 MDM 并行预测冲突，性能大幅下降
- OCN 有效：随机掩码（w/o OCN）最差；加入一致噪声有提升；再加 On-policy 选择最优
- CPD 有效：随机顺序 Beam Search 性能下降，置信度引导顺序填充更优

### OCN 变体分析

| 策略 | Beauty NDCG@10 | Toys NDCG@10 |
|------|---------------|-------------|
| **L-S（最低置信度，静态排序，Ours）** | **0.0502** | **0.0524** |
| L-R（最低置信度，逐步刷新） | 0.0484 | 0.0481 |
| M-S（最高置信度，静态排序） | 0.0476 | 0.0516 |
| M-R（最高置信度，逐步刷新） | 0.0382 | 0.0421 |

- **选择策略**：优先掩码"难位"（低置信度）优于优先掩码"易位"
- **刷新频率**：静态（一次性确定顺序）优于逐步刷新（每步重新估计）
- M-R 最差：先填易位 + 逐步刷新 → 难位长期训练不足，且 CPD 移除后性能骤降 -24.47%

### OCN 样本效率（RQ2）

引入**有效样本轮次（ESP）**度量：$`\text{ESP} = \text{best\_epoch} \times \text{views\_per\_sample\_per\_epoch}`$

实验对比：固定 $k$ 倍一致路径噪声（ESP 线性增长）vs OCN（在线选择难位）

**结论**：OCN 在相同或更低 ESP 下超越 $k$ 倍一致路径设置，说明 On-policy 选择显著提高了样本效率。

### 语义编码器规模分析

| 模型 | 语义编码器 | Sports | Beauty | Toys |
|------|-----------|--------|--------|------|
| RPG | sentence-t5-base | 0.0238 | 0.0429 | 0.0460 |
| RPG | bge-large-en-v1.5 | 0.0248 | 0.0408 | 0.0421 |
| **DiffGRM** | sentence-t5-base | 0.0305 | 0.0502 | 0.0524 |
| **DiffGRM** | bge-large-en-v1.5 | **0.0327** | **0.0564** | 0.0508 |
| **DiffGRM** | gte-large-en-v1.5 | **0.0342** | 0.0549 | 0.0510 |

DiffGRM 从更强语义编码器中受益更多（RPG 提升不稳定，DiffGRM 整体提升明显）。

## 核心贡献总结

1. **首个扩散式生成推荐框架**：用掩码离散扩散模型（MDM）替换 ARM 解码器，消除从左到右约束，实现双向跨位上下文和并行生成

2. **PSE（并行语义编码）**：基于 OPQ 的并行分区量化，消除残差顺序依赖，均衡各位信息量，与 MDM 并行预测天然契合

3. **OCN（在线一致噪声）**：以当前模型为策略，优先对难位施加噪声，构造嵌套视图集中监督预算，显著提升样本效率

4. **CPD（置信度引导并行去噪）**：全局并行 Beam Search，高置信度位优先填充，产生准确且多样的 Top-K 候选集

5. **SOTA 性能**：在 Amazon Sports/Beauty/Toys 三个基准上，NDCG@10 相比最强基线提升 **6.9%–15.5%**
