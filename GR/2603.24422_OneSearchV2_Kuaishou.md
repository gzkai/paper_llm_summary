## 研究背景与动机

### OneSearch-V1 的三大局限

OneSearch-V2 是对已全量部署的 OneSearch-V1 的系统性升级，针对其三大核心缺陷：

| 缺陷 | 具体问题 |
|------|---------|
| **复杂查询理解不足** | 短尾头部查询（如"室内健身器材"）候选空间宽泛，面临"检索哪个"的困境；长尾查询（否定型、疑问型）语义理解困难，面临"能检索什么"的困境。复杂查询占总 PV 约 1/3，但仅贡献 8% 的转化 |
| **个性化意图推理不足** | 依赖历史共现模式和 log-fitting，无法发掘用户真实意图。例如对花粉过敏用户搜索"当季鲜花"时，模型应主动推断季节、过滤过敏品种，但直接推理受延迟约束无法完成 |
| **奖励系统脆弱且有分布偏差** | 分离训练的奖励模型存在采样偏差和 reward hacking，导致模型过拟合历史窄分布偏好，强化长尾分布偏差；多阶段周期性更新无法及时响应新兴查询 |

---

## 核心贡献：三大创新模块

### 总体框架

OneSearch-V2 由三大模块组成，并在 V1 的四阶段训练流程基础上扩展：

| 训练阶段 | 目标 | 核心组件 |
|---------|------|---------|
| **SFT Stage 1**（语义对齐） | query/item ↔ SID + 类目映射 + **CoT 任务** | Keyword-based CoT 四类任务 |
| **SFT Stage 2**（共现同步） | query ↔ item 共现 | SID_q ↔ SID_i |
| **SFT Stage 3**（用户个性化） | uid + 行为序列 → SID | **推理内化自蒸馏** + RAG 关键词注入 |
| **RL Stage**（偏好对齐） | 用户行为反馈排序 | **TPMA-GRPO** 直接行为反馈 |

### 一、单模态 vs 多模态 SID Tokenization 的选择

在正式介绍方法之前，论文通过实验回答了电商生成式检索中 SID 编码范式的选择问题。

**结论：KHQE（关键词层次量化编码）优于所有多模态方案**

| 类型 | 模型 | CUR | ICR | Recall@10 | MRR@10 |
|------|------|-----|-----|-----------|--------|
| 单模态 | bge-base (109M) | 4.54% | 96.88% | 0.2445 | 0.1013 |
| 单模态 | qwen3 (0.6B) | 5.11% | 97.56% | 0.2468 | 0.1025 |
| 多模态 | uniecs (200M) | 4.54% | 94.62% | 0.2368 | 0.1007 |
| 多模态 | qwen3-vl (2B) | 4.86% | 95.27% | 0.2389 | 0.1012 |
| **KHQE** | **bge+kw.** | **5.11%** | **99.50%** | **0.2542** | **0.1085** |

**原因**：多模态编码存在跨模态表示差异和冗余属性干扰（如不同颜色的同款服装图片会引入互斥属性），导致核心属性被淹没。KHQE 通过 Qwen-VL 提取核心关键词构建统一文本表示，兼顾性能与效率。

### 二、思维增强查询理解（Thought-Augmented Query Understanding）

#### 关键词式 CoT（Keyword-based CoT）

区别于完整 CoT（生成冗长自然语言推理链），论文提出三步关键词式 CoT 提取流程，信息密度高且推理成本低：

**Step 1：查询分析（Query Analysis）**
通过 Qwen3-32B 对每个查询进行四维分析：
- **意图理解**：识别主检索目标（商品/店铺/直播主播）
- **类目识别**：从粗到细的层次类目匹配
- **属性识别**：提取查询中明确的属性类型和值
- **主题推荐**：推测满足用户需求的候选主题

**Step 2：关键词提取（Keyword Extraction）**
对商品检索意图的查询，从分析结果中提取关键词，约束包括：
- 意图一致性、类目一致性、属性一致性
- 同义词合并、冗余词删除
- 按物品热度降序排列，最多 8 个关键词

**Step 3：偏好校准（Preference Calibration）**
结合用户画像（历史搜索、交互物品序列），对关键词集合进行过滤或增补，使其更贴合个人兴趣。训练时额外注入当前 session 的交互物品信号，确保 ground-truth 物品对应的关键词被保留。

**训练范式**：
- $`\langle`$query, keywords$`\rangle`$（Step 2）和 $`\langle`$query, user, keywords$`\rangle`$（Step 3）构成训练语料
- 在 SFT Stage 1 中加入四类 CoT 任务，引导模型学习超越历史 log 的查询知识

**推理部署**：
- 关键词式 CoT 异步生成，结果缓存复用，**不增加在线推理延迟**
- 相同 query 或 $`\langle`$query, user$`\rangle`$ 对可直接复用缓存结果

### 三、推理内化自蒸馏（Reasoning-Internalized Self-Distillation）

#### 核心问题

直接将 CoT 关键词拼接到输入（RAG 方式）效果好，但在线获取 CoT 需额外调用大模型，延迟不可接受。直接让模型先生成 CoT 再生成 SID（direct CoT）则因 SID 与文本的表示异质性而严重降低性能。

**关键问题**：能否在不增加推理成本的前提下，保留甚至增强推理带来的性能增益？

#### 自蒸馏框架

**信息不对称原则**：Teacher 观察更丰富的输入（含关键词），Student 从更少信息中匹配 Teacher 的输出分布。Teacher 和 Student **共享同一套模型权重**，无需额外参数，内存占用仅为传统知识蒸馏的一半。

**Teacher 输入**（含关键词）：

```math
x^{(T)} = (\text{uid},\, q,\, \text{SID}_q,\, \text{Seq}_q,\, \text{Seq}_{\text{short}},\, \text{Seq}_{\text{long}}^{\text{emb}},\, \textbf{kw})
```

**Student 输入**（不含关键词）：

```math
x^{(S)} = (\text{uid},\, q,\, \text{SID}_q,\, \text{Seq}_q,\, \text{Seq}_{\text{short}},\, \text{Seq}_{\text{long}}^{\text{emb}})
```

**蒸馏损失**（KL 散度，Teacher logits 不参与梯度）：

```math
\mathcal{L}_{\text{KL}} = \frac{1}{|\mathcal{V}|} \sum_{t \in \mathcal{V}} \text{KL}\!\Big(\text{softmax}\!\big(z_t^{(T)} / \tau\big) \;\Big\|\; \text{softmax}\!\big(z_t^{(S)} / \tau\big)\Big) \cdot \tau^2
```

**基础训练目标**：

```math
\mathcal{L}_{\text{base}} = \mathcal{L}_{\text{CE}}(z^{(S)}, y) + \alpha_{\text{KL}} \cdot \mathcal{L}_{\text{KL}}
```

#### 表示不稳定性的两大解决方案

信息不对称导致 Student 的损失曲面在关键词缺失输入附近变得更尖锐，论文针对两类失效模式各设计了一种正则化技术：

**1. R-Drop（预测一致性正则化）**
两次独立 dropout 前向传播，最小化输出分布的对称 KL 散度：

```math
\mathcal{L}_{\text{R-Drop}} = \frac{1}{2}\Big[\text{KL}(P_1 \| P_2) + \text{KL}(P_2 \| P_1)\Big]
```

**2. FGM（输入对抗扰动）**
沿梯度方向扰动 Embedding 层，平滑输入空间的损失曲面：

```math
r_{\text{adv}} = \epsilon \cdot \frac{\nabla_{e} \mathcal{L}_{\text{base}}}{\|\nabla_{e} \mathcal{L}_{\text{base}}\|_2}
```

**总优化目标**（含 Focal Loss 缓解 SID 词表长尾不平衡）：

```math
\mathcal{L}_{\text{SDFT}} = \mathcal{L}_{\text{CE}} + \alpha_{\text{KL}} \cdot \mathcal{L}_{\text{KL}} + \alpha_{\text{R}} \cdot \mathcal{L}_{\text{R-Drop}} + \mathcal{L}_{\text{adv}}
```

**超参数设置**：$`\tau=1.0`$，$`\alpha_{\text{KL}}=0.1`$，$`\alpha_{\text{R}}=0.5`$，$`\epsilon=0.6`$，Focal Loss $`\alpha=2`$，$`\gamma=3`$

#### 自蒸馏的关键发现

Self-Distill(S)（无关键词推理）**超越** Base(T)（有关键词推理）：

| 方法 | Order HR@10 | Order MRR@10 | Click HR@10 | Click MRR@10 |
|------|------------|-------------|------------|-------------|
| Base (S)（无关键词） | 0.2094 | 0.1008 | 0.2266 | 0.0731 |
| Base (T)（有关键词） | 0.2139 | 0.1011 | 0.2327 | 0.0743 |
| Self-Distill (T) | 0.2155 | 0.1015 | 0.2397 | 0.0756 |
| **Self-Distill (S)** | **0.2163** | **0.1017** | **0.2398** | **0.0757** |

**解释**：Self-mode 蒸馏中 Teacher 和 Student 共享参数，梯度完全由 Student 损失驱动（含 KL 约束），优化方向偏向信息缺失条件下的鲁棒性，使模型泛化能力超越了关键词增强的 Teacher。

### 四、行为反馈偏好对齐（Behavior Feedback Preference Alignment）

#### 设计动机

V1 依赖分离训练的奖励模型，存在采样偏差和 reward hacking 问题。电商搜索场景的特殊性：
- 多物品同时展示，用户交互具有层级性（点击→加购→购单）
- 必须联合优化 query-item 相关性和转化率
- SID 序列（L=5 tokens）具有严格层次语义（粗粒度→细粒度），不同位置贡献不同

#### 复合奖励设计

三类互补奖励信号：

**相关性奖励（$`R_{\mathrm{Rel}}`$）**：利用现有相关性系统将生成物品分为 4 档（3-优秀/2-相关/1-不匹配/0-无关）

**后验转化奖励（$`R_{\mathrm{CTR}}`$）**：使用 V1 的校准后验 CTR 作为稠密反馈，截断到 (0,1) 范围防止高 CTR 低相关物品主导

**点击与购单奖励（$`R_{\mathrm{C\&O}}`$）**：

```math
R_{\mathrm{C\&O}}(o_i) =
\begin{cases}
    V_o, & \text{if } o_i \in \mathcal{S}_{\mathrm{order}}, \\
    V_c, & \text{if } o_i \in \mathcal{S}_{\mathrm{click}} - \mathcal{S}_{\mathrm{order}}, \\
    0, & \text{otherwise}
\end{cases}
```

其中 $`V_o=3`$，$`V_c=4`$（购单 < 点击，因购单信号更强但需平衡稀疏性）

**最终复合奖励**：

```math
R_{\mathrm{item}}(o_i) = R_{\mathrm{C\&O}}(o_i) + R_{\mathrm{CTR}}(o_i) + R_{\mathrm{FR}}(o_i)
```

#### Token-Position Marginal Advantage（TPMA）

**动机**：标准 GRPO 对序列内所有 token 分配相同 advantage，忽略 SID 的层次因果结构（前几位编码粗粒度类目，后几位编码细粒度属性）。

**前缀奖励（Prefix Reward）**：位置 $`l`$ 处的最大累积匹配分：

```math
R_{i,l} = \max_{t \in \mathcal{T}} \sum_{k=1}^{l} [o_i^k = t^k] \cdot \Delta R_{i,l}
```

**边际贡献权重**（前两位权重更高，体现共享层次特征的重要性）：

```math
\Delta R_{i,l} = [l < 3] \cdot 2 + [3 \le l < L] \cdot 1
```

**位置级 Advantage**（独立归一化，每个位置只与同位置的其他 rollout 比较）：

```math
\hat{A}_{i,l} = \frac{\Delta R_{i,l} - \mathrm{mean}_{j \in [G]}(\Delta R_{j,l})}{\mathrm{std}_{j \in [G]}(\Delta R_{j,l}) + \delta}
```

**前缀门控（Prefix Gate）**（前缀错误时抑制后续 token 的梯度，实现层次课程学习）：

```math
g_{i,l} = [l=1] \cdot 1 + [l \geq 2] \cdot \frac{R_{i,l-1}}{l - 1} \in [0, 1]
```

**联合 Advantage**（结合位置级和物品级奖励）：

```math
\hat{A}_{i,l}^{\mathrm{final}} = \hat{A}_{i,l} + w_{\mathrm{item}} \cdot \hat{A}_i^{\mathrm{item}}
```

**TPMA-GRPO 损失**（省略 ratio clipping，由前缀门控自然提供正则化）：

```math
\mathcal{L}_{\mathrm{TPMA}} = -\frac{1}{G}\sum_{i=1}^{G} \frac{1}{L}\sum_{l=1}^{L} g_{i,l} \cdot r_{i,l} \cdot \hat{A}_{i,l}^{\mathrm{final}}
```

> 前缀门控与 OneRec-V2 的 GBPO 精神相似，但通过灵活的结构性门控而非显式截断实现稳定性。

**在线部署策略**：先用 Listwise DPO 从真实搜索日志学习基础用户交互偏好，再用 TPMA-GRPO 平衡多奖励目标和泛化能力。

## 实验结果

### 离线性能（快手商城搜索平台工业数据集）

**测试集**：30,000 个 PV（含 30,000 条点击行为 + 7,229 条购单行为），每个 PV 取 top-10 生成物品

| 方法 | Order HR@10 | Order MRR@10 | Click HR@10 | Click MRR@10 |
|------|------------|-------------|------------|-------------|
| OneSearch（基准） | 0.2046 | 0.0985 | 0.2231 | 0.0728 |
| ↑+CoT 任务 | 0.2094 | 0.1008 | 0.2266 | 0.0731 |
| ↑+自蒸馏 | 0.2163 | 0.1017 | 0.2398 | 0.0757 |
| ↑+R-Drop | 0.2168 | 0.1045 | 0.2398 | 0.0760 |
| ↑+FGM | 0.2180 | 0.1047 | 0.2422 | 0.0766 |
| ↑+Focal Loss | 0.2214 | 0.1048 | 0.2471 | 0.0788 |
| +PARS（V1奖励系统） | 0.2221 | 0.1067 | **0.2538** | 0.0809 |
| +GRPO | 0.2248 | 0.1106 | 0.2481 | 0.0798 |
| +TPMA | 0.2265 | 0.1136 | 0.2498 | 0.0815 |
| **OneSearch-V2** | **0.2314** | **0.1151** | **0.2568** | **0.0833** |

**有效 SID 率**：OneSearch-V2 达到 99.00%（点击）/ 99.20%（购单），几乎每项优化都有贡献。

### 消融实验

**自蒸馏各组件消融**（单独 vs 叠加）：

| 方法 | Order HR@10 | Click HR@10 |
|------|------------|------------|
| Baseline | 0.2046 | 0.2231 |
| +R-Drop alone | 0.2124 | 0.2292 |
| +FGM alone | 0.2109 | 0.2279 |
| +Focal Loss alone | 0.2074 | 0.2237 |
| Self-Distill | 0.2163 | 0.2398 |
| Self-Distill+R-Drop | 0.2168 | 0.2398 |
| Self-Distill+FGM | 0.2168 | 0.2380 |
| Self-Distill+Focal Loss | 0.2161 | 0.2385 |

> 三种正则化技术叠加产生协同效应（22.14% order HR@10），超过各自贡献之和，因为它们分别针对不稳定性的不同维度（输入表示/输出预测/类别分布）。

**与替代推理内化策略的对比**：

| 方法 | Order HR@10 | Click HR@10 |
|------|------------|------------|
| Baseline | 0.2094 | 0.2266 |
| (i) Special-token 蒸馏 | 0.2092 | 0.2335 |
| (ii) CODI 风格隐层对齐 | 0.2105 | 0.2269 |
| (iii) EMA-mode | 0.2097 | 0.2317 |
| (iv) Joint-mode | 0.2156 | 0.2348 |
| **Self-mode（本文）** | **0.2163** | **0.2398** |

> CODI 劣势的两个原因：(1) 监督粒度不足（仅对单个蒸馏 token 的隐层做 L1 对齐，其余 SID 位置无推理指导）；(2) L1 与 KL 损失不兼容（分别约束隐层几何和输出分布，两者竞争）。

**头部/尾部查询消融**：

| 方法 | Head HR@10 | Tail HR@10 |
|------|-----------|-----------|
| Baseline | 0.2362 | 0.1952 |
| +CoT 任务 | 0.2419 | 0.1963 |
| +direct CoT | 0.1116（**严重下降**） | 0.0809 |
| +RAG（关键词注入） | **0.2438** | **0.1973** |

## 核心贡献总结

| 创新点 | 解决的问题 | 核心机制 | 代价 |
|--------|-----------|---------|------|
| **KHQE 单模态编码** | 多模态表示差异导致核心属性被淹没 | Qwen-VL 关键词提取 + RQ-OPQ 量化 | 无 |
| **三步 Keyword-based CoT** | 复杂查询语义理解不足 | 查询分析→关键词提取→偏好校准；异步生成缓存复用 | 无在线延迟 |
| **推理内化自蒸馏** | CoT 推理在线延迟不可接受 | 共享权重的信息不对称自蒸馏（Teacher 含关键词/Student 不含）；KL 蒸馏+R-Drop+FGM+Focal Loss | 无额外参数/token |
| **TPMA-GRPO** | 奖励模型偏差 + 均匀 advantage 忽视 SID 层次结构 | 位置边际 advantage + 前缀门控 + 复合行为反馈奖励（直接替代奖励模型） | 无额外奖励模型 |

## 与 OneSearch-V1 的关键设计差异速查

| 维度 | OneSearch-V1 | OneSearch-V2 |
|------|-------------|-------------|
| 复杂查询处理 | 类目级监督（SFT Stage 1） | 三步 Keyword-based CoT + 四类 CoT 任务 |
| 推理能力 | 无 | 推理内化自蒸馏（无推理 token 开销） |
| 偏好对齐 | 分离训练的奖励模型 + PARS | 直接行为反馈（点击/购单/CTR/相关性）|
| RL 算法 | GRPO（序列级 advantage） | TPMA-GRPO（位置级边际 advantage + 前缀门控） |
| 信息泡沫/长尾偏差 | 存在（受历史 log 约束） | 显著缓解（CoT + 超越 log 的推理） |
| 推理延迟 | 基准 | **零额外延迟** |
