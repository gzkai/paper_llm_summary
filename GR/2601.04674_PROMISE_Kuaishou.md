## 🎯 研究背景与问题

### 生成式推荐范式
生成式推荐将推荐任务转化为序列到序列的生成任务，直接生成下一个物品的 **Semantic ID（语义ID）**。Semantic ID 由 RQ-VAE 或残差 K-means 等层次化量化方法生成的离散 token 序列，具有从粗到细的层次结构（如：第1层→品类，第2层→子类，第3层→具体商品）。

### 核心问题：语义漂移（Semantic Drift）
作者定义了生成式推荐中一个关键但此前被忽视的问题——**语义漂移**：

- **根本原因**：训练与推理之间的 **Exposure Bias（曝光偏差）**
  - 训练阶段：使用 Teacher Forcing，模型根据 ground-truth 历史预测下一个 token
  - 推理阶段：模型自回归生成，依赖自身历史预测（含错误）
- **漂移机制**：早期 token 编码粗粒度语义（如"电子产品"），一旦出错，后续所有精细预测均在错误子空间内进行，无法恢复
- **加剧效应**：模型遇到分布外（OOD）状态时倾向于回归训练数据边缘分布，加剧流行度偏差，使推荐趋向热门商品

### 与 LLM 推理的类比
| LLM 推理 | 生成式推荐 |
|----------|-----------|
| 思维链（CoT）推理步骤 | Semantic ID 层次化生成步骤 |
| 数学推导中的错误步骤 | 早期 token 预测错误 |
| Process Reward Model（PRM） | **PROMISE 中的 Path-level PRM** |

## 🔧 核心方法：PROMISE 框架（详细）

### 总体架构
PROMISE 基于编码器-解码器架构：
- **编码器**：多层双向自注意力，将用户历史序列 `[x₁, ..., x_{t-1}]` 映射为隐藏状态序列 `E^(L)`
- **解码器**：自回归生成 Semantic ID 路径，通过 NTP Loss 训练
- **Path-level PRM**：轻量级过程奖励模型，与主模型端到端联合训练

### 1. Path-Level PRM（核心创新）

#### 正样本构建
目标物品 `x_t` 对应 d 层 Semantic ID 路径 `[s_{t,1}, ..., s_{t,d}]`。在第 b 层的正样本为前缀路径：
```
S^pos_{x_t, b} = [s_{t,1}, ..., s_{t,b}]，其中 b ≤ d
```

#### 负样本采样策略
关键创新：PRM 从**有效路径集合** `V_b`（对应真实物品的路径）中均匀采样 N 条路径作为负样本（排除正样本），构成负样本集 `N_{x_t, b}`。
- **有效路径**：编码空间远大于物品数量，只有映射到真实物品的路径才是有效路径
- **价值**：让 PRM 同时接触相关和无关的 user-SID 组合，学会辨别

#### 训练目标：InfoNCE Loss
在第 b 层，最大化正样本路径的 logit 相对于负样本的 logit：
```
L^InfoNCE_{x_t, b} = -log[ exp(y_pos) / (exp(y_pos) + Σ exp(y_neg)) ]
```
对所有层求和，与 NTP Loss 联合训练：
```
L^Total = L^NTP + L^InfoNCE
```

#### 轻量级 PRM 架构（工业部署友好）
PRM 的映射函数 `F: (u, c, [s₁, ..., s_b]) → y ∈ ℝ`：

1. **路径嵌入**：将 SID 路径映射为单个向量（而非逐 token 处理，确保效率）
   ```
   P_S = EmbLookup([s₁, ..., s_b])
   ```

2. **Path-level 跨注意力**（F 层堆叠）：
   - Query：路径表示 `P_S`（目标侧）
   - Key/Value：编码器输出 `E^(L)`（用户侧）—— **复用编码器特征，减少参数量**
   ```
   P_S^(i)' = P_S^(i) + CrossAttn(P_S^(i-1), E^(L), E^(L))
   P_S^(i)  = P_S^(i)' + FFN(RMSNorm(P_S^(i)'))
   ```

3. **MLP 输出**：将最终路径表示映射为标量奖励分数
   ```
   y_S = MLP(P_S^(F))
   ```

4. **GPU 并行化**：所有候选路径并行通过跨注意力计算，支持大规模候选集评分

### 2. PRM 引导的 Beam Search（推理策略）

#### 标准 Beam Search 的问题
标准 beam search 仅根据 NTP 概率保留 top-K 路径，无法识别中间步骤的语义漂移。

#### PROMISE 的增强 Beam Search（Algorithm 1）
```
初始化 beam B^(0) = {[BOS]}
对每一步 b = 1 到 d：
  1. 对 beam 中每条路径，解码器生成 top-K token
  2. 扩展所有候选，得到 K × |B^(b-1)| 条路径 C
  3. 按 NTP 分数选 top-K+ 条路径（K+ >> K）
  4. Path-level PRM 并行评估所有 K+ 条路径
  5. 按 PRM 分数选 top-K' 条路径（K' << K+，通常 K' = K）
  6. 更新 beam B^(b)
返回 B^(d)
```

#### 关键参数含义
| 参数 | 含义 |
|------|------|
| K | 原始 beam size，决定解码器计算量 |
| K+ | PRM 评估的候选数，Test-Time Scaling 的调节旋钮 |
| K' | PRM 筛选后保留的路径数，通常 K' = K |

**计算开销分析**：
- 解码器计算量：K' = K，**与原来相同**
- 增加的开销：轻量级跨注意力（PRM），可忽略不计
- 通过增大 K+，即可提升推荐质量，而无需改变解码器开销

### 3. 测试时扩展规律（Test-Time Scaling Laws）
通过增大 K+（推理计算量），可以使**小模型超越大模型**：
- 固定模型参数 + 增大 K+ → 比增大模型参数更高效
- 相同 FLOPs 下，测试时扩展优于参数扩展

## 📊 实验结果

### RQ1：公开数据集（Amazon Review）

| 方法 | Sports-Recall@5 | Sports-NDCG@5 | Beauty-Recall@5 | Beauty-NDCG@5 |
|------|----------------|---------------|----------------|---------------|
| TIGER | 0.0264 | 0.0181 | 0.0454 | 0.0321 |
| ActionPiece（最强基线） | 0.0316 | 0.0205 | 0.0511 | 0.0340 |
| **PROMISE** | **0.0450** | **0.0296** | **0.0536** | **0.0345** |
| 提升 | **+42.41%** | **+44.39%** | **+4.90%** | **+1.47%** |

### RQ1：工业数据集（快手，亿级数据）

| 方法 | Recall@100 | NDCG@100 | Recall@500 |
|------|-----------|---------|-----------|
| GRank（最强基线） | 0.1010 | 0.00472 | 0.2396 |
| PROMISE (K+=4000) | 0.1494 (+47.92%) | 0.00652 (+38.14%) | 0.2836 (+18.36%) |
| PROMISE (K+=6000) | 0.1609 (+59.31%) | 0.00663 (+40.47%) | 0.3017 (+25.92%) |

### RQ2：在线 A/B 测试（快手 & 快手极速版，各约2000万用户，7天）

| 应用 | 总使用时长 | 人均使用时长 | 视频总观看时长 |
|------|-----------|------------|--------------|
| 快手 | +0.121% | +0.120% | +0.431% |
| 快手极速版 | +0.131% | +0.160% | +0.398% |

### RQ3 & RQ4：消融实验（HRecall 指标）

逐步在每层 Semantic ID 生成时应用 PRM：
- 每增加一层 PRM 过滤，对应层及之后层的 HRecall 均提升
- 全层应用（配置8）效果最佳：HRecall@3@1000 从 0.2296 提升至 0.3358（+46.3%）

### RQ5：测试时扩展规律验证
- K+ 从 1000 增至 6000，HRecall@3@1000 从 22.98% → 36.37%
- 暴力增大 K（标准 beam size）效果远不如增大 K+（解码器计算大幅增加但收益小）

### RQ6：测试时扩展 vs. 参数扩展
- 相同 FLOPs 下，测试时扩展性能更优
- 实现相同性能，测试时扩展所需 FLOPs 更少

## 🚀 系统部署

部署版本（K+=4000）的优化措施：
1. **单层 PRM Block**：保持轻量（PRM 注意力头数为主模块的 1/4）
2. **并行跨注意力**：所有查询在 GPU 上并行处理，最大化 GPU 利用率
3. **Radix Top-K 优化**：加速大候选集下的 top-k 选择，抵消 K+ 扩大带来的开销

**最终指标**：
- 参数量：仅增加 **15%**（相比无 PRM 版本）
- 推理延迟：仅增加 **10%**
- 在线收益：视频观看时长提升约 **0.4%**（覆盖数亿用户）

## 💡 总结

PROMISE 的核心贡献：
1. **首次定义**生成式推荐中的语义漂移问题，并建立与 LLM CoT 推理的类比
2. **首次将 PRM** 应用于生成式推荐，通过密集的逐步反馈对齐中间生成步骤
3. **解锁测试时扩展规律**：K+ 是推荐系统中的新扩展维度，比参数扩展更高效
4. **工业级验证**：快手亿级平台验证，在线 A/B 测试正向显著

> 这篇论文将 LLM 领域的 PRM + Test-Time Scaling 思想完整迁移到生成式推荐，是该领域的重要进展。
