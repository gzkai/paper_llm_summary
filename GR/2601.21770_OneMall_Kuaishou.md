## 研究背景与动机

### 从 LLM 成功经验到推荐系统

LLM 的成功源于三大支柱：
1. **预训练**：Next-Token Prediction 统一任务范式，支持万亿 Token 数据和千亿参数规模
2. **后训练**：RL 对齐人类偏好（PPO/DPO/GRPO/GSPO 系列）
3. **基础设施**：Transformer 的 GPU 友好架构，支持 ZeRO/张量并行/流水线并行等技术

### 推荐系统现有范式的局限

工业推荐系统分为两个阶段：

| 阶段 | 目标 | 典型方法 | 局限 |
|------|------|---------|------|
| **召回（检索）** | 从海量物品池中检索候选集（多目标"下一物品预测"） | 双塔、TIGER 等 | 与 LLM 预训练范式对齐度高，但多样性建模复杂 |
| **排序（精排）** | 精确估计 CTR/CVR 等行为概率 | DIN/SIM/TWIN、MMoE 等 | 计算不均衡、模块耦合复杂，难以利用 LLM 并行技术 |

### 电商场景的特殊挑战

相比娱乐短视频（OneRec 的场景），电商服务面临三大额外挑战：

1. **多品类物品**：商品卡（Product-Card）、短视频（Short-Video）、直播（Live-Streaming）具有不同的语义和推荐逻辑
2. **正反馈极度稀疏**：用户决策成本高，从曝光到购买的漏斗极长
3. **单一核心指标**：电商只关注 GMV，对模型能力要求更高

## OneMall 框架总览

OneMall 对齐 LLM 的预训练/后训练范式，包含三大核心组件：

```
OneMall
│
├── 1. E-commerce Semantic Tokenizer（电商语义分词器）
│   ├── LLM 微调（商业关系 + 观看体验）
│   └── ResKmeansFSQ 量化（残差 K-means + 有限标量量化）
│
├── 2. Transformer-based Architecture（纯 Transformer 架构）
│   ├── Query-Former：长序列压缩
│   ├── Cross-Attention：多行为序列融合
│   └── Decoder-Style Sparse MoE：自回归生成
│
└── 3. Reinforcement Learning Pipeline（强化学习管道）
    ├── 排序模型作为奖励模型
    └── DPO / GRPO 策略优化
```

## 核心组件详解

### 一、电商语义分词器（E-commerce Semantic Tokenizer）

#### 问题：三类物品的语义需求不同

| 物品类型 | 语义需求 | 特殊挑战 |
|---------|---------|---------|
| 商品卡（Product-Card） | 商业语义（Commercial） | 仅在购物 Tab 曝光，纯购买意图 |
| 短视频（Short-Video） | 商业 + 观看体验（Commercial & Watching） | 同时在购物 Tab 和娱乐 Tab 曝光 |
| 直播（Live-Streaming） | 商业 + 观看 + 动态更新（Commercial & Watching & Dynamic） | 在售商品实时变化，语义需动态更新 |

#### Item2Item 数据构建

**商品卡内在商业关系数据**：
- 来源：双塔检索模型物品嵌入空间中的最近邻对，或 Swing Item2Item 算法
- 过滤：每个物品最多出现 40 次（避免曝光偏差）
- 规模：**7000 万**样本

**短视频观看关系数据**：
- 来源：用户正向点击商品后，选取间隔 1~5 分钟内长时观看的娱乐短视频构成正对 (short-video ID, product-card ID)
- 过滤：剔除新闻/喜剧/舞蹈/影视/自拍类娱乐短视频，下采样高频商品和同用户对
- 规模：**1200 万**样本

#### 语义压缩骨干网络

- **视觉编码器**：Swin-Transformer（224×224 图像）
- **文本编码器**：Qwen2.5 1.5B
- **训练目标**：InfoNCE 对比损失，特殊 Token `<EMB>` 的最后隐藏状态作为最终表示
- 冻结 ViT，微调 Projector 和 LLM

#### ResKmeansFSQ 量化管道

**残差 K-means（前两层）**：

``` math
\mathbf{C}^1 = \texttt{Kmeans}(\mathbf{M}, K), \quad \mathbf{M}^1 = \mathbf{M} - \texttt{NearestRep}(\mathbf{M}, \mathbf{C}^1)
```

``` math
\mathbf{C}^2 = \texttt{Kmeans}(\mathbf{M}^1, K), \quad \ldots
```

**有限标量量化 FSQ（最后一层）**：
- 训练一个 16-bit 二值 MLP，将残差嵌入量化为 4096 维码字
- 固定聚类中心，保证均匀分布，**显著降低码冲突率**

**各物品类型的量化策略**：
- 商品卡：仅使用 LLM 生成的商品卡嵌入
- 短视频：拼接 LLM 生成的商品卡嵌入 + 短视频嵌入
- 直播：使用双塔检索模型的物品塔嵌入（低学习率以减少语义漂移），实时在售商品 Semantic IDs 作为附加特征

### 二、Decoder-Only 模型架构

#### Query-Former：长序列压缩

电商用户行为序列种类多（曝光/点击/购买等），直接拼接序列过长。Q-Former 将每类序列压缩为 $M=10$ 个连续表示：

``` math
\mathbf{F}_{\texttt{click}} = \texttt{QFormer}_{\texttt{click}}(\mathbf{Q}_{\texttt{click}}, \mathbf{H}_{\texttt{click}}, \mathbf{H}_{\texttt{click}})
```

其中 $`\mathbf{H}_{\texttt{click}} \in \mathbb{R}^{500 \times 128}`$ 压缩为 $`\mathbf{F}_{\texttt{click}} \in \mathbb{R}^{10 \times 128}`$，计算量从 34.4 GFLOPs 降至 9.2 GFLOPs，性能损失极小（HR@50 仅 -0.5%）。

#### Cross-Attention：多行为序列融合

将压缩后的多行为序列 $`\{\mathbf{F}_{\texttt{click}}, \mathbf{F}_{\texttt{exposure}}, \mathbf{F}_{\texttt{buy}}, \ldots\}`$ 通过 Cross-Attention 注入目标 Semantic ID 的生成：

``` math
\{\hat{\mathbf{s}}_0^L, \hat{\mathbf{s}}_1^L, \hat{\mathbf{s}}_2^L, \hat{\mathbf{s}}_3^L\} = \texttt{Cross-Att}^L(\{\mathbf{s}_i^{L-1}\} \cdot \mathbf{W}_q^L, \{\mathbf{F}_*\} \cdot \mathbf{W}_k^L, \{\mathbf{F}_*\} \cdot \mathbf{W}_v^L)
```

#### Decoder-Style Sparse MoE：自回归生成

先做因果自注意力保持自回归约束，再过稀疏 MoE 扩展模型容量：

``` math
\{\bar{\mathbf{s}}_i^L\} = \texttt{Causal-Self-Att}(\{\hat{\mathbf{s}}_i^L\})
```

``` math
\{\mathbf{s}_i^L\} = \texttt{Sparse-MoE}(\{\bar{\mathbf{s}}_i^L\}) + \{\mathbf{s}_i^{L-1}\}
```

使用**无损失负载均衡机制**约束专家负载。

#### 监督目标

``` math
\mathcal{L}_{\texttt{NTP}} = \texttt{Softmax}(\{\mathbf{s}_0^L, \mathbf{s}_1^L, \mathbf{s}_2^L\}, \{s_1, s_2, s_3\})
```

``` math
\mathcal{L}_{\texttt{contrastive}} = \texttt{In-batch-contrastive}(\mathbf{s}_3^L, \mathbf{f}_{\texttt{item}})
```

- NTP 损失：主监督信号，自回归预测三层 Semantic ID
- 辅助对比损失：以 $\mathbf{s}_3^L$（已编码完整 Semantic ID 序列）与物品塔嵌入做 in-batch 对比，准确率超 98%（@1）

### 三、强化学习管道（RL Pipeline）

#### 设计思路

以**在线排序模型**作为奖励模型，突破传统召回-排序串联管道的隔离壁垒：

``` math
r = \alpha \cdot \hat{y}_{\texttt{ctr}} + \beta \cdot \hat{y}_{\texttt{ctcvr}} + \gamma \cdot \hat{y}_{\texttt{egpm}} + \ldots
```

（$`\alpha=1.0, \beta=30.0, \gamma=1.0`$，保持各目标量级一致）

#### RL 流程

1. **策略模型** $\pi_\theta$：流式训练中的 NTP 模型
2. **参考模型** $\pi_{\theta_{ref}}$：定期从策略模型同步参数，用于生成候选响应 $\{o_1, \ldots, o_n\}$
3. **奖励模型**：排序模型对候选响应打分，归一化为优势分数 $A_i = \frac{r_i - \text{mean}(r)}{\text{std}(r)}$
4. **优化策略**：DPO 或 GRPO

#### DPO 实现

``` math
\mathcal{L}_{\texttt{DPO}} = -\sigma\left(\lambda \log \frac{\pi_\theta(o_{pos}|q)}{\pi_{\theta_{ref}}(o_{pos}|q) + \delta} - \lambda \log \frac{\pi_\theta(o_{neg}|q)}{\pi_{\theta_{ref}}(o_{neg}|q) + \delta}\right)
```

#### GRPO 实现

``` math
\mathcal{L}_{\texttt{GRPO}} = -\sum_i^m \min\left(\texttt{clip}\left(\frac{\pi_\theta(o_i|q)}{\pi_{\theta_{ref}}(o_i|q) + \delta}, 1-\epsilon, 1+\epsilon\right) \times A_i\right)
```

#### 联合训练目标

``` math
\mathcal{L} = 0.5 \cdot \mathcal{L}_{\texttt{RL}} + \mathcal{L}_{\texttt{NTP}} + \mathcal{L}_{\texttt{contrastive}}
```

- 采用 2% 的训练流量做 RL 训练，beam size = 768
- RL 损失权重不宜过大，否则会损害 SID 准确率和候选有效率

## 实验结果

### 参数规模扩展实验（短视频场景）

| 参数量 | 层数 | Acc-SID1 | Acc-SID2 | Acc-SID3 | HR@50 | HR@100 | HR@500 |
|--------|------|----------|----------|----------|-------|--------|--------|
| 0.05B | 2 | 14.5% | 43.9% | 61.0% | 32.9% | 41.3% | 60.5% |
| 0.1B | 4 | 14.9% | 46.2% | 63.6% | 34.7% | 43.8% | 63.2% |
| 0.5B-A0.1B（MoE） | 4 | 16.0% | 50.6% | 69.3% | 44.7% | 56.0% | 74.2% |
| 1.3B-A0.1B（MoE） | 4 | 16.2% | 51.7% | 71.7% | 45.6% | 57.3% | 76.0% |

> 从 0.1B 稠密到 0.5B-A0.1B 稀疏 MoE，HR@50 提升最显著（+10%），说明 MoE 在保持推理效率的同时有效扩展了模型容量。

### RL 消融实验（短视频场景）

| Beam Top | 策略 | CTR | CTCVR | GPM | Reward |
|----------|------|-----|-------|-----|--------|
| Top10 | Base | 7.816% | 0.243% | 0.0983 | 0.2493 |
| Top10 | DPO | 7.834% | 0.254% | 0.0997 | 0.2543 |
| Top10 | **GRPO** | **7.874%** | **0.266%** | **0.1020** | **0.2607** |
| Top100 | Base | 4.369% | 0.106% | 0.0390 | 0.1147 |

> GRPO 优于 DPO 的原因：GRPO 对 768 个候选归一化计算优势分数，学习完整质量分布；DPO 仅用正负样本对，反馈信息量更少。

### 组件消融实验

**Tokenizer 策略（码冲突率）**：

| 方案 | 冲突率 | 独占率 | HR@50 | HR@100 | HR@500 |
|------|--------|--------|-------|--------|--------|
| ResKmeans | 36% | 86% | 33.9% | 43.0% | 62.6% |
| ResKmeansFSQ | 11% | 95% | 35.4% | 45.3% | 66.7% |
| ResKmeansFSQ + 辅助对比损失 | 11% | 95% | **+1.5%** | **+1.7%** | **+1.7%** |

**Query-Former 压缩效果**：

| 方案 | 序列长度 | GFLOPs | HR@50 | HR@100 | HR@500 |
|------|---------|--------|-------|--------|--------|
| QFormer | 160 | 9.2 | 44.7% | 56.0% | 74.2% |
| w/o QFormer | 1205 | 34.4 | +0.5% | +0.3% | +0.4% |

> Q-Former 将计算量降低约 73%，性能损失仅 0.3~0.5%，工程效益极高。

## 核心贡献总结

1. **统一多品类电商的生成式推荐框架**：首个将 LLM 预训练/后训练范式完整对齐到电商推荐系统的工作，覆盖商品卡、短视频、直播三大场景。

2. **电商专用语义分词器（ResKmeansFSQ）**：
   - 通过 LLM 微调注入商业关系和观看体验语义
   - FSQ 最后一层显著降低码冲突率（36% → 11%），提升独占率（86% → 95%）
   - 直播场景采用动态语义 ID 策略应对实时变化

3. **高效 Transformer 架构**：
   - Q-Former 压缩长序列，计算量降低 73%，性能损失极小
   - Sparse MoE 在激活参数固定（0.1B）的前提下将总参数扩展至 1.3B，大幅提升模型容量

4. **检索-排序端到端 RL 优化**：
   - 以在线排序模型作为奖励信号，打破传统串联管道的隔离
   - GRPO 在 Top10 候选上 Reward 提升 4.6%（相对 Base），全面优于 DPO

5. **大规模工业部署验证**：服务 4 亿日活用户，三大电商场景 GMV 全面提升（+14.7% / +10.3% / +4.9%）。
