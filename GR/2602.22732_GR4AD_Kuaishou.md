## 二、背景与动机

工业广告推荐系统传统上依赖 DLRM（深度学习推荐模型）构建多阶段流水线（检索→预排序→精排）。近年来生成式推荐（Generative Recommendation）因其 Scaling 潜力和更强的模型容量，开始受到工业界关注。

然而，将 **实时生成式推荐** 直接套用在大规模广告系统中面临三大核心挑战：

1. **广告 Tokenization 困难**：广告内容融合视频属性、商品信息、B2B 广告主元数据，还有商业信号（如转化类型），难以用通用 LLM Embedding 直接表征。
2. **学习范式不匹配**：广告推荐要优化列表级商业目标（eCPM、NDCG），而非单条样本预测；需要在线持续更新，LLM 式的离线训练不够适配。
3. **实时 Serving 约束严苛**：需在高流量下低延迟（<100ms）生成多候选，与 LLM 单次生成场景截然不同。

## 三、核心方案

GR4AD 在 **表征、学习、Serving** 三个维度进行了广告原生协同设计。

### 3.1 Unified Advertisement Semantic ID（UA-SID）

类比于 LLM 中的 Token，Semantic ID 是生成式推荐的基础表征单元。GR4AD 提出 **UA-SID**，分两步构建：

#### Step 1：Unified Advertisement Embedding（UAE）

通过对一个 MLLM（多模态大语言模型）进行端到端微调，得到广告专用 Embedding：

**① Instruction Tuning（IT）**
- 针对广告多样化内容格式（视频、商品、广告主）设计 **6 类指令** 进行指令微调
- 例如：直播主播关注地理信息，站外广告主关注品牌行业信息
- 目标：让模型跨多种广告类型具备统一的理解能力

**② Co-occurrence Learning（CL）**
- 利用 **Swing 算法** 估算商品间共现强度
- 对每个锚点商品，频繁共现的商品视为正样本 $\mathcal{P}_i$
- 三元组（视频、商品、广告主）各自扩充正样本
- 用 InfoNCE 对比损失优化：
$$\mathcal{L}_{\mathrm{NCE}}(i) = - \log \frac{\sum_{j \in \mathcal{P}_i} \exp(\mathbf{z}_i^\top \mathbf{z}_j / \tau)}{\sum_{k \neq i} \exp(\mathbf{z}_i^\top \mathbf{z}_k / \tau)}$$

#### Step 2：Multi-Granularity-Multi-Resolution（MGMR）RQ-Kmeans

解决 Codebook 利用率低、SID 碰撞率高两个问题：

**Multi-Resolution（MR）**：
- 借鉴 PLUM 方案，低层级使用更大 Codebook 捕获主要特征，高层级建模低熵残差
- 每层使用平衡 K-means 聚类以提升 Codebook 利用率

**Multi-Granularity（MG）**：
- 广告中存在内容相同但转化类型不同的广告，导致 SID 碰撞严重
- 在最后一层将向量量化替换为 **基于哈希的数值映射**，来自非语义特征（item/account ID、转化类型）
- 几乎无额外量化误差，但大幅改善全局平衡性、降低碰撞率

最终每个广告映射为结构化 SID 序列：
$$\mathbf{y} = (s_1, s_2, \ldots, s_T), \quad s_t \in \mathcal{V}_t$$

### 3.2 Lazy Autoregressive Decoder（LazyAR）

#### 动机

观察到一个矛盾：
- **第一级 SID（$s_1$）** 最难学习（Loss 最大），但在 Beam Search 中消耗计算最少（Beam=1）
- **后续级 SID** 更容易学习，但在 Beam Search 中须对大量候选并行解码，是推理瓶颈

**LazyAR** 通过"延迟自回归依赖"来解决这一矛盾。

#### 方案细节

对于深度为 $L$ 的 Decoder，设超参数 $K$（$1 \le K < L$）：

**前 K 层**：不依赖 $s_{t-1}$，只根据位置编码和上下文并行计算：
$$\mathbf{m}^{(0)}_t = \mathbf{p}_t$$
$$\mathbf{m}^{(\ell)}_t = \mathrm{Dec}^{(\ell)}(\mathbf{m}^{(\ell-1)}_t, \mathbf{X}), \quad \ell=1,\ldots,K$$

**第 K 层注入**：通过轻量级门控投影融合前一级 SID：
$$\tilde{\mathbf{m}}^{(K)}_t = \mathrm{Fuse}(\mathbf{m}^{(K)}_t, \mathbf{s}_{t-1})$$
$$\mathrm{Fuse}(\mathbf{m}, \mathbf{s}) = \mathbf{W}_f[\mathbf{m} \odot (\mathbf{W}_g \mathbf{s}); \mathbf{s}]$$

**后 L-K 层**：正常自回归计算完成解码。

#### 为什么更快？

前 $K$ 层的计算 **不依赖** $s_{t-1}$，可以对所有 SID 层级并行计算一次，并在所有 Beam 候选间复用：
$$\{\mathbf{m}^{(K)}_t\}_{t=1}^{T} = \mathrm{Dec}^{(1:K)}(\{\mathbf{p}_t\}_{t=1}^{T}, \mathbf{X})$$

只有后 $L-K$ 层需要顺序执行，显著降低 Beam Search 的顺序计算量。

#### 为什么不损失性能？

1. 第一级 SID 的解码路径**完全不变**（全 L 层），最难学的部分不受影响
2. 前 K 层的中间状态不是随机的，它们在潜在空间中包含有用的推理信息
3. 引入 **MTP 式辅助损失**：强制 Trunk（前 K 层）在没有自回归信号时也能直接预测目标 token，鼓励其学到更丰富的表征
4. $K$ 是可调超参，提供精度-效率的弹性权衡，不改变模型参数量

**实验设置**：$L=9$，$K=6$（前 2/3 层共享），QPS 提升约 **2 倍**，性能几乎不损失。

#### 与 DeepSeek-MTP 的对比

| 维度 | DeepSeek-MTP | LazyAR |
|------|-------------|--------|
| 额外参数 | 需要新增浅层 Decoder | 无额外参数 |
| 早期层参与推理 | 否 | 是 |
| 推荐性能 | 实验中略有下降 | 基本持平 |
| 适用场景 | LLM 长文本生成 | 推荐短序列多候选生成 |

### 3.3 Value-Aware Supervised Learning（VSL）

广告推荐的核心监督信号需与业务价值对齐，GR4AD 设计了三重机制：

**① 标准 SID 预测损失**：
$$\mathcal{L}_{\mathrm{SID}} = -\sum_{t=1}^{T} \log p(s_t \mid s_{<t}, \mathbf{X})$$

**② eCPM-Aware Token Prediction**：
- 将连续 eCPM 值离散化为等频 Bucket，作为额外预测步骤追加到 SID 序列末尾
- 优化 eCPM Token 预测的交叉熵损失：
$$\mathcal{L}_{\mathrm{eCPM}} = -\log p(v \mid \mathbf{y}, \mathbf{X})$$
- 合并损失：$\mathcal{L}_{\mathrm{NTP}} = \mathcal{L}_{\mathrm{SID}} + \lambda_e \mathcal{L}_{\mathrm{eCPM}}$

**③ Value-Aware Sample Weighting**：
- 每个样本赋权 $w = w_{\mathrm{user}} \cdot w_{\mathrm{behavior}}$
- $w_{\mathrm{user}}$：用户的长期广告价值
- $w_{\mathrm{behavior}}$：用户交互深度（购买行为权重 > 点击行为）

**④ 辅助 MTP 损失**（配合 LazyAR）：
- 强制 Trunk 层在无自回归信号时直接预测目标，丰富中间表征

**最终 VSL 目标**：
$$\mathcal{L}_{\mathrm{VSL}} = \mathbb{E}_{\mathcal{D}} \left[ w \left( \mathcal{L}_{\mathrm{NTP}} + \lambda_{\mathrm{mtp}} \mathcal{L}_{\mathrm{MTP}} \right) \right]$$

### 3.4 Ranking-Guided Reinforcement Learning（RSPO）

VSL 拟合历史数据分布，但不能直接优化列表级排序目标，也不支持对用户偏好的主动探索。GR4AD 在 VSL 之上引入 RL 阶段。

#### RSPO 算法

**设计出发点**：推荐系统要生成 **排序列表** 而非单个输出，因此 Per-item 奖励不够用。

设候选列表 $\mathcal{Y} = \{\mathbf{y}_1, \ldots, \mathbf{y}_n\}$，$v_i$ 为对应 eCPM 奖励，RSPO 直接对 NDCG 目标建立上界进行优化：

$$\mathcal{L}_\text{RSPO} = -\mathbb{E}\left[\log_2 \sigma\left(-\log \sum_{\mathbf{y}_j \in \mathcal{E}_i} \mathcal{M}_{ij} \exp\left(\beta \log \frac{p_\theta(\mathbf{y}_j)}{p_{ref}(\mathbf{y}_j)^{C_{ij}}} - \beta \log \frac{p_\theta(\mathbf{y}_i)}{p_{ref}(\mathbf{y}_i)^{C_{ij}}}\right)\right)\right]$$

- $\mathcal{E}_i = \{\mathbf{y}_j \mid v_j < v_i\}$：排在 $\mathbf{y}_i$ 之下的候选集
- $\mathcal{M}_{ij}$：Lambda 权重，基于 NDCG 位置折扣计算
- $C_{ij}$：参考分布可靠性门控（当 $p_\theta$ 与 $p_{ref}$ 偏离过大时，自动关闭参考约束）

**可证明** $\mathcal{L}_\text{RSPO}$ 是 NDCGcost 的上界（论文附录有完整证明）。

#### VSL 与 RSPO 的统一学习（UVR）

与 LLM 分阶段训练不同，GR4AD 在线学习要求 **VSL 和 RSPO 统一在单个训练流中同时优化**。

引入 **样本级对齐分数** $A^{(i)}$，衡量模型排序与奖励排序的不一致程度：
$$A^{(i)} = \frac{|r_p^{(i)} - r_v^{(i)}|}{n-1}, \quad A^{(i)} \in [0,1]$$

基于 $A^{(i)}$ 自适应调整两个目标的权重：
$$w_{\mathrm{VSL}}^{(i)} = w_0 \cdot \exp(A^{(i)} \cdot \log(1+v_i))$$
$$w_{\mathrm{RL}}^{(i)} = w_0 \cdot Z_{\max}(1 - A^{(i)})$$

- **$A^{(i)}$ 大**（模型排序 vs 奖励排序偏差大）→ 增大 VSL 权重，先学好用户兴趣分布
- **$A^{(i)}$ 小**（已经基本对齐）→ 增大 RSPO 权重，专注优化列表级价值

**最终统一目标**：
$$\mathcal{L} = \mathbb{E}_{i \sim \mathcal{D}} \left[ w_{\mathrm{VSL}}^{(i)} \mathcal{L}_{\mathrm{VSL}}^{(i)} + w_{\mathrm{RL}}^{(i)} \mathcal{L}_{\mathrm{RSPO}}^{(i)} \right]$$

### 3.5 Dynamic Beam Serving（DBS）

#### Dynamic Beam Width（DBW）

**发现**：
- 最终步的 Beam Width 直接决定候选集数量，对 Revenue 影响最大
- 早期步的 Beam Width 对 Revenue 影响较小，但主导解码计算量

**方案**：用 **逐步递增的 Beam 计划** 替代固定 Beam Width
- 例如：将 `512-512-512` 替换为 `128-256-512`
- 降低中间步骤计算量，保持最终候选质量

#### Traffic-Aware Adaptive Beam Search（TABS）

流量存在明显峰谷周期，serving 约束由峰值流量决定：
- **峰值期**：保持 Baseline Beam Width，不超出预算
- **低峰期**：将 Beam Width 提升 60%，利用闲置算力探索更多假设，改善排序质量

自适应控制公式：$B_t = B_{\text{base}} \cdot f(Q_t, \mathcal{C}_{\text{avail}})$

## 四、系统部署

GR4AD 已完整部署在快手广告系统，服务超 **4 亿用户**，闭环架构由四个模块组成：

| 模块 | 职责 |
|------|------|
| **Realtime Serving** | 处理用户请求，返回排序广告列表，并日志记录 |
| **Realtime Index** | UA-SID ↔ Item ID 双向索引，新广告秒级接入，解决冷启动 |
| **Online Learning** | 实时消费请求和交互流，持续执行 VSL + RSPO mini-batch 更新 |
| **Reward System** | 放宽延迟约束探索更多候选，用奖励模型估算 eCPM，产出 RL 日志 |

**其他 Serving 优化**：
- **Beam-Shared KV Caching**：多 Beam 沿 Sequence 维度共享编码器 KV Cache，复杂度从 $\mathcal{O}(B \cdot L)$ 降至 $\mathcal{O}(L)$
- **TopK Pre-Cut**：先各 Beam 并行 Top-K 再全局 Top-K，减少搜索空间，提升 GPU 并行度
- **FP8 精度**：计算精度从 FP32 降至 FP8，降低计算和内存开销
- **Result Cache（短 TTL）**：同一用户短时间内多次请求直接复用缓存结果

## 五、实验结果

### 5.1 总体性能（在线 A/B 测试）

| 模型设置 | vs. DLRM（Base）$\Delta$Revenue | vs. GR-Base $\Delta$QPS |
|---------|-------------------------------|------------------------|
| DLRM（Base） | -- | -- |
| OneRec-V2（GR-Base） | +1.68% | -- |
| + UA-SID | +1.92% | 0% |
| + VSL | +2.80% | -25% |
| + VSL + DPO | +3.16% | -25% |
| + VSL + GRPO | +3.21% | -25% |
| + VSL + RSPO | +3.86% | -25% |
| + UVR（统一 VSL & RSPO） | +4.01% | -25% |
| + UVR + DBS | **+4.32%** | +20% |
| + UVR + DBS + DeepSeek-MTP | +3.98% | **+117%** |
| **GR4AD**（+ UVR + DBS + LazyAR） | **+4.28%** | **+117%** |

> RSPO 是学习优化中贡献最大的单项（vs DPO/GRPO 均更优），LazyAR 以近乎无损的性能换取 QPS 翻倍。

### 5.2 Scaling Laws

**模型 Scaling**（固定 Beam=512）：

| 模型规模 | $\Delta$Revenue |
|---------|----------------|
| 0.03B | +2.13% |
| 0.08B | ~+3.0% |
| 0.16B | ~+4.0%（部署版本） |
| 0.32B | +4.43% |

**推理 Scaling**（固定模型大小，增大 Beam Width）：Revenue 单调递增，QPS 相应下降，两者之间存在平滑 Pareto 曲线，DBS 可动态在曲线上寻优。

### 5.3 业务指标（超越 Revenue）

| 指标 | 提升幅度 |
|------|---------|
| 中小广告主广告投放量 | **+17.5%** |
| 广告转化率（全体用户） | **+10.17%** |
| 广告转化率（低活跃用户） | **+7.28%** |

这些增益来源于：内容 SID 的更强泛化能力 + 实时索引对冷启动广告的更好支持。

## 六、核心贡献总结

1. **UA-SID**：首个端到端微调的广告 MLLM Embedding + MGMR 量化，解决广告 Tokenization 特殊挑战
2. **LazyAR**：推荐专用的延迟自回归解码器，无额外参数、无性能损失，QPS 翻倍
3. **VSL + RSPO + UVR**：广告原生的在线学习框架，eCPM 感知监督 + 列表级 RL + 统一动态权衡
4. **DBS（DBW + TABS）**：动态 Beam Serving，充分利用流量峰谷实现效率-效果最优均衡
5. **完整工业落地**：4 亿+ 用户规模，<100ms 延迟，500+ QPS/L20，广告收入 +4.28%
