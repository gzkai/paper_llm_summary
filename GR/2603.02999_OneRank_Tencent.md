## 核心问题

生成式推荐（GR）在广告场景深入排序阶段时，面临三大关键挑战：

| 挑战 | 描述 |
|------|------|
| **兴趣-价值目标错位** | 基于行为（点击/转化）的 MTP 优化与商业价值（eCPM）目标存在根本冲突：单阶段融合导致优化张力，阶段解耦则在生成阶段系统性过滤高价值候选 |
| **Target-Agnostic 局限** | 生成过程中用户表示静态不变，无法动态感知候选物品特征，限制细粒度用户-物品交互能力 |
| **生成-排序架构断层** | "先生成后排序"引入三重断层：表示异构、计算冗余、误差传播，阻碍系统全局最优 |

---

## OneRanker 整体架构

OneRanker 将生成与排序深度融合为三个逻辑递进的 Step：

```
Step 1（生成）→ Step 2（多任务/目标感知增强）→ Step 3（统一排序）
         ↑_____________KV Pass-through + DC Loss______________↑
```

### Step 1：生成骨干（Generation）

- 延用 GPR 的异构 Token 架构（U/C/X/I Token：用户、内容、上下文、物品）
- 基于 HSTU Decoder-only 架构，自回归 MTP 机制并行生成多条完整语义 ID 路径
- 序列最大长度 2048，G-Decoder 4 层 Transformer

---

### Step 2：多任务 / 目标感知增强（Multi-Task / Target-Aware Enhancement）

#### ① 价值感知多任务解耦架构（Value-Aware Multi-Task Decoupling）

构造可学习任务 Token 序列 $\mathbf{T}=[\mathbf{t}\_{i_1}, \dots, \mathbf{t}\_{i_m}, \mathbf{t}_v]$ :

- $\mathbf{t}\_{i_1}, \dots, \mathbf{t}\_{i_m}$ : **兴趣任务 Token**，捕捉多兴趣分布（6 个）
- $\mathbf{t}_v$ : **价值感知任务 Token**，专门学习业务价值（2 个，Bagging 聚合）

所有 Token 共享底层用户表示，通过独立输出头实现任务空间解耦。引入**任务顺序先验 + 因果掩码**（印象→点击→转化→业务价值），使高级任务动态吸收低级任务知识，实现渐进式精炼。

#### ② Fake Item Token：粗粒度目标感知

- 对全量物品空间做 K-means 聚类，得到 $k=32$ 个聚类中心向量作为 Fake Item Tokens $\mathbf{F} = [\mathbf{f}_1, \dots, \mathbf{f}_k]$
- 构造 Query 序列 $\mathbf{Q} = [\mathbf{T}; \mathbf{F}]$，以 Step 1 输出为 Key/Value 做 Cross-Attention
- 使生成过程动态感知物品语义分布，从静态用户表示进化为动态感知物品分布的表示

#### ③ 异构注意力解码器（Heterogeneous Attention Decoder）

两项关键改进：

**Cross-Attention 优先化（CA-Pri）**：颠倒传统 Transformer Decoder 顺序，先 Cross-Attention（聚合用户多兴趣表示）再 Self-Attention（精炼任务表示），符合"先理解用户意图，再精炼任务表示"的认知逻辑。

**异构掩码策略（H-Mask）**：
- 任务 Token 间：因果掩码（保证时序建模）
- 任务 Token → Fake Item Token：允许访问所有 Fake Item
- Fake Item Token → 任务 Token：允许访问所有任务 Token
- Fake Item Token 间：**互不可见**（避免聚类中心间干扰）

#### ④ 双通道表示构造（Dual-Channel Representation）

最终用户表示融合两个通道：

$$\mathbf{e}_{\text{user}}^{(i)} = \text{Concat}\left(\mathbf{e}_{\text{task}}^{(i)},\, \mathbf{s}_{\text{target}}^{(i)}\right)$$

- **任务语义通道**：Task Token 输出 $\mathbf{e}_{\text{task}}^{(i)}$
- **目标感知通道**：Fake Item Token 与 Task Token 经 MLP 得偏好分 $\mathbf{s}^{(i)}\_{\text{target}} = \sum_{j=1}^{k} \mathbf{s}^{(i)}_j$

物品侧对称增强：

$$\mathbf{e}_{\text{item}}^{\text{enhanced}} = \text{Concat}\left(\mathbf{e}_{\text{item}}, [c_1, \dots, c_k]\right)$$

其中 $c_i = \text{cos-sim}(\mathbf{e}_{\text{item}}, \mathbf{f}_i)$，检索相关性：

$$\text{score}(u, i) = \mathbf{e}_{\text{task}}^{(i)} \mathbf{e}_{\text{item}} + \sum_{l=1}^{k} s^{(i)}_l \cdot c_l$$

内积操作自然融合语义匹配与目标感知匹配。

---

### Step 3：统一排序（Unified Ranking）

#### 细粒度目标感知

排序解码器（R-Decoder）：
- **Query**：排序任务 Token $\mathbf{T}_r$ + $n$ 个候选物品 Token $\mathbf{I}$
- **Key/Value**：Step 1 原始多兴趣表示 ⊕ Step 2 精炼表示（双源融合）
- **掩码**：候选物品 Token 间互不可见（对角掩码），保证评分独立性
- **架构**：同 Step 2 异构注意力解码器，但仅 1 层；输出经轻量 MLP 映射为标量分数

#### 输入侧一致性（Input-Side Consistency）

**Key/Value 透传机制**：
- Step 1 → Step 3：传递用户基础兴趣分布（广度保障）
- Step 2 → Step 3：传递生成过程的隐式决策依据（"为什么生成这批候选"）

#### 输出侧一致性（Output-Side Consistency）

**三元联合损失**：

$$\mathcal{L}_{\text{total}} = \alpha \mathcal{L}_{\text{MTP}} + \beta \mathcal{L}_{\text{rank}} + \gamma \mathcal{L}_{\text{DC}}$$

| 损失 | 公式 | 作用 |
|------|------|------|
| **生成损失** $\mathcal{L}_{\text{MTP}}$ | $-\sum_{j=1}^{m} \sum_{l=1}^{L} \log P(\mathbf{e}\_{\text{item}}^{\text{gt}} \| \mathbf{e}\_{\text{user}}^{(j)})$ | 优化多兴趣路径生成 |
| **排序损失** $\mathcal{L}_{\text{rank}}$ | $-\mathbb{E}_{(i,j)} [\mathbb{I}(y_i > y_j) \cdot \log \sigma(s_i - s_j)]$（BPR） | 优化候选相对排序（eCPM 标签） |
| **分布一致性损失** $\mathcal{L}_{\text{DC}}$ | $-\mathbb{E}\_{i \sim \mathcal{C}} [p_i^{\text{target}} \cdot \log \pi_\theta(i\|\mathbf{u})]$ | 排序器作为教师，对齐生成-排序分布 |

$\mathcal{L}\_{\text{DC}}$ 本质是 RL 目标 $\mathcal{J}(\theta) = \mathbb{E}[s_i] - \lambda \cdot \text{KL}(\pi_\theta \| \pi_{\text{ref}})$ 的监督代理，建立排序→生成的反向梯度通路，使生成器"预判"排序偏好，从根本上纠正价值偏差。

## 实验结果

### 离线对比（RQ1）

数据集：腾讯大规模真实广告+有机内容交互数据，统一异构 Token 架构

| 模型 | HR@1 | HR@5 | HR@15 | NDCG@1 | NDCG@5 | NDCG@15 |
|------|------|------|-------|--------|--------|---------|
| HSTU | 0.1741 | 0.4648 | 0.7953 | 0.6742 | 0.6763 | 0.7396 |
| GPR | 0.1824 | 0.4935 | 0.8207 | 0.6823 | 0.6818 | 0.7445 |
| **OneRanker** | **0.2639** | **0.6213** | **0.8894** | **0.8102** | **0.7904** | **0.8206** |

**OneRanker vs GPR**：HR@1 相对提升 **+44.7%**，NDCG@5：0.6818 → **0.7904**

### 消融实验（RQ2）

| 变体 | HR@5 | NDCG@5 | 说明 |
|------|------|--------|------|
| OneRanker (Full) | **0.6213** | **0.7904** | 完整模型 |
| w/o DC Loss | 0.6173 | 0.7865 | 去掉分布一致性损失 |
| w/o S2 token injection | 0.6161 | 0.7858 | 去掉 Step 2 信息注入 |
| with S3 ranker only | 0.6157 | 0.7849 | 仅 Step 1 信息 |
| OneRanker S2 | 0.5448 | 0.7440 | 去掉 Step 3 |
| S2 w/o Target | 0.5203 | 0.7285 | 去掉 Fake Item Token（-4.5% HR@5） |
| S2 w/o Target & MDA | 0.5066 | 0.7275 | 再去掉多任务解耦架构 |

### 专项设计消融（RQ3）

| 变体 | HR@5 | 说明 |
|------|------|------|
| OneRanker S2 (Baseline) | 0.5448 | 基准 |
| w/o CA-Pri | 0.5277 | 去掉 Cross-Attention 优先化 |
| w/o H-Mask | 0.5335 | 去掉异构掩码策略 |

### DC 损失有效性（RQ4）

- 引入 $\mathcal{L}_{\text{DC}}$ 后，Step 2 与 Step 3 排名差异的 IQR（箱体高度）显著压缩
- Top-K 重叠率曲线在所有 K 值下均更高，验证分布一致性优化的有效性

### 在线 A/B 测试（RQ5）

平台：微信视频号广告系统，数亿活跃用户，千万级动态广告

| 流量阶段 | GMV | GMV-Normal | Costs |
|---------|-----|-----------|-------|
| 5% | +0.41% | **+1.34%** [0.16%, 2.52%] | +0.72% |
| 20% | +0.78% | +0.64% | +1.15% |
| 80% | +0.08% | +0.35% | +0.45% |

已全量部署至 100% 流量，成为微信视频号广告推荐的生产系统。

## 核心贡献总结

| 贡献 | 内容 |
|------|------|
| **价值感知多任务解耦** | 通过任务 Token + 因果掩码，实现兴趣建模与业务价值优化的架构级解耦，消除优化张力 |
| **粗细粒度协同目标感知** | Fake Item Token 实现生成阶段粗粒度感知，排序解码器实现细粒度价值对齐 |
| **双侧一致性保证** | KV 透传（输入侧）+ 三元联合损失（输出侧），实现生成-排序端到端协同优化 |
| **工业落地验证** | GMV-Normal +1.34%，全量部署于腾讯微信视频号广告系统 |

## 与同类工作对比

| 方法 | 生成-排序关系 | 目标感知 | 价值优化 |
|------|------------|---------|---------|
| 传统级联（生成+独立排序） | 阶段解耦 | ❌ | 排序阶段 |
| 单阶段融合（直接注入 eCPM） | 单阶段 | ❌ | 存在优化张力 |
| GRank/PROMISE/GRAM | 软协同 | 粗粒度 | 有限 |
| **OneRanker** | **架构级深度融合** | **粗+细协同** | **闭环优化** |
