## 研究背景与动机

### 核心问题

现有生成式推荐系统（如 TIGER）仅面向语义检索优化，完全忽视了工业平台的商业化需求（广告变现）。在真实平台中，有机内容（Organic）和赞助广告（Sponsored）必须**共存于同一推荐列表**，但两者优化目标截然不同：

| 维度 | 有机内容 | 赞助广告 |
|------|---------|---------|
| 选择依据 | 用户偏好估计 | 相关性 + 出价（Bid）+ 广告拍卖收益 |
| 价格信号 | 无 | 实时波动，反映广告主对库存的置信度 |
| 优化目标 | 语义匹配 | 混合目标（相关性 × 经济价值） |

**两大核心挑战**：
1. **信号混淆**：将有机和广告数据混合训练会造成目标函数冲突
2. **实时出价盲区**：模型训练时仅能看到历史 log，无法感知实时出价变化，导致错过高价值广告机会

---

## GEM-Rec 框架

### 总体设计哲学

GEM-Rec 在 TIGER（基于 Semantic ID 的生成式推荐）基础上，通过两个核心机制解决上述问题：
1. **控制 Token 机制**：在词表中引入显式控制 token，将"展示哪种广告位"与"展示哪个物品"解耦
2. **竞价感知解码（GEM-Decoding）**：推理时将实时出价注入解码过程，无需重新训练即可动态调整策略

---

### 一、统一序列构建

**序列格式**：将用户历史交互表示为模式感知的元组 $`(m, i)`$，其中 $`m \in \{\text{Organic}, \text{Sponsored}\}`$，$`i`$ 为物品 Semantic ID。

扩展词表，引入两个控制 token：$`\mathcal{F} = \{ \texttt{<ORG>}, \texttt{<AD>} \}`$

每步交互的序列片段：

```math
\mathbf{x}_t = [f_t] \oplus [c_{t,1}, c_{t,2}, \dots, c_{t,D}]
```

其中 $`f_t = \texttt{<AD>}`$ 表示赞助广告，$`f_t = \texttt{<ORG>}`$ 表示有机内容。完整序列为所有片段的拼接：$`\mathbf{x} = \mathbf{x}_1 \oplus \mathbf{x}_2 \oplus \dots \oplus \mathbf{x}_T`$

**控制 Token 的双重作用**：
- `<ORG>` 下：模型进入"偏好模式"，检索最匹配用户有机历史的物品
- `<AD>` 下：模型进入"变现模式"，学习历史上既有语义相关性又有商业可行性的广告物品分布

---

### 二、因式分解生成目标

对序列 $`\mathbf{x}`$ 的负对数似然进行因式分解：

```math
P_{\theta}(\mathbf{x}_{t} \mid H_{\lt t}) = \underbrace{P_{\theta}(f_{t} \mid H_{\lt t})}_{\text{广告满意度建模}} \cdot \underbrace{\prod_{k=1}^{D} P_{\theta}(c_{t,k} \mid H_{\lt t}, f_{t}, c_{t,\lt k})}_{\text{模式条件检索}}
```

**两个子目标的含义**：

| 子目标 | 学习内容 | 机制 |
|--------|---------|------|
| $`P(f)`$：广告满意度建模 | 学习广告展示的上下文边界（如用户疲劳、叙事中断等） | 从历史成功交互中学习，隐式压制降低用户体验的广告位 |
| $`P(c\|f)`$：模式条件检索 | 在给定广告位类型下检索最合适的物品 | $`f=\texttt{<ORG>}`$ 时优化纯有机偏好；$`f=\texttt{<AD>}`$ 时检索历史成功广告分布中的最优物品 |

---

### 三、推理时竞价调制（GEM-Decoding）

历史 log 只能提供"安全基线"，无法捕获实时市场机会。GEM-Decoding 将出价 $`b`$ 直接注入推理过程，通过标量参数 $`\lambda \geq 0`$ 控制调制强度。

#### 1. 广告位级调制（动态广告负载）

调制是否开启赞助广告位，使用有效赞助候选中的**最高出价** $`b_{max}`$：

```math
\tilde{z}_{\texttt{<AD>}} = z_{\texttt{<AD>}} + \lambda \cdot \log(1 + b_{max})
```

当高价值库存可用时，该项鼓励更频繁地选择 `<AD>` 标志，使广告负载随市场需求动态伸缩。

#### 2. 物品级调制（收益最大化）

在确定展示广告后，利用 Semantic ID 的层次结构进行**前缀感知出价聚合**：

预计算查找表 $`\mathcal{B}(c_k | c_{<k})`$，存储该语义簇内任意有效物品的最高出价。调制 token logits：

```math
\tilde{z}_{c} = z_{c} + \lambda \cdot \log(1 + \mathcal{B}(c))
```

这确保了**竞价感知 Logit 塑形**：在语义合理的 token 中，解码器偏向包含高出价物品的分支，在序列早期即剪枝低价值分支。

#### 3. 层次化解码策略

1. **标志采样**：首先对 $`\{ \texttt{<ORG>}, \texttt{<AD>} \}`$ 计算调制 logits 并采样，强制在内容生成前确定广告位类型（若不这样做，Beam Search 会因有机序列先验概率更高而剪枝赞助假设）
2. **内容 Beam Search**：以采样到的标志为条件，在调制 logits $`\tilde{z}`$ 上运行宽度 $`K`$ 的 Beam Search 生成 Semantic ID tokens

---

### 四、理论保证

**定义（GEM 分配规则）**：

```math
x_i(b) = \underbrace{P_\lambda(f=\texttt{<AD>} \mid H, b)}_{\text{随机广告位}} \cdot \underbrace{\mathbb{I}[\mathcal{D}_K(H, b) = i]}_{\text{确定性物品}}
```

**命题 1（分配单调性）**：GEM 分配规则是单调的——对任意上下文 $`H`$ 和固定竞争出价 $`b_{-i}`$，赞助物品 $`i`$ 的曝光概率 $`x_i(b_i, b_{-i})`$ 关于 $`b_i`$ 单调不减。

> **证明思路**：分两部分。(1) 广告位决策：$`b_{max}`$ 关于 $`b_i`$ 单调不减，softmax 关于目标 logit 严格递增，故 $`P(\texttt{<AD>})`$ 单调不减。(2) Beam Search 单调性：对任意竞争假设 $`h_j`$，分支分歧点处目标 token 的 logit 关于 $b_i$ 单调不减，而竞争 token 的 logit 不变但分母增大，故相对分数差 $S_t(h_i) - S_t(h_j)$ 单调不减。

**命题 2（结构一致性）**，包含三项设计保证：

| 保证 | 含义 |
|------|------|
| **安全回退** | $`\lambda=0`$ 时，所有调制 logits 等于基础模型 logits，系统退化为标准生成式推荐 |
| **有机完整性** | 调制项严格以 $`f=\texttt{<AD>}`$ 为门控，有机内容的相对排序对任意 $`\lambda`$ 保持不变 |
| **泛化性** | 训练语料仅含有机交互时，$`P(f=\texttt{<AD>}) \to 0`$，系统退化为标准 TIGER |

## 实验设计

### 合成市场环境构建

由于标准基准数据集缺乏出价 log 和统一交互历史，论文构建了合成市场环境：

**数据集**：Steam、Amazon Beauty、Sports、Toys（各取 20% 物品作为赞助库存，分配对数正态出价）

**两阶段数据生成策略**：
1. **语义相关性过滤**：候选广告物品必须与有机目标物品共享深度 $`d=2`$ 的 Semantic ID 前缀
2. **概率拍卖**：在相关候选中按出价 softmax 采样赢家（温度 $`\tau=0.1`$，强偏向高价值物品）
3. **频率上限**：基于上次广告曝光后的步数线性恢复接受率（防止广告疲劳）

**主实验**：基础广告密度约 3-6%；**消融实验**：高密度设置约 10-20%

### 评估维度

| 维度 | 指标 | 说明 |
|------|------|------|
| **严格策略拟合** | Total NDCG@10 / Recall@10 | 同时预测正确广告位类型 AND 正确物品 ID |
| **有机完整性** | Conditional Organic NDCG@10 | 仅在模型选择有机广告位时评估有机排序质量 |
| **经济价值与可控性** | Ad Rate / Revenue / Ad Relevance | 衡量系统对 $`\lambda`$ 的响应性 |

## 实验结果

### 主实验：市场性能

| 数据集 | 方法 | Ad Rate | Revenue | NDCG@10 | Recall@10 | O.NDCG@10 | O.Recall@10 |
|--------|------|---------|---------|---------|-----------|-----------|-------------|
| **Steam** | TIGER（基准） | 0.0% | - | 0.1442 | 0.1818 | 0.1487 | 0.1875 |
| (Train Ad: 3.49%) | GEM-Rec ($`\lambda`$=0.0) | 2.5% | 535 | 0.1411 | 0.1782 | 0.1468 | 0.1857 |
| | GEM-Rec ($`\lambda`$=1.0) | 4.7% | 1,173 | 0.1381 | 0.1742 | 0.1467 | 0.1853 |
| **Beauty** | TIGER（基准） | 0.0% | - | 0.0282 | 0.0529 | 0.0293 | 0.0550 |
| (Train Ad: 4.23%) | GEM-Rec ($`\lambda`$=0.0) | 3.1% | 345 | **0.0301** | **0.0569** | 0.0318 | 0.0602 |
| | GEM-Rec ($`\lambda`$=1.0) | 6.0% | 726 | 0.0295 | 0.0559 | 0.0320 | 0.0606 |
| **Sports** | TIGER（基准） | 0.0% | - | 0.0176 | 0.0308 | 0.0187 | 0.0327 |
| (Train Ad: 6.85%) | GEM-Rec ($`\lambda`$=0.0) | 4.7% | 804 | 0.0173 | 0.0335 | 0.0187 | 0.0364 |
| | GEM-Rec ($`\lambda`$=1.0) | 9.0% | 1,804 | 0.0168 | 0.0323 | 0.0185 | 0.0359 |
| **Toys** | TIGER（基准） | 0.0% | - | 0.0278 | 0.0536 | 0.0291 | 0.0561 |
| (Train Ad: 4.52%) | GEM-Rec ($`\lambda`$=0.0) | 3.5% | 319 | 0.0277 | 0.0536 | 0.0295 | 0.0570 |
| | GEM-Rec ($`\lambda`$=1.0) | 7.1% | 717 | 0.0272 | 0.0522 | 0.0298 | 0.0575 |

> **关键观察**：有机指标（O.NDCG、O.Recall）在 $`\lambda`$ 增大时基本保持稳定，验证了有机完整性命题。Total NDCG 的下降是预期内的（高 $`\lambda`$ 强制将有机位替换为广告位，造成与有机主导测试集的偏离）。

### 可控性验证（Steam 数据集）

$`\lambda`$ 提供单调的广告负载控制，扫描范围 $`\lambda \in [0, 10]`$：

| $`\lambda`$ | Ad Rate | Revenue |
|-----------|---------|---------|
| 0.0 | 2.5% | 535 |
| 0.5 | 3.6% | 815 |
| 1.0 | 4.7% | 1,174 |
| 2.0 | 8.9% | 2,390 |
| 5.0 | 40.6% | 12,224 |
| 10.0 | 93.7% | 30,599 |

**有机完整性验证**：Total NDCG 随 $`\lambda`$ 下降（红线），但 Conditional Organic NDCG 保持平稳（绿线），确认广告注入不污染有机排序逻辑。

### 市场波动适应性（Bid Shock 实验）

模拟"出价冲击"：随机选取 5% 库存，将其出价乘以 $`10\times`$，观察系统适应能力（**High-Value Share** = 展示广告中高价值物品的占比）：

**Steam 数据集**：

| 设置 | Ad Rate | High-Value Share | Revenue Uplift |
|------|---------|-----------------|----------------|
| Baseline ($`\lambda`$=0.0) | 2.4% | 21.8% | $`1.0\times`$ |
| GEM-Rec ($`\lambda`$=0.5) | 7.1% | **81.5%** | $`9.0\times`$ |
| GEM-Rec ($`\lambda`$=1.0) | 18.0% | **97.4%** | $`28.2\times`$ |
| GEM-Rec ($`\lambda`$=2.0) | 62.4% | **99.9%** | $`104.6\times`$ |

**关键洞察**：GEM-Rec 不仅增加广告量，更主动**用高价值曝光替换低价值曝光**——$`\lambda=0.5`$ 时总广告率仅从 2.4% 升至 7.1%，但高价值份额从 21.8% 跃升至 81.5%，实现 $`9\times`$ 收益提升。

其他数据集规律一致（Toys: $`1.0\times \to 202.4\times`$，Beauty: $`1.0\times \to 240.6\times`$，Sports: $`1.0\times \to 136.2\times`$）。

### 广告质量权衡

使用**前缀匹配深度**（Prefix Match Depth）量化广告与用户有机意图的语义相似度：

- 总体趋势：$`\lambda`$ 增大 → 广告相关性下降（优先经济价值必然牺牲部分语义匹配）
- 局部例外：在稀疏数据集中，出价注入有时充当"上下文感知的平局裁决者"——模型已正确识别目标类目，但缺乏足够样本在簇内排序，此时出价信号引导模型选中历史上更高出价（也更可能是用户真实购买目标）的物品，略微提升相关性

### 生成有效性

GEM-Rec 在所有数据集和所有 $`\lambda`$ 设置下，广告 ID 生成有效率均为 **100%**，无需前缀树约束即实现零幻觉。

## 定价机制与 DSIC 讨论

**当前实现**：采用**一价拍卖**（First-Price），赢家直接支付其出价。理由：
- 与工业界趋势一致（Google、Amazon 等均已迁移至一价拍卖）
- 无需反事实计算，部署成本低

**DSIC 的挑战**：真正的主导策略激励相容（DSIC）机制需要：
1. 单调的分配规则（GEM-Rec 已满足，见命题 1）
2. 基于"临界出价"的支付规则（需要反事实解码：出价降低多少后模型会生成不同结果？）

在自回归解码中，临界出价的计算需要推理反事实解码路径，计算复杂度高；且 Beam Search 等近似解码使分配函数不连续，难以精确刻画阈值。**DSIC 实现留作未来工作**。

## 核心贡献总结

| 创新点 | 解决的问题 | 核心机制 |
|--------|-----------|---------|
| **统一生成架构** | 有机/广告目标混淆 | 控制 Token `<ORG>`/`<AD>` 在词表级解耦广告位决策与内容检索 |
| **端到端市场策略学习** | 历史 log 的隐含约束难以显式建模 | 从成功交互 log 中自动学习广告展示的上下文可行性边界 |
| **竞价感知约束解码** | 模型对实时出价变化盲目 | 推理时将出价注入两级 logit 调制（广告位级 + 物品级），理论证明分配单调性和有机完整性 |

## 与相关工作的关键区别

| 对比维度 | GEM-Rec | 其他工作 |
|---------|---------|---------|
| vs TIGER / OneRec | 新增广告变现能力，保持语义检索 | 仅优化有机偏好 |
| vs RL 广告插入（DEAR 等） | 生成式检索模型，输出可被实时出价调制 | 非生成式，无法在 token 级注入出价 |
| vs GPR | 生成混合有机/广告列表，分离"何时展示广告"与"展示哪个广告" | 仅以广告为生成目标 |
| vs 拍卖机制设计（RegretNet 等） | 不学习新的拍卖机制，专注推理时竞价调制，固定一价支付 | 学习分配/支付网络 |

---

## 局限性与未来工作

1. **合成数据验证**：受限于真实广告 log 不公开，实验基于合成市场环境，实际部署效果有待验证
2. **DSIC 支付机制**：当前一价拍卖不保证真实出价，完整 DSIC 实现需解决反事实解码计算问题
3. **独立调制参数**：引入 $`\lambda_{slot}`$（广告位频率）和 $`\lambda_{item}`$（出价排序）两个独立参数，可提供更精细控制
4. **代码开源**：论文接受后将开源
