## 研究背景与动机

### 工业界多模态推荐的主流范式

当前工业界普遍采用**两阶段级联范式**：
1. **预训练**：用多模态大模型（MLLM）对短视频的文本、图像、音频信息进行压缩，生成全能表示，存入缓存
2. **下游使用**：推荐模型将缓存中的多模态表示作为**额外固定输入特征**进行训练

快手的推荐系统同样采用此方案，在购物、广告等多个业务取得了显著 A/B 增益。

### 现有方案的两大核心问题

| 问题 | 描述 |
|------|------|
| **表示不匹配（Representation Unmatching）** | 多模态预训练任务（如图文匹配 CLIP）与推荐任务（用户点击行为）目标不一致，表示分布存在根本性偏差 |
| **表示不可学（Representation Unlearning）** | 多模态表示以固定特征形式输入，无法随推荐模型梯度更新，限制模型收敛 |

## 核心方法：QARM

QARM（**Q**uantitative **A**lignment Multi-Modal **R**ecommendation）包含两大机制：

### 一、Item Alignment（物品对齐）机制

**目标**：解决表示不匹配问题，使 MLLM 表示与下游业务用户行为分布对齐。

**步骤**：

1. **构建高质量 Item2Item 对数据集** $\mathcal{D}$：
   - 基于 User2Item 检索模型：对用户正向点击的目标物品，从其历史 50 个正向点击物品中找到 ID 表示空间最相似的物品作为触发物品
   - 基于 Item2Item 检索模型（如 Swing 模型）：导出具有高相似度的稳定物品对

2. **对 MLLM 进行业务定制化微调**：构建纯多模态输入的 Item2Item 对齐模型，用批次对比损失训练：

$$\mathbf{M}_{\texttt{trigger}} = \texttt{MLLM}(\mathbf{T}^{\texttt{text}}_{\texttt{trigger}}, \mathbf{T}^{\texttt{audio}}_{\texttt{trigger}}, \mathbf{T}^{\texttt{image}}_{\texttt{trigger}})$$

$$\mathbf{M}_{\texttt{target}} = \texttt{MLLM}(\mathbf{T}^{\texttt{text}}_{\texttt{target}}, \mathbf{T}^{\texttt{audio}}_{\texttt{target}}, \mathbf{T}^{\texttt{image}}_{\texttt{target}})$$

$$\mathcal{L}_{\texttt{align}} = \texttt{Batch-Contrastive}(\mathbf{M}_{\texttt{trigger}}, \mathbf{M}_{\texttt{target}}, \mathcal{B})$$

**关键点**：不同业务需要不同的对齐特性（如购物推荐关注同品类关系，短视频推荐关注跨品类因果关系），因此对齐模型需针对各业务定制化微调。

---

### 二、Quantitative Code（量化码）机制

**目标**：解决表示不可学问题，将对齐后的多模态表示转化为可端到端训练的 Semantic ID。

受 VQ-VAE、RQ-VAE 和 Straight-Through Estimator 启发，QARM 设计了两种量化码：

#### 1. VQ Code（向量量化码）

直接以所有物品的对齐表示矩阵 $\mathbf{M}$ 作为码本（无需训练新码本）：

$$\mathbf{V} = \mathbf{M}$$

对任意物品表示 $\mathbf{m}$，通过 TopK 近邻搜索得到 $K$ 个 VQ 码：

$$v_1, v_2, \dots, v_K = \texttt{TopKCode}(\mathbf{V}, \mathbf{m}, K)$$

默认设置：$K = 25$

#### 2. RQ Code（残差量化码）

使用 $L$ 层 K-means 码本，以粗到细的方式递归量化：

$$\mathbf{R}^1 = \texttt{Kmeans}(\mathbf{M}, N), \quad \mathbf{M}^1 = \mathbf{M} - \texttt{NearestRep}(\mathbf{M}, \mathbf{R}^1)$$
$$\mathbf{R}^2 = \texttt{Kmeans}(\mathbf{M}^1, N), \quad \dots$$

对任意物品表示 $\mathbf{m}$，逐层搜索最近邻索引得到 $L$ 个 RQ 码：

$$r_l = \texttt{NearestCode}(\mathbf{R}^l, \mathbf{m}^{l-1}, 1)$$

默认设置：$L = 6$，码本维度 $d = 64$

**两种码的互补性**：
- VQ Code：用 TopK 相似物品邻居表示目标物品（邻域信息）
- RQ Code：将整个表示编码为层次残差路径（全局结构信息）

---

### 三、下游推荐模型中的使用方式

量化码作为可学习的 Semantic ID 输入下游模型，支持端到端训练：

| 特征类型 | 描述 |
|---------|------|
| **物品侧特征** | 将 VQ/RQ 码作为 Item ID，分配对应 Embedding 空间进行查表 |
| **用户侧特征** | 将用户最近正向交互物品序列的量化码作为用户兴趣表示 |
| **目标物品感知特征** | 用目标物品的 RQ 码搜索匹配序列，生成 Cross Feature（如一码匹配序列、两码匹配序列等）|

整体训练目标（多任务 MoE 框架）：

$$\hat{y}^\texttt{ctr}, \hat{y}^\texttt{lvtr}, \dots = \texttt{MoE}([\texttt{UserCodeRep}, \texttt{ItemCodeRep}, \texttt{CrossCodeRep}, \texttt{OtherFeaRep}])$$

$$\mathcal{L} = -\sum_{\texttt{xtr}} \left( y^{\texttt{xtr}} \log(\hat{y}^{\texttt{xtr}}) + (1 - y^{\texttt{xtr}}) \log(1 - \hat{y}^{\texttt{xtr}}) \right)$$

## 系统部署

QARM 部署于快手分布式训练系统，作为**特征工程模块**的一部分：
- 对用户新上传的短视频进行预处理，生成 ID 码
- 离线数据引擎每日汇聚数千亿用户-物品对特征和交互标签
- 训练参数实时同步到在线推理模型

## 实验结果

### 离线结果

**广告服务（Advertising）**

| 模型变体 | CTR-AUC | CTR-UAUC | CTR-GAUC | CVR-AUC | CVR-UAUC | CVR-GAUC |
|---------|---------|---------|---------|---------|---------|---------|
| Baseline | 90.85% | 70.88% | 72.43% | 88.29% | 69.01% | 74.03% |
| + IA Rep | +0.02% | +0.05% | -0.01% | +0.03% | +0.04% | -0.02% |
| + VQ Code | +0.11% | +0.20% | +0.17% | +0.09% | +0.05% | +0.11% |
| + RQ Code | +0.10% | +0.15% | +0.13% | +0.07% | +0.09% | +0.10% |
| **+ VQ & RQ Code** | **+0.18%** | **+0.29%** | **+0.25%** | **+0.17%** | **+0.08%** | **+0.12%** |

**购物服务（Shopping）**

| 模型变体 | CTR-AUC | CTR-UAUC | CTCVR-AUC | CTCVR-UAUC |
|---------|---------|---------|---------|---------|
| Baseline | 79.31% | 66.48% | 87.92% | 70.70% |
| + IA Rep | -0.03% | +0.02% | -0.07% | +0.13% |

**关键观察**：
1. 单独使用对齐表示（IA Rep）提升有限，说明仅解决不匹配问题不够
2. 量化码（VQ/RQ）带来显著提升，说明端到端可学习对推荐模型至关重要
3. VQ + RQ 联合使用效果最佳，两者互补

## 核心贡献总结

1. **提出 QARM 框架**，系统性解决工业多模态推荐的两大核心问题：表示不匹配（Item Alignment）和表示不可学（Quantitative Code）

2. **Item Alignment 机制**：利用下游业务 Item2Item 对数据，通过批次对比损失微调 MLLM，使多模态表示与真实业务交互分布对齐，且支持不同业务定制化

3. **Quantitative Code 机制**：设计 VQ（TopK 近邻索引）和 RQ（层次 K-means 残差量化）两种互补量化码，将多模态表示转化为可端到端训练的 Semantic ID，支持物品侧、用户侧、目标物品感知三类特征

4. **显著工业效果**：广告业务 Revenue +9.704%，购物业务 GMV +2.296%；对冷启动和长尾物品尤为友好，改善平台内容生态公平性

5. **大规模落地**：2024 年 3 月起在快手多个业务全量部署，日服务 4 亿用户
