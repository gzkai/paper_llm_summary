## 研究背景与动机

推荐系统是大规模互联网平台的核心基础设施，历经十余年从协同过滤到深度学习的演进。然而，**传统多级级联架构**（召回 → 预排 → 精排）存在三大根本性缺陷：

### 缺陷一：计算碎片化（Fragmented Compute）

以快手为例，服务期间超过 **50% 的资源**消耗在通信与存储上，而非高精度计算。现有排序模型训练 MFU 仅 **4.6%**，推理 MFU 仅 **11.2%**，远低于 LLM 社区约 40% 的水平（H100 上）。高 QPS（>400k）和低延迟（<500ms）的要求进一步限制了模型规模的扩展。

### 缺陷二：目标冲突（Objective Collision）

- 快手内部有**数百个**来自用户、创作者、平台生态的优化目标，各阶段独立优化导致系统一致性差；
- 不同阶段的模型结构和规模差异，使得召回阶段的效果受到排序模型局限性的制约，形成跨阶段建模冲突。

### 缺陷三：落后于 AI 演进（Lag Behind AI Evolution）

级联架构在结构上阻碍了 Scaling Law、强化学习等已在 LLM 领域验证的突破性技术在推荐场景中的应用。

## 核心方法：OneRec 架构

**OneRec** 通过**端到端生成式**方法重塑推荐系统，采用编码器-解码器架构，将召回与排序融合为单一阶段。

### 整体流程

1. **训练阶段**：Tokenizer 将视频转化为语义 ID → 编码器压缩用户全生命周期行为 → 解码器通过下一 Token 预测生成目标物品 → 奖励系统进行强化学习对齐
2. **推理阶段**：生成语义 ID → 映射回视频推荐 → 可选奖励模型二次筛选

---

### 1. Tokenizer：协同感知多模态语义 ID

**问题**：十亿级视频空间无法使用原子 ID，需要将视频 tokenize 为固定词表的语义 ID。

**方案**：整合协同信号与多模态特征，使用 RQ-Kmeans 生成高质量层级语义 ID。

#### 1.1 协同感知多模态表示（Aligned Collaborative-Aware Multimodal Representation）

**多模态输入**：每个视频的字幕、标签、ASR、OCR、封面图、5 帧均匀采样帧，通过 miniCPM-V-8B 生成 $N_M=1280$ 个 token 向量 $`\mathbf{M}\in\mathbb{R}^{N_M\times d_t}`$（$`d_t=512`$）。

**QFormer 压缩**：用 4 个可学习查询向量将 1280 个 token 压缩为 4 个：

```math
\mathbf{Q}^{(i+1)} = \text{CrossAttn}\left(\mathbf{Q}^{(i)}, \mathbf{M}, \mathbf{M}\right)
```

**物品对构建**：通过用户点击对（User-to-Item）和物品相似度（Item-to-Item，如 Swing 相似度）构建高质量物品对数据集 $\mathcal{D}_{pair}$。

**双目标训练**：
- **I2I 对比损失**：对齐协同相似物品对的多模态表示：

```math
\mathcal{L}_{I2I} = -\frac{1}{|\mathcal{B}|} \sum_{(i,j)\in\mathcal{B}} \log \frac{\exp(\text{sim}(\tilde{\mathbf{M}}_i, \tilde{\mathbf{M}}_j)/\tau)}{\sum_{(i',j')\in\mathcal{B}} \exp(\text{sim}(\tilde{\mathbf{M}}_i, \tilde{\mathbf{M}}_{j'})/\tau)}
```

- **Caption 生成损失**：使用 LLaMA3 进行字幕下一 Token 预测，防止幻觉，保留内容理解能力。

#### 1.2 RQ-Kmeans Tokenization

对压缩后的多模态表示进行残差量化，迭代 $`L_t=3`$ 层：

```math
\mathcal{C}^{(l)} = \text{K-means}(\mathcal{R}^{(l)}, N_t), \quad s^l_i = \arg\min_k \|\mathcal{R}^{(l)}_i - \mathbf{c}^{(l)}_k\|
```

```math
\mathcal{R}^{(l+1)}_i = \mathcal{R}^{(l)}_i - \mathbf{c}^{(l)}_{s^l_i}
```

每个视频最终用 3 个从粗到细的语义 ID 表示：$`\{s^1_m, s^2_m, s^3_m\}`$。

**优势**：相比 RQ-VAE，RQ-Kmeans 具有更好的重建质量、码本利用率和均衡性。

### 2. 编码器：多尺度特征工程

编码器通过**四路多尺度**行为表示建模用户兴趣：

| 路径 | 序列长度 | 内容 |
|------|----------|------|
| **用户静态路径** | 1 | uid、age、gender 等静态特征 |
| **短期路径** | $`L_s=20`$ | 最近 20 次交互：vid/sid、aid、tag、ts、playtime、dur、label |
| **正反馈路径** | $`L_p=256`$ | 256 条高互动记录 |
| **终身路径** | $`L_l=2000`$ | 最多 100,000 条历史，经层级 K-means 压缩后 QFormer 进一步压缩为 $`N_q=128`$ 个向量 |

**终身路径压缩策略**：
1. **行为压缩**：对用户交互序列进行层级 K-means 聚类，动态调整聚类数为 $`\lfloor\sqrt[3]{|D|}\rfloor`$，选择最近聚类中心的物品作为代表；
2. **特征聚合**：离散特征（vid、aid、label）继承代表视频特征；连续特征（tag、ts、playtime、duration）取聚类内均值；
3. **QFormer 进一步压缩**：2 层 QFormer 将 2000 条压缩为 128 个查询向量。

**编码器架构**：四路输出拼接后经 $`L_{enc}`$ 层 Transformer 处理：

```math
\mathbf{z}^{(1)} = [\mathbf{h}_u; \mathbf{h}_s; \mathbf{h}_p; \mathbf{h}_l] + \mathbf{e}_{\text{pos}}
```

### 3. 解码器：MoE 增强的点对点生成

解码器采用**点对点生成**范式，对每个目标视频 $m$ 构建输入序列：

```math
\mathcal{S}_m = \{s_{[\text{BOS}]}, s^1_m, s^2_m, s^3_m\}
```

每层解码器执行：

```math
\mathbf{d}^{(i+1)}_m = \mathbf{d}^{(i)}_m + \text{CausalSelfAttn}(\mathbf{d}^{(i)}_m)
```
```math
\mathbf{d}^{(i+1)}_m = \mathbf{d}^{(i+1)}_m + \text{CrossAttn}(\mathbf{d}^{(i+1)}_m, \mathbf{Z}_{\text{enc}}, \mathbf{Z}_{\text{enc}})
```
```math
\mathbf{d}^{(i+1)}_m = \mathbf{d}^{(i+1)}_m + \text{MoE}(\text{RMSNorm}(\mathbf{d}^{(i+1)}_m))
```

**MoE 层**：$`N_{\text{experts}}`$ 个专家 + top-$`k`$ 路由，采用 DeepSeek 的无损失负载均衡策略：

```math
\text{MoE}(\mathbf{x}) = \sum_{j=1}^k \text{Gate}_j(\mathbf{x}) \cdot \text{Expert}_j(\mathbf{x})
```

**训练目标**：语义 ID 的下一 Token 预测（NTP）：

```math
\mathcal{L}_{\text{NTP}} = -\sum_{j=0}^{L_t-1} \log P(s_m^{j+1} | [s_{[\text{BOS}]}, s^1_m, \ldots, s^j_m])
```

### 4. 奖励系统：三层强化学习对齐

预训练模型仅拟合历史曝光分布，无法突破传统推荐天花板。奖励系统通过 on-policy 强化学习在生成物品空间中训练模型。

#### 4.1 用户偏好对齐：P-Score + ECPO

**P-Score（偏好分数）**：基于 SIM 架构的神经网络，学习个性化多目标融合分数（取代人工调权）。多塔结构对 CTR、LVR、LTR、VTR 等各目标分别计算 BCE 损失，最终 MLP 输出统一的 P-Score：

```math
\mathcal{L}_{\text{P-Score}} = \sum_{xtr \in S_o} w^{xtr} \mathcal{L}_{\text{P-Score}}^{xtr}
```

其中 $`S_o = \{\text{ctr, lvtr, ltr, vtr, ...}\}`$。

**ECPO（Early Clipped GRPO）**：对 GRPO 的改进，解决负优势下梯度爆炸问题：

```math
\mathcal{J}_{ECPO}(\theta) = \mathbb{E}\left[\frac{1}{G}\sum_{i=1}^G \min\left(\frac{\pi_\theta(o_i|u)}{\pi_{\theta_{old}}'(o_i|u)}A_i, \text{clip}\left(\frac{\pi_\theta(o_i|u)}{\pi_{\theta_{old}}'(o_i|u)}, 1-\epsilon, 1+\epsilon\right)A_i\right)\right]
```

**关键改进**：对负优势样本进行提前裁剪（Early Clipping），限制策略比率的最大值：

```math
\pi_{\theta_{old}}'(o_i|u) = \max\left(\frac{\text{sg}(\pi_\theta(o_i|u))}{1+\epsilon+\delta}, \pi_{\theta_{old}}(o_i|u)\right), \quad \delta > 0
```

实践中设 $`\delta=0.1`$，移除 KL 散度损失（由 SFT 损失保证稳定性）。

#### 4.2 生成格式正则化

**问题**：RL 训练后的"挤压效应"导致非法生成（语义 ID 无法映射到实际视频）比例超过 50%。

**解决方案**：引入格式奖励（Format Reward），对合法样本设置优势为 1，非法样本直接丢弃：

```math
A_i = \begin{cases} 1 & \text{if } o_i \in I_{\text{legal}} \\ 0 & \text{if } o_i \notin I_{\text{legal}} \end{cases}
```

采用随机采样（而非 Top-k 采样）选取样本，最终使合法率稳定提升至 95%，带来 +0.13% APP 停留时长、+0.30% 观看时长的在线提升。

#### 4.3 工业场景对齐（Specific Industrial Reward, SIR）

针对快手平台"搬运号"泛滥问题，当病毒式内容曝光比例超过阈值 $f$ 时，下调其 P-Score 奖励：

```math
r_i' = \begin{cases} r_i & \text{if } o_i \notin I_{\text{viral}} \\ \alpha r_i & \text{if } o_i \in I_{\text{viral}} \end{cases}
```

实验结果：病毒式内容曝光减少 **9.59%**，核心指标（Watch Time、APP Stay Time）保持稳定。

- 模型越大，loss 越低，呈现清晰的 Scaling 趋势；
- 训练数据约 100 亿样本时快速收敛，超过后持续但缓慢提升；

### RL 实验

#### P-Score 奖励最终效果

| 场景 | Watch Time | App Stay Time | Video View |
|------|------------|---------------|------------|
| 快手 | +0.21% | +0.26% | +0.17% |
| 快手极速版 | +0.71% | +0.22% | +0.35% |

#### 关键 RL 发现

- **采样策略**：Beam Search 显著优于 Top-k+Top-p（App Stay Time +0.60%），因语义 ID 的前缀树结构与 Beam Search 的系统探索高度契合；
- **搜索空间**：ECPO 训练 Group Size 从 128 扩大到 512 有显著提升，建议设为推理输出数量的约 4 倍；
- **参考模型**：On-policy（当前策略模型）优于 Off-policy（预训练模型），但存在轻微奖励 Hacking。

## 核心贡献总结

1. **端到端生成式架构**：将召回、预排、精排统一为单一编码器-解码器框架，消除跨阶段优化冲突，实现真正的端到端优化。

2. **协同感知多模态 Tokenizer**：整合协同信号与多模态特征（miniCPM-V-8B + QFormer + RQ-Kmeans），相比纯内容特征的 RQ-VAE 显著提升重建质量。

3. **四路多尺度用户建模**：短期（20）+ 正反馈（256）+ 终身（100k→QFormer→128）+ 静态特征，全面捕捉用户兴趣演变。

4. **ECPO 强化学习框架**：改进 GRPO 的 Early Clipping 机制 + P-Score 个性化偏好奖励 + 格式奖励 + 工业场景奖励，实现稳定高效的推荐对齐。

5. **工程效率突破**：训练 MFU 从 4.6% 提升至 23.7%，推理 MFU 达 28.8%，OPEX 仅为传统系统的 10.6%，向 LLM 社区看齐。

6. **推荐 Scaling Law**：在推荐系统中首次系统验证了参数 Scaling、特征 Scaling、码本 Scaling 和推理 Scaling 的规律。

---

## 局限性与未来工作

1. **推理阶段 Scaling**：当前推理阶段缺乏强推理能力，步骤 Scaling 效果不明显；
2. **多模态集成**：尚未与 LLM/VLM 深度融合，未来计划将用户行为作为原生多模态对齐；
3. **奖励系统设计**：奖励系统仍较初级，存在奖励 Hacking 风险，是最具潜力的改进方向。
