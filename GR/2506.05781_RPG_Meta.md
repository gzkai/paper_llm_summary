## 研究背景与动机

### 现有语义 ID 推荐模型的两大缺陷

语义 ID（Semantic ID）将物品编码为一组离散 token，消除了与物品数量成比例的大型 Embedding 表。近年来生成式范式（如 TIGER）在语义 ID 推荐中取得了显著进展，但存在两大根本性缺陷：

#### 缺陷一：推理效率低

生成式模型（如 TIGER）采用**逐 token 自回归生成**：每次生成下一个 token 需要一次 Transformer 前向传播，生成 Top-K 列表还需要 Beam Search（维护 $b$ 个 Beam，每步前向传播 $b$ 次）。

- 生成一个物品的语义 ID（4个token）需约 $4b$ 次前向传播
- 与检索式方法（单步解码）相比，推理开销极大

#### 缺陷二：语义 ID 表达能力受限

为保证自回归推理的可接受延迟，现有模型将语义 ID 长度限制为极短（如 TIGER 仅用 **4个 token**）。而检索式方法（如 VQ-Rec）可使用 32 个 token，但 32 步 Beam Search 在生成式模型中计算代价极高。

**实验验证**：给 TIGER 增加 token 数量：4→8→16 性能持续下降，推理时间从 259s 增至 2544s；32 个 token 直接 OOM。

## 核心方法：RPG

**RPG**（**R**ecommendation with **P**arallel Semantic ID **G**eneration）的核心思想：**将语义 ID 的所有 token 并行预测**，从而将解码步数从 $`O(m)`$ 降至 $`O(1)`$，进而支持长达 64 个 token 的语义 ID。

### 整体架构

```
输入：用户历史交互序列 {i₁, i₂, ..., i_{t-1}}
         ↓
[物品表示] 每个物品的 OPQ 语义 ID → 聚合 Embedding → 单向量 v_i
         ↓
[序列建模] Transformer Decoder → 序列表示 s ∈ ℝ^d
         ↓
[并行预测] 每个 digit 独立的 Projection Head g_j → 并行预测 m 个 token
         ↓
[图约束解码] 迭代图传播 → Top-K 语义 ID → 推荐结果
```

---

### 模块一：长语义 ID 构造（OPQ）

**为什么选 OPQ 而非 RQ（残差量化）？**

| 对比 | RQ-VAE（TIGER用） | OPQ（RPG用） |
|------|-----------------|------------|
| token 间依赖 | 强顺序依赖（后层量化前层残差） | **无顺序依赖**（各子向量独立量化） |
| 信息分布 | 不均衡（前层信息多，后层少） | **均衡**（各子向量独立编码） |
| 适合并行 | ❌ | ✅ |

**OPQ 构造方式**：将物品 Embedding 分割为 $m$ 个子向量，每个子向量独立量化为一个 token：

```math
(c_1, c_2, \ldots, c_m), \quad c_j \in \mathcal{C}^{(j)} = \{(j-1)M+1, \ldots, jM\}
```

每个 codebook 大小为 $M$，最大语义 ID 长度 $`m = 64`$（远超 TIGER 的 4）。

**物品表示聚合**：将 $m$ 个 token Embedding 通过 Mean/Max Pooling 聚合为单一向量：

```math
\mathbf{v}_i = \text{Aggr}(\mathbf{e}_{1,c_1}, \ldots, \mathbf{e}_{m,c_m})
```

避免将 64 个 token 直接拼接输入序列（会导致上下文过长）。

---

### 模块二：多 Token 预测损失（MTP）

由于 OPQ 各子向量独立量化，给定序列表示 $`\mathbf{s}`$，各 token **条件独立**：

```math
\mathbb{P}(c_{t,1}, \ldots, c_{t,m} | s) = \prod_{j=1}^{m} \mathbb{P}^{(j)}(c_{t,j} | s)
```

**MTP 损失**（对 $m$ 个 token 独立预测，每个 token 有独立的 Projection Head $`\text{g}_j`$）：

```math
\mathcal{L} = -\sum_{j=1}^{m} \log \mathbb{P}^{(j)}(c_{t,j}|s) = -\sum_{j=1}^{m} \log \frac{\exp(\mathbf{e}_{c_{t,j}}^\top \cdot \text{g}_j(\mathbf{s}) / \tau)}{\sum_{c \in \mathcal{C}^{(j)}} \exp(\mathbf{e}_{c}^\top \cdot \text{g}_j(\mathbf{s}) / \tau)}
```

- $`\tau`$：温度超参数
- $`\text{g}_j(\cdot)`$：第 $j$ 个 digit 专属的 MLP Projection Head
- **每个 digit 有独立的 Projection Head**，将序列表示映射到不同语义子空间（消融实验验证必要性）

**与传统损失的对比**：

| 损失类型 | 优化目标 | 语义粒度 |
|---------|---------|---------|
| 传统 Cross-Entropy（SASRec等） | 物品空间内概率 | 物品级 |
| 自回归 NTP（TIGER） | token 空间内概率（有顺序依赖） | Sub-item 级 |
| **MTP（RPG）** | **token 空间内概率（无顺序依赖）** | **Sub-item 级** |

---

### 模块三：高效 Logit 计算

**朴素方法**：对每个候选物品逐一计算 logit，时间复杂度 $`O(Nmd)`$（$N$ 为物品数）。

**优化方法**：先缓存序列表示与所有 token Embedding 的点积：

```math
\mathbf{p}^{(j)} = \text{softmax}\left(\mathbf{E}_j \cdot \text{g}_j(\mathbf{s}) / \tau\right) \in \mathbb{R}^{M}
```

对候选语义 ID $`(c_1, \ldots, c_m)`$，logit 计算简化为：

```math
\text{logit} = \sum_{j=1}^{m} \log \mathbf{p}^{(j)}_{c_j}
```

**优化后时间复杂度**：$`O(Mmd + Nm)`$，当 $`N > \frac{d}{d-1}M`$ 时更高效。

---

### 模块四：图约束解码（Graph-Constrained Decoding）

#### 关键观察

由于 logit 是各 token logit 之和，**两个仅差少数 digit 的语义 ID，其预测 logit 应该相近**（实验验证：差异 digit 数越多，logit 差距越大）。

#### 构建解码图

- **节点**：物品池中所有有效语义 ID
- **边权重**：两个语义 ID 的语义相似度：

```math
\text{sim}((c_{1,1},\ldots,c_{1,m}), (c_{2,1},\ldots,c_{2,m})) = \sum_{j=1}^{m} \mathbf{e}_{j,c_{1,j}}^\top \cdot \mathbf{e}_{j,c_{2,j}}
```

- **稀疏化**：每个节点仅保留相似度最高的 Top-$k$ 条边
- 图构建后可**复用**，直到模型重训或物品重新量化

#### 迭代图传播解码

1. **初始化 Beam**：从物品池随机采样 $b$ 个语义 ID 作为初始 Beam
2. **图传播扩展**：将当前 Beam 中所有节点的 $`b \times k`$ 个邻居加入候选集
3. **保留最优**：用缓存 logit 计算候选集中所有节点的分数，保留 Top-$b$
4. **迭代 $q$ 步**：重复步骤 2-3，最终 Beam 中的语义 ID 即为 Top-K 推荐结果

#### 复杂度分析

| 方法 | 时间复杂度 | 是否与物品数 $N$ 相关 |
|------|-----------|---------------------|
| SASRec（检索） | $`O(Nd)`$ | ✅ 正比 |
| TIGER（生成） | $`O(mathbf \cdot Nd)`$ | ✅ 正比 |
| **RPG（并行生成）** | $`O(Mmd + bqkm)`$ | **❌ 无关** |

**实测效率**（Sports 数据集，物品数变化实验）：
- 运行时内存：RPG 比 TIGER 少约 **25×**
- 推理时间：RPG 比 TIGER 快约 **15×**
- 与 SASRec/VQ-Rec 不同，RPG 内存和时间**不随物品数增加而增长**

## 实验结果

### 数据集

| 数据集 | #用户 | #物品 | #交互 | 平均序列长度 |
|--------|-------|-------|-------|------------|

| Beauty | 22,363 | 12,101 | 176,139 | 8.87 |
| Toys | 19,412 | 11,924 | 148,185 | 8.63 |
| CDs | 75,258 | 64,443 | 1,022,334 | 14.58 |

### 整体性能对比（NDCG@10）

| 方法 | Sports | Beauty | Toys | CDs |
|------|--------|--------|------|-----|
| SASRec（Item ID） | 0.0192 | 0.0318 | 0.0374 | 0.0263 |
| VQ-Rec（语义ID检索） | 0.0173 | 0.0383 | 0.0423 | 0.0292 |
| TIGER（语义ID生成） | 0.0225 | 0.0384 | **0.0432** | 0.0411 |
| HSTU（语义ID生成） | 0.0215 | 0.0389 | 0.0357 | 0.0346 |
| **RPG（ours）** | **0.0263** | **0.0464** | **0.0490** | **0.0415** |

- **12 个指标中排名第一 11 个**
- 平均比最强基线 NDCG@10 提升 **12.6%**

### 消融实验（NDCG@10）

| 变体 | Sports | Beauty | Toys | CDs |
|------|--------|--------|------|-----|
| **(1.1) OPQ → 随机 token** | 0.0179 | 0.0359 | 0.0288 | 0.0078 |
| **(1.2) OPQ → RQ** | 0.0242 | 0.0421 | 0.0458 | 0.0406 |
| **(2.1) 无 Projection Head** | 0.0252 | 0.0423 | 0.0430 | 0.0361 |
| **(2.2) 共享 Projection Head** | 0.0256 | 0.0424 | 0.0438 | 0.0368 |
| **(3.1) Beam Search 解码** | 0.0000 | 0.0000 | 0.0000 | 0.0000 |
| **(3.2) 无图约束** | 0.0082 | 0.0214 | 0.0205 | 0.0183 |
| **RPG（完整）** | **0.0263** | **0.0464** | **0.0490** | **0.0415** |

**关键发现**：
- Beam Search 完全失效（NDCG@10 = 0.0000）：OPQ 无顺序依赖，Beam Search 陷入局部最优
- 无图约束性能大幅下降：图约束对引导长语义 ID 生成至关重要
- 随机 token 性能最差：验证 MTP 损失有效学习了语义 ID 中的语义信息
- 独立 Projection Head > 共享 Head > 无 Head：不同 digit 需映射到不同语义子空间

### 语义 ID 长度扩展性

| 语义 ID 长度 $m$ | Sports | Beauty | Toys | CDs |
|----------------|--------|--------|------|-----|
| 4 | 差 | 差 | 差 | 差 |
| 8 | ↑ | ↑ | ↑ | ↑ |
| 16 | **最优** | ↑ | **最优** | ↑ |
| 32 | ≈16 | **最优** | ≈16 | ↑ |
| 64 | ≈32 | ≈32 | ≈32 | **最优** |

- 整体趋势：语义 ID 越长，性能越好（尤其大数据集）
- 小数据集较早收敛（Sports/Toys 在 m=16 达最优）
- 大数据集（CDs）在 m=64 仍有提升

### 冷启动分析（Sports 数据集）

按训练集出现次数分 4 组（[0,5]、[6,10]、[11,15]、[16,20]）：
- RPG 在所有组别均优于 VQ-Rec 和 TIGER
- 冷启动物品（[0,5]）上优势尤为显著
- 原因：MTP 损失直接在 token 空间优化，长语义 ID 丰富了 token 语义

### 超参数分析（Sports 数据集）

| 超参 | 最优值 | 规律 |
|------|--------|------|
| Beam size $b$ | 10 | $`b > 10`$ 后收益递减，$`b=10`$ 已足够 |
| 每节点边数 $k$ | 100 | $k$ 增大性能提升，$`k>100`$ 趋于饱和 |
| 迭代步数 $q$ | 2~3 | $`q=0→2`$ 显著提升，$`q>2`$ 收敛 |

## 与现有方法的关系

### vs. 检索式模型（VQ-Rec）

RPG 具备生成式模型的两大独有优势：
1. 推理时间复杂度与物品数量**无关**
2. 将 Sub-item 级语义**显式**整合进学习目标

### vs. 生成式模型（TIGER）

| 对比维度 | TIGER | RPG |
|---------|-------|-----|
| 序列编码前向传播次数 | $`O(mathbf)`$ | $`O(1)`$ |
| 支持长语义 ID | ❌（性能下降+OOM） | ✅（最长64，性能提升） |
| 语义 ID 类型 | RQ-VAE（有序） | OPQ（无序） |
| 解码方法 | Beam Search | 图约束迭代传播 |
| 推理内存 | 随物品数增长 | **固定，与物品数无关** |

---

## 核心贡献总结

1. **OPQ 长语义 ID**：采用 Optimized Product Quantization 构造无序、长达 64 token 的语义 ID，突破 RQ 的顺序依赖限制

2. **MTP 并行预测**：基于 token 条件独立假设，用多 token 预测损失并行预测所有 digit，每个 digit 使用独立 Projection Head 映射到不同语义子空间

3. **高效 Logit 缓存**：预先缓存序列表示与所有 token 的点积，将 logit 计算复杂度从 $`O(Nmd)`$ 降至 $`O(Mmd + Nm)`$

4. **图约束迭代解码**：利用"相似语义 ID 的 logit 相近"的观察，构建语义相似图，通过迭代图传播高效探索高质量候选，推理复杂度与物品数量无关

5. **效果与效率双优**：平均 NDCG@10 比最强基线提升 **12.6%**，推理内存比 TIGER 少 **25×**，速度快 **15×**
