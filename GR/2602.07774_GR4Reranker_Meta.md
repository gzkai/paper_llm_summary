## 研究背景与动机

### 现有 LLM 推荐系统的三大局限

| 局限 | 描述 |
|------|------|
| **重排阶段被忽视** | 现有工作大多聚焦于检索（Retrieval）和排序（Ranking），对最终精化推荐结果的**重排（Re-ranking）**阶段研究严重不足 |
| **推理能力未充分利用** | LLM 通常以零样本或监督微调方式使用，未充分发挥其通过强化学习（RL）和高质量推理数据增强的推理能力 |
| **非语义 ID 的可扩展性瓶颈** | 工业系统中物品 ID 数量可达数十亿，直接使用非语义 ID 导致 LLM 词表膨胀，可扩展性极差 |

### GR2 的核心思路

> 将语义物品表示（Semantic ID）与世界知识、高质量推理能力相结合，通过三阶段训练流程实现推荐系统的生成式推理重排。

## 核心方法：GR2 三阶段训练流程

### 整体架构

```
GR2 三阶段训练流程
│
├── 阶段 1：语义 ID 分词器 + 中间训练（Tokenized Mid-Training）
│   ├── RQ-VAE 训练（残差量化变分自编码器）
│   ├── 五大码本均衡技术（EMA 更新、Random Last Level 等）
│   └── 多任务中间训练（SID 序列 + 世界知识交织）
│
├── 阶段 2：推理数据生成（Reasoning Trace Generation）
│   ├── 教师 LLM（Qwen3-32B）生成高质量分层推理链
│   ├── 两种生成策略：Targeted Sampling / Rejection Sampling
│   └── 六原则 Chat 格式训练样本构建
│
└── 阶段 3：推理能力激活（Reasoning Enablement for Re-Ranking）
    ├── SFT：在推理链上监督微调，解耦推理/排序损失权重
    └── RL：DAPO 算法 + 专为重排设计的条件奖励函数
```

---

### 阶段 1：语义 ID 分词器与中间训练

#### 1.1 Semantic ID（SID）与 RQ-VAE

给定物品文本特征 $x$，分词器将其映射为离散整数序列（语义 ID）：

$$\text{Tokenizer}(x) = (z_1, z_2, \ldots, z_K), \quad z_k \in \{1,\ldots,C_k\}$$

核心是**残差量化变分自编码器（RQ-VAE）**：
- 文本编码器生成连续嵌入：$\mathbf{h} = f_{\text{enc}}(x)$
- 残差初始化：$\mathbf{r}_1 = \mathbf{h}$
- 逐层最近邻量化：
  $$z_k = \arg\min_{j} \|\mathbf{r}_k - \mathbf{e}^{(k)}_j\|_2^2, \quad \mathbf{q}_k = \mathbf{e}^{(k)}_{z_k}, \quad \mathbf{r}_{k+1} = \mathbf{r}_k - \mathbf{q}_k$$
- 重建表示：$\hat{\mathbf{h}} = \sum_{k=1}^K \mathbf{q}_k$

**训练损失**：
$$\mathcal{L}(x) = \mathcal{L}_{\text{rec}} + \beta \sum_{k=1}^K \mathcal{L}_{\text{cb}}^{(k)} + \gamma \sum_{k=1}^K \mathcal{L}_{\text{com}}^{(k)}$$
其中重建损失 $\mathcal{L}_{\text{rec}} = \|\mathbf{h} - \hat{\mathbf{h}}\|_2^2$，码本损失 $\mathcal{L}_{\text{cb}}^{(k)} = \|\text{sg}[\mathbf{r}_k] - \mathbf{q}_k\|_2^2$，承诺损失 $\mathcal{L}_{\text{com}}^{(k)} = \|\mathbf{r}_k - \text{sg}[\mathbf{q}_k]\|_2^2$。

#### 1.2 五大码本均衡技术

| 技术 | 描述 | 效果 |
|------|------|------|
| **RQ-K-means 初始化** | 用 k-means++ 对残差聚类初始化码本，替代随机初始化 | 默认配置 |
| **EMA 码本更新** | 用指数移动平均更新码本，$\mathbf{e}^{(k)} \leftarrow \gamma \mathbf{e}^{(k)} + (1-\gamma)\tilde{\mathbf{e}}^{(k)}$，$\gamma \in [0.95, 0.99]$ | **关键技术**：平均唯一率从 31.32% → 97.50%（+66.2%） |
| **多样性损失** | $\mathcal{L}_{\text{div}} = \lambda_{\text{div}} \sum_k C_k \sum_j p_{k,j}^2$，鼓励码本均匀使用 | 边际效果（+0.6%） |
| **Dead Code Reset** | 追踪未使用码本条目，超过 $\tau$ 批次未使用则重置为重建误差最大的样本 | 轻微负效果（-4.0%） |
| **Random Last Level** | 推理时最后 $M$ 层随机分配，牺牲少量重建精度换取 ID 唯一性 | **关键技术**：平均唯一率从 54.06% → 82.76%（+28.7%） |

> **结论**：实现 ≥99% 唯一率必须同时启用 **EMA 更新** 和 **Random Last Level**；对比损失反而因鼓励相似物品共享码而降低唯一性（-13.2%）。

#### 1.3 对比损失（Contrastive Loss）

虽然对比损失会降低 SID 唯一率，但能编码物品共同交互历史，提升下游推荐质量：

$$\mathcal{L}_{\mathrm{ctr}}(x_i) = -\frac{1}{|\mathcal{P}(i)|} \sum_{p \in \mathcal{P}(i)} \log \frac{\exp(s(x_i, x_p))}{\exp(s(x_i, x_p)) + \sum_{n \in \mathcal{N}(i)} \exp(s(x_i, x_n))}$$

相似度为温度缩放余弦相似度：$s(x_i, x_j) = \tilde{\mathbf{h}}_i^\top \tilde{\mathbf{h}}_j / T$

#### 1.4 多任务中间训练（Mid-Training）

受 OneRec-Think（快手）和 Google PLUM 启发，关键思路是：
- **将 SID 与自然语言 token 在同一序列中交织**
- 通过**下一 token 预测目标**优化 SID 嵌入表
- 支持单任务（仅序列推荐）和多任务训练

### 阶段 2：推理数据生成

#### 2.1 两种推理链生成策略

**Targeted Sampling（目标导向采样）**：
$$\tau \sim P_\theta(\cdot | \mathcal{P}_{\text{targeted}}([\mathbf{s}_{v_1},...,\mathbf{s}_{v_k}], [\mathbf{s}_{y_1},...,\mathbf{s}_{y_c}], \mathbf{s}_{v_{n+1}}))$$
- 将目标物品 $\mathbf{s}_{v_{n+1}}$ 包含在 Prompt 中，要求 LLM 解释为何用户会选择该物品
- **优点**：始终生成目标物品的合理化解释；**缺点**：可能产生无真实信念的幻觉

**Rejection Sampling（拒绝采样）**：
$$(\tau, \mathbf{\hat{s}}_{y_c}) \sim P_\theta(\cdot | \mathcal{P}_{\text{rejection}}([\mathbf{s}_{v_1},...,\mathbf{s}_{v_k}], [\mathbf{s}_{y_1},...,\mathbf{s}_{y_c}])) \quad \text{s.t.} \quad \mathbf{\hat{s}}_{y_c} = \mathbf{s}_{v_{n+1}}$$
- 不提供目标物品，持续查询 LLM 预测，直到预测与真实目标一致
- 通过迭代验证确保推理真实性

#### 2.2 六原则 Chat 格式训练样本

| 原则 | 描述 |
|------|------|
| **P1 角色定义** | 系统消息设定"专业电商推荐专家"角色，明确重排目标 |
| **P2 SID 锚定物品表示** | 历史物品和候选物品均附带 SID token（`<|sid_begin|>...<|sid_end|>`） |
| **P3 丰富物品元数据** | 每个物品附带标题和类目层级，提供多模态上下文 |
| **P4 统一物品格式** | 历史物品和候选物品共享 SID + 标题 + 类目结构 |
| **P5 链式推理轨迹** | 助手回复包含逐步推理，明确引用物品 SID |
| **P6 结构化 JSON 输出** | 回复格式化为含推理说明和排序列表的 JSON 对象 |

**五大 Prompt 设计原则**：
- P1：具体的系统角色和重排任务定义
- P2：协作上下文呈现（历史 + 候选同时提供）
- P3：领域知识启动（如洗发水→护发素→造型产品的购买模式）
- P4：以 SID 引用物品作为输出约束（防止幻觉）
- P5：结构化多步推理格式（广泛模式识别→互补产品类型→匹配具体候选）

### 阶段 3：推理能力激活与强化学习

#### 3.1 SFT 解耦损失

$$\mathcal{L}_{\text{SFT}} = -\lambda_r \sum_{i=1}^{M} \log P(r_i | \mathcal{P}, r_{<i}) - \lambda_o \sum_{j=1}^{T} \log P(o_j | \mathcal{P}, \tau, o_{<j})$$

其中 $\lambda_r < \lambda_o$，推理 token 权重低于排序 token 权重，在推理流畅性和排序精度之间取得平衡。

#### 3.2 强化学习奖励设计

**排序奖励**（衡量目标物品被提升的幅度）：
$$R_{\text{rank}} = \frac{r^{\mathcal{D}}_{\mathbf{s}_{v_{n+1}}} - r^{\mathbf{o}}_{\mathbf{s}_{v_{n+1}}}}{|\mathcal{D}|}$$

其中 $r^{\mathcal{D}}$ 和 $r^{\mathbf{o}}$ 分别为目标物品在预排和重排列表中的排名。

**条件格式奖励**（防止奖励 Hacking）：

$$R = \begin{cases} R_{\text{rank}} + \alpha R_{\text{fmt}}, & \text{if } R_{\text{rank}} > 0 \text{ or } r^{\mathcal{D}}_{\mathbf{s}_{v_{n+1}}}=1 \\ R_{\text{rank}}, & \text{otherwise} \end{cases}$$

> **关键洞察**：朴素地将排序奖励和格式奖励相加会导致**奖励 Hacking**——模型倾向于保持原始排序来获取格式奖励。条件格式奖励只在实际提升排名或目标已在第1位时才给予格式奖励，有效缓解此问题。

#### 3.3 DAPO 算法

基于 GRPO 改进的 DAPO（Decoupled Clip and Dynamic sAmpling Policy Optimization）：

$$\mathcal{J}_{\text{DAPO}}(\theta) = \mathbb{E} \left[ \frac{1}{\sum_{i=1}^G |o_i|} \sum_{i=1}^G \sum_{t=1}^{|o_i|} \min\left(r_{i,t}(\theta) \hat{A}_{i,t}, \text{clip}(r_{i,t}(\theta), 1-\varepsilon_{\text{low}}, 1+\varepsilon_{\text{high}}) \hat{A}_{i,t}\right) \right]$$

其中优势估计 $\hat{A}_{i,t} = \frac{R_i - \text{mean}(\{R_i\})}{\text{std}(\{R_i\})}$

**DAPO 相比 GRPO 的改进**：
- **解耦裁剪范围**：$\varepsilon_{\text{low}}$ 和 $\varepsilon_{\text{high}}$ 分别控制上下裁剪（Clip-Higher 策略）
- **动态采样**：过滤准确率为 0 或 1 的 Prompt，避免零梯度，提升样本效率
- **解决熵崩溃问题**：防止 GRPO 训练中的策略退化

## 实验结果

### SID 唯一率消融

**关键发现**：
- EMA 更新是防止码本崩溃的最关键技术（+66.2%）
- Random Last Level 是实现 ≥99% 唯一率的必要条件（+28.7%）
- 两者同时启用可达到 **99.98%** 唯一率
- 对比损失反而降低唯一率（-13.2%），但提升推荐质量

### 最佳 SID 配置（Amazon Beauty，TIGER 风格模型）

最优配置：**Dead Code Reset + EMA Update + Random Last Level + Contrastive Loss**

| 配置 | Recall@5 | Recall@10 | NDCG@5 | NDCG@10 |

| 无 Diversity Loss，有 Dead Code Reset，有 EMA，有 Random，有 Contrastive | **0.0227** | **0.0367** | **0.0145** | **0.0190** |

## 核心发现与洞察

### 发现一：推理链质量至关重要

在 Beauty 数据集上，**RL-rejection-KP** vs **RL-zeroshot**：
- Recall@1：+3.00%
- Recall@5：+3.30%
- NDCG@5：+2.60%

结构化的拒绝采样推理链提供了更强的学习信号。

### 发现二：RL 是将推理能力转化为重排提升的必要条件

SFT 虽然能学习高质量推理链，但并不能始终转化为更好的重排性能，在某些情况下甚至会降低 Recall@1。这揭示了**推理质量与排序最优性之间的不匹配**，强调了 RL 显式优化推理行动空间的必要性。

### 发现三：奖励 Hacking 是重排场景的特殊挑战

LLM 倾向于通过**保留原始排序**来获取格式奖励——这是重排场景特有的奖励 Hacking 形式。条件格式奖励有效缓解了这一问题。

### 发现四：对比损失的双重效应

对比损失会降低 SID 唯一率（-13.2%），但能编码物品共同交互历史，**一致性地提升推荐性能**。需要在唯一率和语义质量之间权衡。

## 核心贡献总结

1. **重排阶段的 LLM 设计原则**：首次系统研究并建立针对推荐系统重排阶段的 LLM 设计原则，填补了现有工作的空白。

2. **高唯一率语义 ID 生成**：提出五大互补技术（尤其是 EMA 更新 + Random Last Level），实现 ≥99% 的 SID 唯一率；通过多任务中间训练将 SID 与世界知识对齐。

3. **高质量分层推理链生成**：专为重排场景设计 Prompt，结合拒绝采样过滤噪声，生成有据可查的 SID 锚定推理链。

4. **DAPO + 条件可验证奖励**：将 DAPO 算法适配到重排任务，设计条件格式奖励防止保序奖励 Hacking，提供可扩展的奖励驱动监督。

5. **全面实验验证**：在两个真实数据集上超越 SOTA OneRec-Think，消融研究验证了推理和 RL 的关键贡献。
