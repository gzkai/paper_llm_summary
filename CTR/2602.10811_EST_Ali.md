## 研究背景与动机

### 工业 CTR 预测的挑战

工业 CTR 预测模型的输入极为复杂，包含三类特征：

- **非行为特征** $\mathcal{N}$：用户画像、候选物品属性等，数量少但信息密度高
- **用户行为序列** $\mathcal{B}_u$：短期历史行为（截断至 300 条）
- **候选相关行为序列** $\mathcal{B}_c$：从终身行为（5,000 条）中通过 GSU 检索的 Top-50 与候选物品最相关的子序列

此外，行为中的物品还包含多模态内容特征（图像、文本）。

整体输入序列规模可达 $`10^3`$ 个 token，在**毫秒级延迟**约束下进行规模化扩展是核心难题。

### 现有方法的两大范式及其缺陷

| 范式 | 代表工作 | 做法 | 缺陷 |
|------|---------|------|------|
| **层级建模（Hierarchical）** | WuKong, RankMixer, HiFormer | 先用 DIN/LONGER 将行为序列压缩为固定长度向量，再与非行为特征联合建模 | **信息瓶颈**：早期聚合丢弃了 token 级细粒度信号，限制了规模化收益 |
| **统一建模（Unified，部分）** | MTGR, OneTrans | 将 $`\mathcal{B}_u`$ 和 $`\mathcal{N}`$ 放入单一序列，通过候选共享计算降低延迟 | **部分统一**：候选相关行为 $`\mathcal{B}_c`$ 无法跨候选共享，仍被 Pooling 压缩，造成信息损失 |

### 两个关键洞察

通过对注意力矩阵的分析，作者发现 CTR 输入与 LLM token 序列存在根本性区别：

**洞察一：信息密度不对称（Asymmetry in Information Density）**

将行为 token $`\mathbf{B}`$ 和非行为 token $`\mathbf{N}`$ 拼接后做全自注意力，可视化注意力矩阵 $`\mathbf{A}`$，发现：

- $`\langle \mathbf{N}, \mathbf{B} \rangle`$ 块（非行为作 Query，行为作 Key/Value）：注意力分布**多样**，有效秩（effective rank）**最高**
- $`\langle \mathbf{B}, \mathbf{B} \rangle`$、$`\langle \mathbf{B}, \mathbf{N} \rangle`$、$`\langle \mathbf{N}, \mathbf{N} \rangle`$ 块：出现**秩坍缩**（rank collapse），注意力分布趋于均一

**块级掩码实验验证**：

| 掩码块 | $`\Delta`$AUC | $`\Delta`$GAUC |
|--------|-------------|--------------|
| $`\langle \mathbf{B}, \mathbf{B} \rangle`$ | -0.01% | -0.00% |
| $`\langle \mathbf{B}, \mathbf{N} \rangle`$ | -0.03% | -0.02% |
| $`\langle \mathbf{N}, \mathbf{N} \rangle`$ | -0.03% | -0.02% |
| **$`\langle \mathbf{N}, \mathbf{B} \rangle`$** | **-0.20%** | **-0.20%** |

结论：**只有 $`\langle \mathbf{N}, \mathbf{B} \rangle`$ 交叉注意力是关键，其余三块均为冗余计算**。

**洞察二：模态特定先验（Modality-Specific Priors）**

对比三种多模态融合策略：

| 策略 | $`\Delta`$AUC | $`\Delta`$GAUC |
|------|-------------|--------------|
| Side Information（拼接内容特征为属性） | +0.44% | +0.89% |
| Semantic ID（内容特征离散化为聚类 ID） | +0.47% | +0.64% |
| **SimTier（内容相似度作为先验信号）** | **+1.18%** | **+1.58%** |
| SimTier + Side Information | +0.95% | +1.47% |

结论：**内容特征作为相似度先验（而非直接 token 注入）效果最佳**；两种策略混合反而下降，说明不同范式难以协同优化。

## 核心方法：EST

**EST**（**E**fficiently **S**calable **T**ransformer）基于上述两个洞察，实现了**完全统一建模**（Fully Unified Modeling）——所有原始输入无损地组织为单一序列，无需任何早期聚合。

### 整体架构

```
输入：非行为特征 N，用户行为 B_u，候选相关行为 B_c，冻结内容特征 M_u, M_c
         ↓ 特征特定 Tokenizer
非行为 token 序列 N ∈ ℝ^{L_N × d}
行为 token 序列 B = [B_u, B_c] ∈ ℝ^{L_B × d}
         ↓ S 层 EST Block（LCA + CSA 并行）
N^(S), B^(S)
         ↓ Mean Pooling + Concat
X^(S) = [N^(S), Mean(B^(S))]
         ↓ MLP
预测点击概率 ŷ_{u,c}
```

每层 EST Block 的计算：

```math
\mathbf{N}^{(s)} \leftarrow \text{LCA}\left(\text{Norm}(\mathbf{N}^{(s-1)}), \text{Norm}(\mathbf{B}^{(s-1)})\right) + \mathbf{N}^{(s-1)}
```

```math
\mathbf{B}^{(s)} \leftarrow \text{CSA}\left(\text{Norm}(\mathbf{B}^{(s-1)})\right) + \mathbf{B}^{(s-1)}
```

FFN 设计上，非行为 token 使用**每 token 独立的 PerToken-FFN**，行为 token 使用**共享的 Shared-FFN**。

---

### 模块一：特征 Tokenization

**非行为特征 Tokenization**：每个原始非行为特征 $`n_i \in \mathcal{N}`$ 使用特征特定的 MLP 映射为 token $`\mathbf{n}_i \in \mathbb{R}^d`$，**不做聚合**：

```math
\mathbf{n}_i = \text{MLP}_i(n_i), \quad \mathbf{N} = [\mathbf{n}_1, \ldots, \mathbf{n}_{L_N}]
```

设计亮点：**即插即用的特征扩展**——新特征可通过 warm-start 加入，无需改变架构或重新初始化参数。

**行为特征 Tokenization**：对用户行为序列 $`\mathcal{B}_u`$ 和候选相关序列 $`\mathcal{B}_c`$ 分别使用序列特定 MLP：

```math
\mathbf{B}_\gamma = [\text{MLP}_\gamma(b_{\gamma,1}), \ldots, \text{MLP}_\gamma(b_{\gamma,L_{B_\gamma}})], \quad \mathbf{B} = [\mathbf{B}_u, \mathbf{B}_c]
```

**内容特征映射**：通过 lookup 获取每个行为物品的预训练内容特征 $`\mathbf{m} \in \mathbb{R}^{d_M}`$，**训练期间冻结**（保留预训练语义，避免 ID 嵌入与高维稠密向量联合优化的不稳定性）。

---

### 模块二：轻量级交叉注意力（LCA）

**核心思想**：只保留最关键的 $`\langle \mathbf{N}, \mathbf{B} \rangle`$ 交叉注意力，丢弃其余三块冗余交互。

非行为 token $`\mathbf{N}`$ 作 Query，行为 token $`\mathbf{B}`$ 作 Key/Value：

```math
(\mathbf{Q}_\mathbf{N}, \mathbf{K}_\mathbf{B}, \mathbf{V}_\mathbf{B}) = \left(\phi(\mathbf{N}, \mathbf{W}^Q),\ \mathbf{B}\mathbf{W}^K,\ \mathbf{B}\mathbf{W}^V\right)
```

```math
\text{LCA}(\mathbf{N}, \mathbf{B}) = \phi\left(\text{Softmax}\left(\frac{\mathbf{Q}_\mathbf{N}\mathbf{K}_\mathbf{B}^\top}{\sqrt{d}}\right)\mathbf{V}_\mathbf{B},\ \mathbf{W}^O\right)
```

**参数设计**：
- $`\mathbf{W}^Q, \mathbf{W}^O \in \mathbb{R}^{L_N \times d \times d}`$：**每个非行为 token 独立的投影权重**，捕捉各特征的独特信号
- $`\mathbf{W}^K, \mathbf{W}^V \in \mathbb{R}^{d \times d}`$：**所有行为 token 共享的投影权重**，保证计算可扩展性

**工业部署优势——用户-候选解耦计算**：

用户行为序列 $`\mathbf{B}_u`$ 与候选无关，其 Key/Value 投影 $`\mathbf{B}_u\mathbf{W}^K`$ 和 $`\mathbf{B}_u\mathbf{W}^V`$ 可**计算一次，在同一请求的所有候选间复用**，显著降低推理延迟。

**计算复杂度对比**：

| 操作 | 标准自注意力 | LCA |
|------|-------------|-----|
| 投影 | $`O((L_N+L_B)d^2)`$ | $`O(L_N d^2 + L_B d^2)`$ |
| 注意力 | $`O((L_N+L_B)^2 d)`$ | $`O(L_N L_B d)`$（双线性，$`L_B \gg L_N`$ 时远小于二次） |

---

### 模块三：内容稀疏注意力（CSA）

**核心思想**：利用冻结内容特征的相似度矩阵作为**训练无关的注意力矩阵**，引导行为序列内部的 token 交互。

**算法流程**：

1. 计算行为序列 $`\mathcal{B}_\gamma`$ 的内容相似度矩阵：

```math
\mathbf{G}_\gamma = \mathbf{M}_\gamma \mathbf{M}_\gamma^\top \in \mathbb{R}^{L_{B_\gamma} \times L_{B_\gamma}}
```

2. 全稠密注意力（朴素版）：

```math
\mathbf{O}_{\mathbf{B}_\gamma} = \mathbf{G}_\gamma \mathbf{B}_\gamma
```

3. **Top-K 稀疏化**（降低复杂度）：对 $`\mathbf{G}_\gamma`$ 每行保留最相似的 $K$ 个邻居，复杂度从 $`O(L_{B_\gamma}^2 d)`$ 降至 $`O(L_{B_\gamma} K d)`$（线性）

4. 稀疏加权聚合（$`K=5`$）：

```math
\mathbf{O}_{\mathbf{B}_\gamma} = \text{Sum}(\text{Gather}(\mathbf{B}_\gamma, \mathbf{\Omega}_\gamma) \odot \mathbf{E}_\gamma,\ \text{axis}=1)
```

其中 $`\mathbf{\Omega}_\gamma`$ 为 Top-K 索引，$`\mathbf{E}_\gamma`$ 为对应相似度值。

**三大工程优势**：
1. **并行预计算**：内容相似矩阵和 Top-K 稀疏化可与其他特征处理并行，各 EST 层复用，几乎零延迟开销
2. **用户-候选解耦**：$`\mathbf{B}_u`$ 和 $`\mathbf{B}_c`$ 独立处理，用户侧计算跨候选共享
3. **线性复杂度**：$`K=5 \ll L_{B_\gamma}`$，复杂度线性于序列长度

**CSA 的本质**：实现了 ID 特征捕捉的**共现关系**与内容特征隐含的**语义关系**的互补融合。

---

### 训练与部署策略

**初始化策略**：工业生产模型已在**数万亿样本**上训练多年，难以从头超越。EST 采用：
- **稀疏参数**（Embedding 表）：从历史生产模型初始化
- **稠密参数**（Transformer 权重）：从头随机初始化

在数千亿样本上训练后，EST 超越生产模型 **0.74% GAUC**。

**多轮训练策略（解决 one-epoch 限制）**：

推荐系统常见问题：第二轮训练开始时因过拟合导致性能下降。EST 采用**异步多轮范式**：

```math
\boldsymbol{\theta}_s^{(e)} \xleftarrow{\text{init}} \boldsymbol{\theta}_s^{(0)}, \quad \forall e \geq 1
```

```math
\boldsymbol{\theta}_d^{(e)} \xleftarrow{\text{init}} \begin{cases} \text{random}, & e = 1 \\ \boldsymbol{\theta}_d^{(e-1)}, & e \geq 2 \end{cases}
```

- 每轮重置稀疏参数到原始生产模型初始化 $`\boldsymbol{\theta}_s^{(0)}`$
- 稠密参数跨轮继承（仅第一轮随机初始化）

此策略额外带来 **+0.44% GAUC** 提升。

## 实验结果

### 数据集

| 数据集 | 规模 | 用途 |
|--------|------|------|
| 淘宝生产日志（标准） | 10 亿+ 用户曝光，6 天 | 主要离线评估 |

| 层级建模 | HiFormer | 0.6928 | +0.49% | 0.6493 | +0.53% | 113.78 | 14.17 |

1. EST 以 18.48 GFLOPs（比 OneTrans-F 少 **65%**）实现最优 GAUC +0.87%

3. 候选相关行为序列的充分建模至关重要（OneTrans-D/F 均优于 OneTrans）

### 消融实验（RQ2）

| 方法 | FA | LCA | CSA | $`\Delta`$AUC | $`\Delta`$GAUC | $`\Delta`$GFLOPs |
|------|----|----|-----|-------------|--------------|----------------|
| Full-Attention | ✓ | - | - | 基准(0.6951) | 基准(0.6506) | 基准(66.27) |
| EST（仅LCA） | - | ✓ | - | -0.01% | -0.03% | **-72.23%** |
| **EST（LCA+CSA）** | - | ✓ | ✓ | **+0.17%** | **+0.14%** | **-72.11%** |

- LCA：以 -72.23% 计算量换取几乎无损的性能（仅 -0.03% GAUC）
- CSA：在 LCA 基础上额外提升 **+0.14% GAUC**，计算开销可忽略
- LCA + CSA 组合：**性能超越全自注意力，计算量减少 72%**

### 规模化定律（RQ3）

验证 EST 在两个维度上的幂律规模化行为：

**计算量维度**（推理 GFLOPs，范围 5~50）：

```math
\Delta\text{GAUC} = 0.61 \times C^{0.12} \quad \text{（增加深度）}
```

```math
\Delta\text{GAUC} = 0.68 \times C^{0.10} \quad \text{（增加宽度）}
```

**模型参数维度**（稠密参数，范围 30M~300M）：

```math
\Delta\text{GAUC} = 0.46 \times P^{0.14} \quad \text{（增加深度）}
```

```math
\Delta\text{GAUC} = 0.63 \times P^{0.08} \quad \text{（增加宽度）}
```

**关键发现**：
- $`\Delta`$GAUC 随计算量和参数量单调增长，符合幂律规律
- **增加深度的增益斜率高于增加宽度**：增加层数能挖掘更复杂的特征交互

### 在线 A/B 测试（RQ4）

部署于**淘宝展示广告系统**（QuanZhanTui 全站推广产品）：

| 场景 | CTR | RPM |
|------|-----|-----|

## 与现有方法的关系

### vs. 层级建模（HiFormer, RankMixer）

层级建模通过 DIN/LONGER 早期压缩行为序列，丢失 token 级细粒度信号。EST 通过 LCA 的交叉注意力设计，在**不压缩行为序列**的前提下实现等效甚至更低的计算量。

### vs. 部分统一建模（OneTrans, MTGR）

OneTrans/MTGR 对候选相关行为 $`\mathcal{B}_c`$ 仍需 Pooling 压缩（因其不可跨候选共享）。EST 通过 LCA 的**用户-候选解耦设计**，支持 $`\mathcal{B}_c`$ 完整输入，同时通过 CSA 的**独立序列处理**维持候选共享计算。

### vs. 全自注意力（Full-Attention）

全自注意力包含大量冗余的 $`\langle \mathbf{B}, \mathbf{B} \rangle`$ 等块。EST 通过 LCA 精准保留关键的 $`\langle \mathbf{N}, \mathbf{B} \rangle`$ 交叉注意力，**性能反而更优（+0.14% GAUC），计算量减少 72%**。

## 核心贡献总结

1. **两个基础洞察**：识别 CTR 输入与 LLM token 的根本区别——信息密度不对称（$`\langle \mathbf{N}, \mathbf{B} \rangle`$ 是关键交互）和模态特定先验（内容相似度先验 > 直接 token 注入），为高效可扩展架构设计提供理论依据

2. **LCA（轻量级交叉注意力）**：仅保留 $`\langle \mathbf{N}, \mathbf{B} \rangle`$ 交叉注意力，非行为 token 使用独立投影权重，行为 token 使用共享权重；支持用户-候选解耦计算，将注意力复杂度从 $`O((L_N+L_B)^2 d)`$ 降至 $`O(L_N L_B d)`$

3. **CSA（内容稀疏注意力）**：利用冻结内容特征的相似度矩阵作为训练无关的注意力权重，Top-K 稀疏化将复杂度降至 $`O(L_{B_\gamma} K d)`$（线性），实现 ID 共现关系与内容语义关系的互补

4. **完全统一建模**：所有原始输入（$`\mathcal{N}, \mathcal{B}_u, \mathcal{B}_c`$）无损组织为单一序列，消除早期聚合的信息瓶颈

5. **高效幂律规模化**：在淘宝生产系统验证了稳定的幂律规模化关系，在线 A/B 测试 RPM +3.27%，CTR +1.22%
