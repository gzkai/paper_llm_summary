## 研究背景与动机

### 核心问题

现代大规模推荐系统（如 Meta 广告系统）需要同时处理两类异构特征：
- **非序列特征（Non-Sequential）**：用户画像、广告元数据、上下文信号等，反映**静态显式**偏好；
- **序列特征（Sequential）**：用户历史行为序列（点击、转化、曝光），反映**动态隐式**偏好。

### 为什么推荐系统难以建立 Scaling Laws？

LLM 的 Scaling Laws（Kaplan 等）已被充分验证，但推荐系统的 Scaling Laws 至今是开放挑战，尤其是**同时建模序列与非序列特征**的场景。

论文识别出**缩放效率（Scaling Efficiency）**低是核心障碍，由两大瓶颈构成：

1. **低效模块（Inefficient Modules）**：
   - 现有模型 MFU（Model FLOPs Utilization）仅为 **3–15%**，而 LLM 可达 **40–60%**；
   - 根本原因：异构特征空间导致嵌入维度小、张量形状不规则、内存受限操作（memory-bound）；
   - 典型案例：InterFormer 的 PFFN 模块因小矩阵、不可融合的反向传播而严重限制 GPU 利用率。

2. **低效计算资源分配（Inefficient Computation Resource Allocation）**：
   - 对所有组件等量扩展会产生收益递减；
   - 不同层、不同事件类型（点击 vs 曝光）对计算模式的需求差异巨大；
   - 现有方法缺乏智能的资源分配策略。

### Scaling Efficiency 的数学定义

论文将 Scaling Laws 建模为：

```math
\text{NE}(C) = \text{NE}_0 - \eta \cdot \log(C / C_0)
```

其中 $C$ 为总计算量（FLOPs），$`\eta > 0`$ 为**缩放系数**，越大表示每单位 log 计算量带来更多 NE 改善。

**Scaling Efficiency** 定义为相对于基线的缩放系数之比：

```math
\text{Scaling Efficiency} = \frac{\eta}{\eta_{\text{baseline}}} = \frac{\Delta \text{NE} / \log(C / C_0)}{\eta_{\text{baseline}}}
```

缩放效率由两个因素决定：
- **算法有效性**：每 FLOP 带来的 NE 改善（依赖模型架构与训练算法）；
- **计算效率**：硬件实际利用率，即 MFU（低 MFU = GPU 等待内存传输，浪费算力）。

## 核心方法：Kunlun 架构

Kunlun 在 InterFormer 框架基础上，通过**模型-效率协同设计（Model-Efficiency Codesign）**系统性解决上述瓶颈，分为**低层模块优化**和**高层计算重分配**两个层次。

### 整体架构概览

```
输入: 非序列特征 X^(0) + 序列特征 S^(0)
       ↓ 特征预处理（ROTE 时序旋转位置编码）
       ↓
  ┌──────────────────────────────────────────────────┐
  │  Layer l = 0, 1, ..., L-1                        │
  │  ┌─────────────────────────────────────────────┐ │
  │  │  Kunlun Interaction Block                   │ │
  │  │  ├── Weight Generation (非序列→PFFN权重)     │ │
  │  │  ├── HSP (序列→紧凑摘要)                    │ │
  │  │  └── Global Interaction (混合Wukong专家)    │ │
  │  └─────────────────────────────────────────────┘ │
  │  ┌─────────────────────────────────────────────┐ │
  │  │  Kunlun Transformer Block                   │ │
  │  │  ├── GDPA-enhanced PFFN (个性化序列变换)     │ │
  │  │  └── Sliding Window MHA (局部序列建模)       │ │
  │  └─────────────────────────────────────────────┘ │
  └──────────────────────────────────────────────────┘
       ↓
  最终预测: ŷ = σ(MLP(X^(L)))
```

### 特征预处理：Rotary Temporal Embeddings (ROTE)

Kunlun 将 LLM 中的旋转位置编码（RoPE）扩展为**旋转时序嵌入（ROTE）**，专门针对推荐序列的时间特性：

- **动机**：推荐序列中，时间间隔（而非位置）才是关键信号——昨天的点击与上个月的点击有根本差异，即便它们在序列中相邻；
- **实现**：计算相邻事件的时间间隔，通过对数缩放编码 $`\tau_t = \log(1 + \Delta t / \tau_{\text{scale}})`$，融入旋转矩阵：

```math
\text{ROTE}(\mathbf{x}, t, \tau_t) = \mathbf{R}_{t,\tau_t} \mathbf{x}
```

其中 $`\mathbf{R}_{t,\tau_t}`$ 的元素为 $`\cos(t\theta_i + \tau_t\phi_i)`$ 和 $`\sin(t\theta_i + \tau_t\phi_i)`$，同时编码**序列顺序**和**时间邻近性**。

---

### 低层优化一：GDPA（广义点积注意力）

#### 原始 PFFN 的问题

InterFormer 的 PFFN 公式为：

```math
\text{PFFN}(\mathbf{X}_{\text{sum}}^{(l)}, \mathbf{S}^{(l)}) = f(\mathbf{X}_{\text{sum}}^{(l)}) \mathbf{S}^{(l)}
```

其中 $`f(\mathbf{X}_{\text{sum}}^{(l)})`$ 为两层 MLP 生成的变换权重。

**问题**：小矩阵运算 memory-bound，激活函数不可融合，反向传播中有多个不可融合的矩阵乘，导致 MFU 极低。

#### GDPA 重构

**关键洞察**：将 PFFN 变换视为一种交叉注意力——序列 token 作为 Query，非序列上下文生成 Key 和 Value：

对每个注意力头 $`h \in \{1, \dots, H\}`$：

```math
\text{GDPA}_h(\mathbf{Q},\mathbf{K},\mathbf{V})=\text{Activation}_h\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\tau}\right)\mathbf{V}
```

- $`\mathbf{Q} = \mathbf{S}^{(l)}`$（序列 token 作为查询）
- $`\mathbf{K} = \mathbf{w}_1^{(h)}(\mathbf{X}_{\text{sum}}^{(l)})`$，$`\mathbf{V} = \mathbf{w}_2^{(h)}(\mathbf{X}_{\text{sum}}^{(l)})`$（非序列上下文生成 K/V）
- $`\tau = \text{maxlen}(seq)`$（温度参数，实验中优于 $d$ 或 $`\sqrt{d}`$）
- $`\text{Activation}_h`$：每个头可使用不同激活函数（提供集成多样性）

**工程实现**：
- 参照 FlashAttention 风格，块式内存高效执行，避免在反向传播中实例化大型中间张量；
- 针对推荐系统特殊性（短小非对称 K/V、可变长度 jagged 输入、非 softmax 激活）重新设计 kernel pipeline；
- 简化 warp 专用化，外循环级软件流水线，SFU 瓶颈优化；
- 最终 PFFN 块 MFU 提升 **6×**。

**额外收益**：GDPA 引入残差连接，支持 PFFN 跨层堆叠（原始 PFFN 不可堆叠），带来额外 NE 收益。

---

### 低层优化二：HSP（层次种子池化）

#### 原始 PMA 的问题

InterFormer 使用 PMA（Pooling by Multihead Attention）做序列摘要：

```math
\text{PMA}(\mathbf{S}) = \text{MHA}(\mathbf{Q}_{\text{learnable}}, \mathbf{S}, \mathbf{S})
```

使用随机初始化的可学习查询，初始化质量差，训练不稳定。

#### HSP 三阶段设计

**Stage 1：种子嵌入初始化**
- 定义可学习种子嵌入 $`\mathbf{E}_{\text{seed}} \in \mathbb{R}^{n_{\text{seeds}} \times d}`$，$`n_{\text{seeds}} > n_{\text{tokens}}`$（过完备表示）
- 种子在 batch 内共享

**Stage 2：种子级注意力**

```math
\mathbf{H}_{\text{seed}} = \text{MHA}(\text{Norm}(\mathbf{E}_{\text{seed}}), \mathbf{S}, \mathbf{S}) \in \mathbb{R}^{B \times n_{\text{seeds}} \times d}
```

**Stage 3：SumKronLinear 参数高效压缩**

将 $`n_{\text{seeds}}`$ 个种子向量压缩为 $`n_{\text{tokens}}`$ 个输出 token：

```math
\mathbf{Y}_b = \sum_{i=1}^{k} \mathbf{Z}_i^\top \mathbf{X}_b \mathbf{W}_i, \quad \mathbf{Z}_i \in \mathbb{R}^{S \times T}, \mathbf{W}_i \in \mathbb{R}^{D \times D}
```

**优势**：
- **参数效率**：参数从 $`O(S \cdot D \cdot T \cdot D)`$ 降至 $`O(k \cdot (S \cdot T + D^2))`$，典型配置（$`S{=}256, T{=}32, D{=}384, k{=}8`$）节省 **14×** 参数；
- **跨维度表达力**：Rank-$`k`$ Kronecker 结构捕获序列与嵌入维度的联合相关性（优于假设独立的可分离分解）；
- **可扩展性**： $D$ 增大时参数节省比例增加，大规模场景优势更显著。

---

### 低层优化三：滑动窗口注意力（Sliding Window Attention）

**动机**：推荐序列中存在强**局部性偏置**——近期交互的预测价值远高于久远历史，因此无需全局 $`O(T^2)`$ 注意力。

```math
\text{Attention}(Q_t, K, V) = \text{softmax}\left(\frac{Q_t K_{[t-w:t+w]}^{\top}}{\sqrt{d}}\right) V_{[t-w:t+w]}
```

将复杂度从 $`O(T^2)`$ 降至 $`O(Tw)`$，实现：
- QPS 提升 **31.1%**
- FLOPs 减少 **29.5%**
- NE 无损失（局部依赖足以捕获序列模式）

窗口大小 $w$ 可通过事件级个性化按事件类型独立配置。

---

### 高层优化一：CompSkip（计算跳过）

**动机**：类似 LLM 中的层交错策略（Gemma 2）和层冗余发现（ShortGPT），深度推荐模型中不同层对不同计算模式的需求不同。

**交替层模式**：

| 层类型 | 跳过操作 | 执行操作 |
|-------|---------|---------|
| 偶数层（0, 2, 4, ...） | 跳过 Self-Attention | 执行 HSP（新鲜序列摘要）|
| 奇数层（1, 3, 5, ...） | 跳过 HSP（复用前层摘要）+ PFFN | 执行 Self-Attention |

**设计逻辑**：每层要么专注局部精炼（Self-Attention），要么专注全局摘要（HSP），避免重复计算，保持表达力。

**效率收益**：
- FLOPs 减少 **43.1%**
- QPS 提升 **35%**
- NE 无损失

---

### 高层优化二：事件级个性化（Event-Level Personalization）

**动机**：不同事件类型（点击、转化、曝光）的重要性和特性各异，应按比例分配计算资源。

对每个事件类型 $e$ 独立配置：

| 配置项 | 含义 |
|-------|------|
| $`d_{\text{model}}^{(e)}`$ | 模型维度（控制容量） |
| $`n_{\text{heads}}^{(e)}`$ | 注意力头数（控制多面学习） |
| $`n_{\text{tokens}}^{(e)}`$ | 传入非序列交互的摘要 token 数 |
| $`L^{(e)}`$ | 处理层数（控制深度） |
| $`w^{(e)}`$ | 滑动窗口大小（控制感受野） |

**示例配置**：
- 点击序列：$`d_{\text{model}}=256, n_{\text{heads}}=8, n_{\text{tokens}}=32, L=3, w=100`$
- 曝光序列：$`d_{\text{model}}=128, n_{\text{heads}}=4, n_{\text{tokens}}=16, L=2, w=50`$

**效率收益**：QPS 提升 **13%**，FLOPs 减少 **11%**，NE 提升 **0.02%**（高价值事件获得更多资源）。

---

### 高层优化三：混合 Wukong 专家全局交互（Global Interaction）

**架构**：Global Interaction 模块采用**混合专家（MoE）**架构，$M$ 个 Wukong 模块并行处理不同特征子集：

```math
\mathbf{H}_i^{(l)} = \text{Wukong}_i(\mathbf{X}_{\text{global}, i}^{(l)})
```

其中 $`\mathbf{X}_{\text{global}}^{(l)} = \text{Concat}(\mathbf{X}_{\text{sum}}^{(l)}, \mathbf{H}_{\text{summary}}^{(l)})`$ 包含非序列摘要和序列摘要。

每个 Wukong 专家内部包含 DOT product（线性交互）和深度交互网络（层次交互），作为集成单元捕获不同交互模式。

**双向扩展**：
- **水平扩展（专家并行）**：增加每层专家数 $M$ → 宽度方向 NE 收益；
- **垂直扩展（层堆叠）**：增加层数 $L$ → 深度方向 NE 收益，满足幂律缩放；

---

### 完整前向算法

```
输入: X^(l), S^(l), H_prev（前层HSP摘要）, 层配置
输出: X^(l+1), S^(l+1), H_curr

=== Kunlun Interaction Block ===
1. W^(l) ← WeightGen(Summarize(X^(l)))    // 生成PFFN个性化权重
2. if not skip_hsp:
      H_curr ← HSP(S^(l))                  // 三阶段层次种子池化
   else:
      H_curr ← H_prev                       // 复用前层摘要（CompSkip）
3. X^(l+1) ← GlobalInteraction([X^(l) | H_curr])  // 混合Wukong专家交互

=== Kunlun Transformer Block ===
4. if not skip_pffn:
      S̃ ← GDPA(S^(l), W^(l)) + S^(l)      // 个性化序列变换（残差）
   else:
      S̃ ← S^(l)                             // CompSkip跳过
5. if not skip_self_attention:
      S^(l+1) ← SlidingWindowMHA(S̃) + S̃   // 局部序列建模
   else:
      S^(l+1) ← S̃

Return: X^(l+1), S^(l+1), H_curr
```

### 各组件复杂度对比

| 组件 | 时间复杂度 |
|------|-----------|
| GDPA | $`O(T \cdot N \cdot d)`$ |
| 滑动窗口注意力 | $`O(T \cdot w \cdot d)`$ |
| 完整自注意力 | $`O(T^2 \cdot d)`$ |
| HSP | $`O(n_s \cdot T \cdot d)`$ |
| 全局交互（MoE Wukong） | $`O(M \cdot (N + n_t)^2 \cdot d)`$ |

## 实验结果

### 数据集与基线

**数据集**：Meta Ads 内部大规模数据集，70B+ 样本，数百至数千个非序列特征，10+ 条序列特征（长度数百至数千），多种事件类型（点击、转化、曝光）。

**基线**（均为 Meta 生产级模型，每日服务数十亿请求）：
- **Wukong**：纯非序列特征交互缩放基准（FM-based 堆叠交互模块）
- **InterFormer**：当前序列+非序列联合建模 SOTA（双向信息流）

### 主要结果：不同计算规模下的架构对比（RQ1）

| 方法 | 6 GFLOPs NE增益 | 60 GFLOPs NE增益 | 180 GFLOPs NE增益 |
|------|----------------|-----------------|------------------|
| Wukong | 基线 | 基线 | 基线 |
| Wukong + PMA | 0.16% | 0.11% | 0.20% |
| InterFormer | 0.31% | 0.36% | 0.50% |
| **Kunlun（本文）** | **0.31%** | **0.66%** | **0.79%** |

**关键发现**：
- 小规模（6 GFLOPs）：Kunlun 与 InterFormer 持平（均为 0.31%）；
- 大规模（180 GFLOPs）：Kunlun 0.79% vs InterFormer 0.50%，差距显著扩大（**1.58× 优势**）；
- Kunlun 在 180 GFLOPs 下实际 FLOPs 更低（154.9 vs 172.3），同时性能更好；
- 缩放效率为 InterFormer 的 **2×**（NE 改善速率是其两倍）。

### 消融实验（RQ2 & RQ3）

| 类别 | 配置 | MFU | ΔQPS | ΔNE | ΔGFLOPs |
|------|------|-----|------|-----|---------|
| **低层（RQ2）** | w/o GDPA | 34.0% | -8% | +0.04% | 中性 |
| | w/o HSP（用PMA） | 30.21% | **+8.8%** | +0.08% | -25% |
| | w/o SumKronLinear | 34.55% | 中性 | +0.03% | -7.5% |
| | w/o 滑动窗口 | 32.90% | -31.1% | 中性 | +29.5% |
| **高层（RQ3）** | w/o CompSkip | 34.50% | -35% | 中性 | +43.1% |
| | w/o 事件级个性化 | 35.7% | -13% | **-0.02%** | +11% |
| | w/o 专家并行 | 35.5% | -4% | 中性 | 中性 |
| **完整模型** | **Kunlun（Full）** | **37.0%** | 基线 | 基线 | 154.9 |

**关键洞察**：
- **GDPA**：MFU 提升 3pp（34%→37%），QPS +8%，NE +0.04%——效率和质量双收益；
- **HSP vs PMA**：HSP 虽 QPS 稍低（PMA 简单，QPS 高 8.8%），但 NE 改善 0.08%，生产中 NE 直接影响业务指标，值得取舍；
- **SumKronLinear**：NE +0.03%，QPS 中性，参数节省 14×；
- **滑动窗口**：QPS +31.1%，FLOPs -29.5%，NE 无损——验证推荐序列强局部性偏置；
- **CompSkip**：FLOPs -43.1%，QPS +35%，NE 中性——验证深度模型的层冗余性；
- **事件级个性化**：QPS +13%，FLOPs -11%，NE 甚至略有改善（0.02%）——资源按重要性分配更合理。

### Scaling Laws 验证

- **预测性**：Kunlun NE 随 log(Compute) 呈现一致的幂律下降，满足 Eq.(1) 的函数形式；
- **层数扩展**：1→6 层，每层 NE 增益遵循 $`\Delta\text{NE}_l \approx c / \log(l+1)`$，递减但可预测；
- **缩放系数**：Kunlun 的 $`\eta`$ 约为 InterFormer 的 **2×**（NE-log(Compute) 曲线斜率是其两倍）。

### 生产部署结果（RQ4）

- 在 Meta Ads 主要模型全面部署；
- Topline 指标提升 **1.2%**；
- NVIDIA B200 GPU 上 MFU 从 17% 提升至 **37%**；
- 相比 InterFormer 实现 **2× 缩放效率**。

## 核心贡献总结

1. **问题形式化**：首次将推荐系统的缩放效率挑战形式化，识别两大瓶颈——低效模块（MFU 3-15%）与低效计算分配；

2. **架构设计**：提出 Kunlun，通过六大技术系统性解决瓶颈：
   - 低层：**GDPA**（FlashAttention 风格 PFFN 融合内核，MFU +6×）、**HSP + SumKronLinear**（层次种子池化，14× 参数节省）、**滑动窗口注意力**（$O(T^2)→O(Tw)$，QPS +31%）
   - 高层：**CompSkip**（交替层模式，FLOPs -43%，QPS +35%）、**事件级个性化**（按重要性分配资源）、**混合 Wukong 专家**（水平+垂直双向扩展）

3. **Scaling Laws 建立**：首次为序列+非序列联合建模推荐系统建立可预测的幂律缩放规律，缩放效率是 SOTA 的 **2×**；

4. **工业部署**：在 Meta Ads 大规模部署，topline +1.2%，B200 MFU 17%→37%；

5. **关键创新点**：
   - ROTE（Rotary Temporal Embeddings）：将 RoPE 扩展为时序感知旋转嵌入
   - GDPA 可以统一 self-attention、PMA、PFFN 为同一注意力风格算子，实现单一 kernel 融合
   - SumKronLinear：Kronecker 积分解实现参数高效的跨维度压缩

## 与 InterFormer 的关系

Kunlun 明确建立在 InterFormer 框架之上：
- **继承**：双向信息流（非序列→序列，序列→非序列）、多层堆叠架构、PFFN 个性化序列变换；
- **超越**：将 PFFN 重构为 GDPA（效率提升 6×）、PMA 升级为 HSP（质量提升）、引入 CompSkip/事件级个性化/滑动窗口（系统效率大幅提升）；
- **核心差异**：InterFormer 关注算法效果，Kunlun 在此基础上系统解决工业规模下的**效率-效果协同优化**，最终实现可预测的 Scaling Laws。
