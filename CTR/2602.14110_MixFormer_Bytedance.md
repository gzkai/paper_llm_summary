## 研究背景与动机

### 问题定义

工业推荐系统中，Transformer 架构在两个方向上独立发展：
1. **序列建模（Sequential Modeling）**：捕捉用户行为序列中的长程时序依赖（DIN、BST、SIM、STCA 等）
2. **特征交叉（Feature Interaction）**：建模异构稠密特征之间的高阶关系（WuKong、RankMixer、DCNv2 等）

### 现有方法的核心缺陷：Co-Scaling 困境

当前工业系统通常采用两种方式组合上述两类模块：

- **层叠式（Stacked）**：序列模块的输出作为特征交叉模块的输入（如 STCA→RankMixer）
- **并行式（Parallel）**：两个模块并行运行，输出拼接后送入任务网络（如 OneTrans）

两种范式均保持严格的**参数隔离**，导致：
- 序列计算开销随序列长度快速增长，占据大部分计算预算
- 稠密特征交叉模块无法按比例扩展
- 跨模块的特征语义无法深度交互
- 在有限计算预算下，两个组件的扩展存在**根本性的资源竞争**

**核心观点**：有效的 Co-Scaling 需要将序列建模与特征交叉**统一在单一参数空间**内联合优化。

## 核心方法：MixFormer

MixFormer 是一个完全统一的 Transformer 风格大型推荐模型（Large Recommender Model），采用**共享参数集**同时建模序列行为和稠密特征交叉，架构上对齐 Transformer Decoder。

### 1. 特征嵌入与切分（Feature Embedding and Splitting）

**序列特征**：用户历史交互序列，每个时间步 $`t`$ 的行为表示为 $`\boldsymbol{s}_t`$，长度为 $`T`$ 的序列记为：

```math
\boldsymbol{S} = [\boldsymbol{s}_1, \boldsymbol{s}_2, \cdots, \boldsymbol{s}_T]
```

**非序列特征**：用户特征、物品特征、上下文特征，拼接后得到：

```math
\mathbf{e}_{\text{ns}} = [\boldsymbol{e}_1; \boldsymbol{e}_2; \dots; \boldsymbol{e}_M] \in \mathbb{R}^{D_{\text{ns}}}
```

**多头切分**：将 $`\mathbf{e}_{\text{ns}}`$ 均匀分成 $`N`$ 个子向量，每个子向量通过线性映射投影为 $`D`$ 维头：

```math
\boldsymbol{x}_j = \boldsymbol{W}_j\cdot \boldsymbol{e}_{\text{ns}}[d\cdot(j-1) : d\cdot j], \quad j = 1, \dots, N
```

其中 $`d = D_{\text{ns}}/N`$，$`\boldsymbol{W}_j\in \mathbb{R}^{D\times d}`$。最终得到非序列输入 $`\boldsymbol{X}=[\boldsymbol{x}_1, \dots, \boldsymbol{x}_N]\in\mathbb{R}^{N \times D}`$。

> **设计意图**：多头切分保留了特征的表示多样性，为下游模块提供了高效的接口，避免了将所有特征压缩为单一向量的信息损失。

---

### 2. MixFormer Block（核心模块）

每个 MixFormer Block 包含三个子模块，对应标准 Transformer Decoder 的三个组件：

| MixFormer 子模块 | 对应 Transformer 组件 | 核心作用 |
|-----------------|----------------------|---------|
| Query Mixer | Self-Attention | 异构特征的高阶交叉 |
| Cross Attention | Cross-Attention | 序列聚合（以高阶特征为 Query） |
| Output Fusion | FFN | 序列与非序列信号的深度融合 |

#### 2.1 Query Mixer

**动机**：推荐场景中，Query 来自高度异构的特征域（用户属性、物品属性、上下文信号），这些特征来自不同语义空间，直接用内积计算注意力权重不可靠。因此用轻量的 **HeadMixing** 替代 Self-Attention。

**HeadMixing**（零参数操作）：将 $`\boldsymbol{X} \in \mathbb{R}^{N \times D}`$ reshape 为 $`\mathbb{R}^{N \times N \times \frac{D}{N}}`$，转置前两维后 flatten 回 $`\mathbb{R}^{N \times D}`$，实现跨头信息交换。

**完整 Query Mixer 公式**：

```math
\boldsymbol{P} = \text{HeadMixing}(\text{Norm}(\boldsymbol{X})) + \boldsymbol{X}
```

```math
\boldsymbol{q}_i = \text{SwiGLUFFN}_i(\text{Norm}(\boldsymbol{p}_i)) + \boldsymbol{p}_i
```

其中 $`\text{SwiGLUFFN}_i`$ 为**逐头独立 FFN**（per-head FFN），显式处理特征异构性。

#### 2.2 Cross Attention

Query Mixer 的 $`N`$ 个输出头直接作为 Cross Attention 的 $`N`$ 个头，每个头专注于非序列特征的不同子空间，无需额外的 Query 投影矩阵。

序列中每个行为 $`\boldsymbol{s}_t`$ 先经过**逐层独立 SwiGLUFFN** 映射（per-layer FFN，每层独立参数化，逐层精炼序列表示），再投影为 Key 和 Value：

```math
\boldsymbol{h}_t = \text{SwiGLUFFN}^{(l)}(\text{Norm}(\boldsymbol{s}_t))+\boldsymbol{s}_t
```

```math
\boldsymbol{k}_{t}^{i} = \boldsymbol{W}_k^i\boldsymbol{h}_{t}^{i}; \quad \boldsymbol{v}_t^i = \boldsymbol{W}_v^i\boldsymbol{h}_t^i
```

第 $`i`$ 个 Query 头的聚合输出：

```math
\boldsymbol{z}_i = \sum_{t=1}^{T} 
\text{softmax}\!\left(
\frac{\boldsymbol{q}_i^\top \boldsymbol{k}_t^i}{\sqrt{D}}
\right)
\boldsymbol{v}_t^i +\boldsymbol{q}_i, \quad i = 1, \dots, N
```

#### 2.3 Output Fusion

对每个 Cross Attention 输出 $`\boldsymbol{z}_i`$ 应用**逐头独立 SwiGLUFFN** 进行深度非线性融合：

```math
\boldsymbol{o}_i = \text{SwiGLUFFN}_i(\text{Norm}(\boldsymbol{z}_i)) + \boldsymbol{z}_i
```

> **统一参数化的关键**：Query Mixer 和 Output Fusion 中的 per-head FFN 同时服务于序列特征和非序列特征，形成**统一参数化范式**，从根本上解决了两个独立模块之间的参数分配难题。

---

### 3. User-Item Decoupling（UI-MixFormer）

**动机**：统一 MixFormer 中用户侧和物品侧计算混合，无法利用 Request Level Batching（RLB）——在一个请求中对多个候选物品共享用户侧计算。

#### 3.1 特征解耦

将非序列特征分为用户侧（ $`N_U`$ 个头）和物品侧（ $`N_G`$ 个头），按嵌入维度比例分配：

```math
N_G = \left\lfloor\frac{D_{ns}^G N}{D_{ns}}\right\rfloor, \quad N_U = N - N_G
```

实践中 $`N_U:N_G = 1:1`$。

#### 3.2 带掩码的 Query Mixer

为保证用户侧头可被请求级共享，设计**单向用户→物品融合掩码** $`\mathcal{M} \in \mathbb{R}^{N \times D}`$：

```math
\mathcal{M}[i,j] = 
\begin{cases} 
0, & i < N_U \;\text{且}\; j \ge N_U \cdot \frac{D}{N}\\
1, & \text{otherwise}
\end{cases}
```

掩码后的 HeadMixing：

```math
\text{HeadMixing}_{\text{decouple}}(\cdot) = \mathcal{M} \odot \text{HeadMixing}(\cdot)
```

> **与双塔模型的区别**：该方法保留了用户侧到物品侧的信息传播，维持了用户-物品特征交叉建模能力，而非完全隔离。

## 实验设置

### 数据集

| 数据集 | 来源 | 规模 |
|--------|------|------|
| 抖音工业数据集 | Douyin 推荐系统 | 连续两周，万亿级用户-物品交互记录 |
| 特征数量 | — | 超过 300 个特征（非序列 + 序列两大类） |

序列长度设置：$`L = 512`$（主实验），扩展实验覆盖 $`\{512, 2048, 8192, 10000\}`$。

### 评估指标

- **准确性**：AUC（ROC 曲线下面积）、UAUC（用户级 AUC）
- **效率**：参数量（M）、FLOPs/Batch（GFLOPs）

### 基线方法

| 类型 | 方法 | 描述 |
|------|------|------|
| Stacked | TA→DLRM | 简单目标注意力 + MLP |
| Stacked | TA→DCNv2 | 目标注意力 + DCNv2 |
| Stacked | TA→DHEN | 目标注意力 + DHEN |
| Stacked | TA→WuKong | 目标注意力 + WuKong |
| Stacked | STCA→DCNv2 | 堆叠目标交叉注意力 + DCNv2 |
| Stacked | TA→RankMixer | 目标注意力 + RankMixer |
| Stacked | STCA→RankMixer | 堆叠目标交叉注意力 + RankMixer（最强基线） |
| Parallel | OneTrans | 异构 Token 序列统一 Transformer |
| Parallel | STCA⊕RankMixer | 并行拼接 |

### 实现细节

- 训练：数百 GPU，混合分布式（稀疏部分异步更新，稠密部分同步更新）
- 优化器：稠密部分 RMSProp（lr=0.01），稀疏部分 Adagrad
- Batch size：1,500
- MixFormer-small：$`N=16, L=4, D=386`$
- MixFormer-medium：$`N=16, L=4, D=768`$

## 实验结果

### 主实验（准确性 + 效率，序列长度 L=512）

| 类型 | 模型 | Finish AUC↑ | Finish UAUC↑ | Skip AUC↑ | Skip UAUC↑ | #Params(M) | GFLOPs/Batch |
|------|------|------------|-------------|----------|-----------|-----------|-------------|
| Stacked | TA→DLRM（基准） | 0.8554 | 0.8270 | 0.8124 | 0.7294 | 9 | 52 |
| Stacked | STCA→RankMixer | +1.12% | +1.40% | +1.43% | +2.14% | 1,255 | 6,736 |
| Parallel | OneTrans | +1.05% | +1.31% | +1.30% | +1.95% | 316 | 23,371 |
| Parallel | STCA⊕RankMixer | +1.11% | +1.38% | +1.42% | +2.11% | 1,255 | 6,736 |
| **Unified** | **MixFormer-medium** | **+1.28%** | **+1.60%** | **+1.60%** | **+2.46%** | 1,226 | 3,503 |
| **Unified** | **UI-MixFormer-medium** | **+1.28%** | **+1.60%** | **+1.60%** | **+2.46%** | 1,226 | **2,242** |

**关键发现**：
- MixFormer-medium 在与最强基线 STCA→RankMixer 参数量相当的情况下，AUC 全面超越
- UI-MixFormer 在性能不变的情况下，FLOPs 降低约 **36%**（6,736 → 2,242 GFLOPs）
- OneTrans 虽也是统一架构，但 FLOPs 高达 23,371，是 MixFormer 的 6.7 倍

### 消融实验

| 变体 | 描述 | 影响 |
|------|------|------|
| [QM] wo HM | 去掉 HeadMixing | 性能明显下降，高阶特征对序列 Query 至关重要 |
| [QM] HM→SA | 用 Self-Attention 替换 HeadMixing | 性能无明显提升，但计算开销大幅增加 |
| [QM] wo FFN | 去掉 per-head FFN | 性能下降 |
| [CA] PL-FFN→FFN | Cross Attention 中用共享 FFN 替换逐层 FFN | 表示多样性降低 |
| [OF] PFFN→FFN | Output Fusion 中用共享 FFN 替换 per-head FFN | 性能下降，FLOPs 不变 |
| Pre-RN→Post-LN | 用 Post-LayerNorm 替换 Pre-RMSNorm | 训练稳定性下降 |

### Scaling 分析

#### Dense Scaling（固定序列长度，扩展参数量）

- RankMixer（非序列模型）在固定序列长度下扩展稠密参数的边际 AUC 增益 > 序列模型
- MixFormer 具有**更高的截距（intercept）和有竞争力的扩展斜率**，在各 FLOPs 预算下均优于其他设计

#### Sequence Length Scaling（固定参数量，扩展序列长度）

序列长度：$`\{512, 2048, 8192, 10000\}`$

- 序列扩展的收益趋势与稠密扩展**相反**：序列模型（STCA）从更长序列中获益更多
- MixFormer 的序列扩展斜率与 SOTA 的 STCA 相当，得益于统一参数化

**Co-Scaling 结论**：独立参数化的模型在稠密扩展和序列扩展之间存在显著的**权衡困境**；MixFormer 在两个维度上均达到 SOTA 级扩展行为。

### 在线 A/B 测试（两周实验）

对比基线：线上部署的 STCA→RankMixer（参数量超 1B）

| 平台 | Active Day↑ | Duration↑ | Like↑ | Finish↑ | Comment↑ |
|------|------------|----------|------|--------|---------|
| **Douyin** | +0.0415% | +0.2799% | +0.1766% | +0.3897% | +0.7035% |
| **Douyin Lite** | +0.0252% | +0.4105% | +0.2125% | +0.2924% | +1.9097% |

> 低活跃用户群体（Low-active）提升尤为显著，如 Douyin Lite 的 Like 指标提升 +3.06%。实验结束时增益仍未收敛，持续提升中。

## 核心贡献总结

1. **MixFormer 统一架构**：首个在单一参数空间内联合建模序列动态与稠密特征交叉的工业推荐 Transformer，通过 Query Mixer（HeadMixing + per-head FFN）+ Cross Attention（per-layer FFN）+ Output Fusion（per-head FFN）三模块实现深度集成

2. **UI-MixFormer 解耦优化**：通过单向掩码 HeadMixing 实现用户-物品特征解耦，支持请求级批处理（RLB），在保持性能的同时降低约 36% FLOPs，并在候选物品数量增大时显著抑制延迟增长

3. **Co-Scaling 优越性**：在稠密扩展和序列扩展两个维度上均达到 SOTA，从根本上解决了独立参数化设计中序列模块与稠密模块之间的资源竞争困境

4. **工业验证**：在抖音和抖音极速版两个亿级用户 App 的 Feed 推荐场景中完成大规模 A/B 测试，用户活跃天数、使用时长、互动率等核心指标全面提升

## 局限性与未来工作

论文未显式列出局限性，但从内容可归纳：
- 实验仅在字节跳动内部数据集上验证，公开数据集上的泛化性有待验证
- 序列长度扩展实验最长至 10,000，更长序列（如百万级）的适用性未探索
- 多模态特征（图像、视频等）的统一建模未涉及
