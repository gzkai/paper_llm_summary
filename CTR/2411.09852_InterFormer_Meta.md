## 研究背景与动机

### 问题定义

CTR 预测（Click-Through Rate Prediction）估计用户点击某个广告/物品的概率，是推荐系统和在线广告的核心任务。现代 CTR 系统面临**异构信息**的挑战：

- **非序列特征（Non-Sequence）**：用户画像（年龄、性别）、广告内容特征、上下文特征等，反映用户**静态、长期**兴趣；
- **序列特征（Sequence）**：用户历史行为序列（点击、购买记录），反映用户**动态、即时**兴趣。

### 现有方法的两大核心缺陷

1. **不充分的跨模态交互（Insufficient Inter-mode Interaction）**：
   - 现有序列方法（DIN、DIEN、BST 等）仅支持**单向信息流**：非序列特征 → 序列学习（non-sequence guides sequence）；
   - 反向信息流（sequence → non-sequence）被忽视，导致非序列特征无法感知用户当前即时兴趣；
   - 例如：用户画像显示其对电子产品感兴趣，但最近浏览记录集中在智能手机，这种即时偏好应反哺非序列特征的学习。

2. **激进的信息聚合（Aggressive Information Aggregation）**：
   - 由于计算挑战，现有方法在早期阶段对序列做激进压缩（求和、池化、拼接），导致大量信息损失；
   - 这种"早期总结"使得后续模块无法充分利用序列中的细粒度信息。

## 核心方法：InterFormer

InterFormer 提出了一种**交错式（Interleaving）**异构交互学习框架，包含三大核心模块：

### 整体架构概览

```
输入: 非序列特征 X^(0) + 序列特征 S^(0)
        ↓ 预处理（Feature Preprocessing）
        ↓ 
   ┌────────────────────────────────────┐
   │  Layer l = 1, 2, ..., L            │
   │  ┌──────────────┐                  │
   │  │  Cross Arch  │ ← 信息选择与总结  │
   │  └──────┬───────┘                  │
   │         │ X_sum^(l), S_sum^(l)     │
   │  ┌──────┴──────┐  ┌─────────────┐  │
   │  │Interaction  │  │  Sequence   │  │
   │  │   Arch      │  │    Arch     │  │
   │  │(非序列交互)  │  │ (序列建模)  │  │
   │  └──────┬──────┘  └──────┬──────┘  │
   │         │ X^(l+1)        │ S^(l+1) │
   └─────────┼────────────────┼─────────┘
             ↓                ↓
     最终预测: ŷ = MLP([X_sum^(L) || S_sum^(L)])
```

---

### 模块一：特征预处理（Feature Preprocessing）

**非序列特征预处理**：
- 稠密特征 $`x_{\text{dense}_i}^{(0)}`$ 拼接后经线性变换得到 $d$ 维嵌入 $`\mathbf{x}_{\text{dense}}^{(1)} = \mathbf{W}_{\text{dense}} \mathbf{x}_{\text{dense}}^{(0)}`$
- 稀疏特征 one-hot 编码后映射到 $d$ 维嵌入
- 输入非序列嵌入矩阵：

```math
\mathbf{X}^{(1)} = \left[\mathbf{x}_{\text{dense}}^{(1)} \| \mathbf{x}_{\text{sparse}_1}^{(1)} \| \dots \| \mathbf{x}_{\text{sparse}_n}^{(1)}\right]
```

**序列特征预处理**：
- 每个历史交互物品映射到 $d$ 维向量 $`\mathbf{s}_t`$
- 支持多序列（如点击序列、转化序列）：通过 **MaskNet** 统一多序列并过滤内部噪声：

```math
\text{MaskNet}(\mathbf{S}) = \text{MLP}_{\text{lce}}\left(\mathbf{S} \odot \text{MLP}_{\text{mask}}(\mathbf{S})\right)
```

其中 $`\text{MLP}_{\text{mask}}`$ 计算自掩码以选择相关信息，$`\text{MLP}_{\text{lce}}`$（Linear Compressed Embedding）将多序列合并为单序列。

---

### 模块二：Interaction Arch（行为感知的非序列交互学习）

**目标**：让非序列特征感知行为序列，学习**行为感知（Behavior-Aware）**的非序列嵌入。

给定第 $l$ 层的非序列输入 $`\mathbf{X}^{(l)}`$ 和序列总结 $`\mathbf{S}_{\text{sum}}^{(l)}`$，输出为：

```math
\mathbf{X}^{(l+1)} = \text{MLP}^{(l)}\left(\text{Interaction}^{(l)}\left(\left[\mathbf{X}^{(l)} \| \mathbf{S}_{\text{sum}}^{(l)}\right]\right)\right)
```

- **灵活的 Interaction 模块**：支持 Inner Product、DCNv2、DHEN 等多种骨干网络（论文实验中使用 DHEN）；
- 通过将 $`\mathbf{X}^{(l)}`$ 和 $`\mathbf{S}_{\text{sum}}^{(l)}`$ 拼接后做交互，实现：
  - 非序列特征间的交互 → 捕获显式用户兴趣（用户画像 × 目标物品内容）；
  - 非序列与序列间的交互 → 捕获隐式用户兴趣（目标物品 × 用户当前需求）；
- MLP 变换保持输入输出形状一致，便于多层堆叠。

---

### 模块三：Sequence Arch（上下文感知的序列建模）

**目标**：让序列建模感知非序列上下文，学习**上下文感知（Context-Aware）**的序列嵌入。

基于两个核心设计：

#### 3.1 Personalized FFN (PFFN)

以非序列总结 $`\mathbf{X}_{\text{sum}}^{(l)}`$ 为查询，对序列嵌入 $`\mathbf{S}^{(l)}`$ 做个性化变换：

```math
\text{PFFN}\left(\mathbf{X}_{\text{sum}}^{(l)}, \mathbf{S}^{(l)}\right) = f\left(\mathbf{X}_{\text{sum}}^{(l)}\right) \mathbf{S}^{(l)}
```

其中 $`f(\mathbf{X}_{\text{sum}}^{(l)})`$ 是一个 MLP，基于非序列总结学习序列上的线性投影权重。

> PFFN 本质上是计算序列嵌入与非序列嵌入之间的增强点积交互，为后续序列建模注入个性化上下文。

#### 3.2 Multi-Head Attention (MHA) + CLS Token

- 在第一层前，将非序列总结 $`\mathbf{X}_{\text{sum}}^{(1)}`$ 作为 **CLS token** 前置到序列中：$`\mathbf{S}^{(1)} = [\mathbf{X}_{\text{sum}}^{(1)} \| \mathbf{S}^{(1)}]`$
- 后续 MHA 中 CLS token 以非序列信息为查询，聚合序列信息
- 应用 **旋转位置编码（RoPE）** 有效利用序列位置信息

整体 Sequence Arch：

```math
\mathbf{S}^{(l+1)} = \text{MHA}^{(l)}\left(\text{PFFN}\left(\mathbf{X}_{\text{sum}}^{(l)}, \mathbf{S}^{(l)}\right)\right)
```

输出与输入形状相同，避免激进聚合，支持多层堆叠。

---

### 模块四：Cross Arch（信息选择与总结）

**目标**：在 Interaction Arch 和 Sequence Arch 之间高效交换信息，解决高维特征直接交换的低效性和噪声问题。

#### 4.1 非序列总结（$`\mathbf{X}_{\text{sum}}^{(l)}`$）

通过个性化门控选择机制从高维非序列嵌入中提炼关键信息：

```math
\mathbf{X}_{\text{sum}}^{(l)} = \text{Gating}(\text{MLP}(\mathbf{X}^{(l)}))
```

```math
\text{Gating}(\mathbf{X}) = \sigma\left(\mathbf{X} \odot \text{MLP}(\mathbf{X})\right)
```

其中 MLP: $`\mathbb{R}^{d \times n} \to \mathbb{R}^{d \times n_{\text{sum}}}`$（$`n_{\text{sum}} \ll n`$），自门控提供稀疏掩码，保留相关信息过滤噪声。

#### 4.2 序列总结（$`\mathbf{S}_{\text{sum}}^{(l)}`$）

设计三类序列总结，互补融合：

| 总结类型 | 来源 | 作用 |
|---------|------|------|
| **CLS tokens** $`\mathbf{S}_{\text{CLS}}`$ | MHA 学习的 CLS token | 上下文感知的全局序列总结 |
| **PMA tokens** $`\mathbf{S}_{\text{PMA}}`$ | 可学习查询的 PMA 注意力 | 弥补 CLS 对非序列上下文的依赖，提供更灵活的序列总结 |
| **Recent tokens** $`\mathbf{S}_{\text{recent}}`$ | 最近 $K$ 个交互物品 | 捕获用户最新即时兴趣 |

三类总结经门控融合：

```math
\mathbf{S}_{\text{sum}}^{(l)} = \text{Gating}\left(\left[\mathbf{S}_{\text{CLS}}^{(l)} \| \mathbf{S}_{\text{PMA}}^{(l)} \| \mathbf{S}_{\text{recent}}^{(l)}\right]\right)
```

---

### 完整算法流程

```
输入: 非序列特征 X^(0), 序列特征 S^(0), 层数 L
输出: CTR 预测 ŷ

1. 预处理 X^(0), S^(0) → X^(1), S^(1)
2. 计算非序列总结 X_sum^(1)，将其作为 CLS token 前置到 S^(1)
3. for l = 1, 2, ..., L:
   a. Cross Arch: 计算 X_sum^(l) 和 S_sum^(l)
   b. Interaction Arch: X^(l+1) = MLP(Interaction([X^(l) || S_sum^(l)]))
   c. Sequence Arch: S^(l+1) = MHA(PFFN(X_sum^(l), S^(l)))
4. 最终预测: ŷ = MLP([X_sum^(L) || S_sum^(L)])
```

## 实验结果

### 数据集

| 数据集 | 样本数 | 特征数（序列/非序列） | 序列长度 |
|--------|--------|----------------------|---------|
| AmazonElectronics | 3.0M | 6 (2/4) | 100 |
| TaobaoAds | 25.0M | 22 (3/19) | 50 |
| KuaiVideo | 13.7M | 9 (4/5) | 100 |
| Internal (Meta) | **70B** | 数百（非序列）+ 10序列 | 200~1000 |

### 基准数据集结果

与 14 个 SOTA 基线对比（非序列方法：FM、xDeepFM、AutoInt+、DCNv2、FmFM、DOT、DHEN、Wukong；序列方法：DIN、DIEN、BST、DMIN、DMR、TransAct）：

| 数据集 | 指标 | 最强基线 | **InterFormer** | 提升 |
|--------|------|---------|----------------|------|
| AmazonElectronics | gAUC | 0.8835 (TransAct) | **0.8843** | +0.09% |
| AmazonElectronics | AUC | 0.8856 (DIEN) | **0.8865** | +0.10% |
| AmazonElectronics | LogLoss | 0.4276 (DIEN) | **0.4253** | -0.54% |
| TaobaoAds | gAUC | 0.5723 (DMIN) | **0.5728** | +0.09% |
| TaobaoAds | AUC | 0.6519 (DIEN) | **0.6528** | +0.14% |
| TaobaoAds | LogLoss | 0.1929 (DIEN) | **0.1926** | -0.03% |
| KuaiVideo | AUC | 0.7451 (DIEN) | **0.7453** | +0.03% |
| KuaiVideo | LogLoss | 0.4343 (DIEN) | **0.4340** | -0.07% |

> InterFormer 在 3 个数据集上全面超越所有基线，最大提升：gAUC +0.9%，AUC +0.14%，LogLoss -0.54%。

### 工业数据集结果（Meta 内部）

| 指标 | 结果 |
|------|------|
| NE 提升 | **+0.15%**（相比内部 SOTA 模型，FLOPs 相近） |
| QPS 提升 | **+24%** |
| 在线 topline 指标 | **+0.6%**（2024 年试点上线） |

**序列特征扩展（Feature Scaling）**：
- 增加 2 条长序列（长度 1000）后，NE 额外提升 **+0.14%**；
- InterFormer 的 NE 曲线持续下降，优于内部强基线 **0.06%**；
- 合并 6 条序列为单条长序列（长度 600）：QPS +20%，MFU +17%，NE 仅损失 0.02%。

**模型层数扩展（Model Scaling）**：
- 1→2 层：NE +0.13%；2→3 层：+0.05%；3→4 层：+0.04%；
- 展示良好的层数扩展性。

## 消融实验

### 交错学习方式分析

| 场景 | 描述 | 性能趋势 |
|------|------|---------|
| `sole` | 仅 Interaction Arch，序列早期聚合 | 最低 |
| `sep` | 双 Arch 独立训练，无跨模态交互 | 次低 |
| `s2n` | 仅序列→非序列单向信息流 | 中等 |
| `n2s` | 仅非序列→序列单向信息流 | 中等（≈s2n） |
| `int` | 双向信息流（InterFormer） | **最高** |

**结论**：`sole < sep < n2s ≈ s2n < int`，双向信息流带来最大收益，最高 AUC 提升 **1.46%**。

### 信息聚合方式对比

| 聚合方式 | 性能 |
|---------|------|
| Average Pooling（激进） | 最低 |
| MLP 压缩 | 次低 |
| MHA 聚合 | 中等 |
| InterFormer 选择性聚合 | **最高** |

**结论**：聚合越选择性，CTR 预测质量越高。

### 组件消融

| 消融项 | AUC 影响 |
|--------|---------|
| 去除 PMA tokens | -0.004 |
| 去除 Gating 模块 | -0.003 |
| 去除 PFFN | 略有下降 |
| 去除 MHA | 略有下降 |

## 系统优化：模型-系统协同设计

InterFormer 架构支持两项关键系统优化，总体训练效率提升 **>30%**：

### 1. 通信开销降低
- Interaction Arch（DHEN，参数重、计算轻）倾向于 FSDP 通信瓶颈；
- Sequence Arch（Transformer，计算密集）倾向于计算瓶颈；
- **并行设计**：两个 Arch 并行执行，DHEN 的通信与序列计算重叠 → QPS 提升 **20%**。

### 2. 计算效率优化
- FLOPs 从低 ROI 模块迁移到高 ROI 模块；
- 小算子合并以更好利用 GPU 资源；
- 交互模块 MFU: 11% → 16%，DHEN MFU: 38% → 45%，整体 MFU 提升 **19%**。

## 核心贡献总结

1. **问题识别**：首次系统识别异构交互学习的两大瓶颈——不充分的跨模态交互 + 激进的信息聚合；

2. **交错式学习框架**：提出 InterFormer，通过三个协同模块（Interaction Arch、Sequence Arch、Cross Arch）实现双向信息流和选择性信息聚合；

3. **关键技术创新**：
   - **PFFN**：以非序列总结为权重的个性化 FFN，实现跨模态序列变换；
   - **三类序列总结**（CLS + PMA + Recent）：互补融合，全面捕获用户兴趣；
   - **自门控选择机制**：稀疏掩码过滤噪声，高质量跨模态信息交换；
   - **模型-系统协同设计**：并行 Arch 设计降低通信瓶颈，实现 24% QPS 提升；

4. **工业验证**：在 Meta 广告系统大规模部署，NE +0.15%，QPS +24%，在线 topline 指标 +0.6%。
