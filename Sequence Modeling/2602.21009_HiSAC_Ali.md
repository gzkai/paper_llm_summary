## 一、背景与问题

现代推荐系统（如淘宝、视频流媒体）依赖用户行为序列捕捉动态偏好。高活跃用户的历史行为可达 **数千乃至 10,000+ 条**，直接建模代价极高（延迟、内存），线上不可行。

现有压缩方案的两大核心缺陷：
1. **无法准确识别用户特定的兴趣中心**：ELASTIC、PolyEncoder 使用固定数量的全局兴趣嵌入，忽略了不同用户兴趣粒度的差异；K-Means、LSH、时间分割等方法不稳定且语义一致性差。
2. **行为分配错误导致量化误差**：硬分配将语义相似的行为映射到不同簇，丢失长尾偏好。

## 二、方法：HiSAC（Hierarchical Sparse Activation Compression）

HiSAC 通过三阶段实现超长序列的个性化压缩：

### 阶段一：用户历史行为的 Tokenization

**多模态编码器**
- 对商品库 $\mathcal{I}$ 中所有商品，提取文本属性（标题、类目、元数据）+ 产品图像
- 使用大规模多模态编码器进行跨模态对齐与融合，得到统一语义空间的嵌入 $\mathbf{e} \in \mathbb{R}^d$
- 与推荐模型解耦，避免曝光偏差

**RQ-VAE 离散化（残差量化变分自编码器）**
- 将连续嵌入 $\mathbf{e}$ 离散化为 $L$ 层层级码书（Codebook）
- 残差量化：第 1 层直接量化编码器输出，后续每层量化前一层的残差，逐步减小量化误差
- 每个商品映射为 $L$ 层语义 ID（Semantic ID, SID）：$\mathbf{s}_h = (s_h^{(1)}, \ldots, s_h^{(L)})$
- 用户历史序列相应 tokenize 为 SID 序列

### 阶段二：层级投票的稀疏兴趣激活（Hierarchical Voting）

**构建全局层级语义树**
- 将整个 SID 空间组织为 $(L+1)$ 层全局树，根节点为虚拟节点，叶节点对应完整 SID
- 从根到叶的路径唯一定义一个细粒度语义概念
- 该树静态共享于所有用户，作为语义索引

**投票产生兴趣代理（Interest-Agents）**
1. 将用户历史中每件商品映射到对应叶节点
2. **自底向上**传播投票：叶节点票数 = 映射到该节点的商品数；内部节点票数 = 所有后代之和
3. **自顶向下稀疏激活**：从根出发，在每层 $l$ 对每个已激活父节点，仅保留票数最高的 top-$k^{(l)}$ 子节点，其余剪枝

最终保留的活跃叶节点集合即为用户的兴趣代理集合：
$$\mathcal{A}_u = \{a_1, \ldots, a_K\}, \quad K = \prod_{l=1}^{L} k^{(l)}$$

每个 agent $a$ 由完整根到叶路径（即 $L$ 层 SID）表示，权重 $w_a$ = softmax 归一化的该 agent 对应商品数（反映兴趣强度）。

**设计优势**
- 稀疏激活避免组合爆炸，聚焦最显著的语义分支
- 无需超参敏感的 K-Means/LSH，天然适应用户兴趣粒度差异
- 粗细粒度兼顾（层级结构保留多层次语义）

### 阶段三：Soft-Routing Attention 压缩

**Hard-Routing 的缺陷**：按精确 SID 匹配进行硬分配，会放大量化误差，且完全丢弃长尾交互（SID 不匹配任何 agent 的商品）。

**核心思想**：路由决策基于去偏语义相似度，聚合内容使用面向排序任务的表示。HiSAC 使用两套不同的商品嵌入：

| 嵌入类型 | 来源 | 用途 | 是否可训练 |
|---------|------|------|----------|
| **语义嵌入** $\mathbf{e}_i \in \mathbb{R}^d$ | 冻结的多模态编码器 + RQ-VAE | 计算与 agent 的相似度（路由权重） | ❌ 固定 |
| **排序嵌入** $\mathbf{e}'_i \in \mathbb{R}^{d'}$ | 端到端推荐模型训练（CTR/排序损失）| 聚合内容（加权求和的值） | ✅ 可训练 |

**Soft-Routing Attention 公式**：
对每个 agent $a_k$，其语义原型 $\mathbf{z}_{a,k}$ 由 RQ-VAE 对应码字求和得到，与语义嵌入 $\mathbf{e}_i$ 同空间，可直接计算 $L_2$ 距离。
$$\hat{z}_k = w_{a,k} \cdot \sum_{i=1}^{N} \underbrace{\frac{\exp\left(-\frac{\|\mathbf{e}_i - \mathbf{z}_{a,k}\|_2}{\tau}\right)}{\sum_{j=1}^{N}\exp\left(-\frac{\|\mathbf{e}_j - \mathbf{z}_{a,k}\|_2}{\tau}\right)}}_{\text{基于语义的注意力权重}} \cdot \mathbf{e}'_i$$

- $\tau > 0$：温度参数，控制分布尖锐程度（$\tau$ 越小越尖锐，趋向硬路由；$\tau$ 越大越平滑）
- 最终压缩序列：$\hat{H}_u = \{\hat{z}_1, \ldots, \hat{z}_K\}$，长度 $K \ll N$

**两套嵌入解耦的意义**：
- 语义嵌入去偏（不受热度影响），保证路由公平性
- 排序嵌入任务对齐，保证聚合内容对 CTR 预测有效
- 若共用同一嵌入，RQ-VAE 码书被任务偏置，平均每用户激活 agent 数减少 13%（从 182→158），兴趣覆盖度下降，损害异质偏好用户

### 工业部署优化

**（1）离线构建兴趣代理**
- 用户主要兴趣在短时间窗口内稳定，每天离线构建一次兴趣树，在线直接查询，大幅降低延迟

**（2）请求级序列压缩**
- 同一请求中，序列对所有候选商品不变，只压缩一次（$\mathcal{O}(NK)$），再与 $M$ 个候选进行 MHA（$\mathcal{O}(KM)$），总计 $\mathcal{O}(NK + KM)$，远优于原始 $\mathcal{O}(NM)$

**（3）缓存增强 MHA**
- MHA 中超 70% FLOPs 来自 Q/K/V 线性投影
- **候选 Query**：从静态特征离线预计算并缓存
- **用户 Key/Value**：每请求压缩后计算一次并缓存复用
- 结果：projection FLOPs 减少约 **72%**，每请求推理时间降低 **38%**，命中率 >99%
- 端到端延迟总体降低 **40%+**

### 超参分析

| 超参 | 最优配置 | 结论 |
|------|---------|------|
| RQ-VAE 层数 $L$ | $L=2$ | 多层优于单层；$L=3$ 时 Soft-Routing 持平，Hard-Routing 下降（量化错配） |
| 层级预算分配 $(k^{(1)}, k^{(2)})$ | $(200, 1)$ 优于 $(100,2)$ 优于 $(50,4)$ | 粗粒度覆盖比细粒度分化更重要 |
| 总 agent 数 $K$ | $K \approx 200$ | Soft-Routing 更高效，Hard-Routing 需更大 $K$ 才饱和 |
| 温度系数 $\tau$ | $\log(\tau) \approx -0.18$ | 过小→退化为硬路由；过大→注意力过平滑 |

## 四、总结

HiSAC 通过三个关键创新解决了工业级超长序列推荐的核心难题：

1. **多模态 RQ-VAE Tokenization**：构建去偏、多层次语义空间，为后续层级建模提供稳定基础
2. **层级投票稀疏激活**：自适应识别每个用户特有的兴趣代理，同时将 agent 数控制在可负担的量（约 200 个），计算效率高
3. **Soft-Routing Attention + 语义/排序双空间解耦**：最小化量化误差，保留长尾兴趣，同时保证聚合表示与排序目标对齐

工程层面三重部署优化（离线树构建 + 请求级压缩 + 缓存 MHA）使 HiSAC 在超过 40% 延迟压缩的同时，实现了线上 1.65% CTR 提升。
