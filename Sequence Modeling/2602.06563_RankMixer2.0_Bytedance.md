## 研究背景与动机

### RankMixer（TokenMixer）的五大局限

TokenMixer-Large 是对 RankMixer（SOTA TokenMixer 架构）的系统性升级，针对其五大核心缺陷：

| 缺陷 | 具体问题 |
|------|---------|
| **次优残差设计（Sub-optimal Residual）** | Mixing 后 token 数量 $`H`$ 与原始 token 数量 $`T`$ 不一致时无法建立正常残差连接；直接相加导致语义错位 |
| **架构不纯（Impure Model Architecture）** | 历史迭代遗留大量碎片化算子（LHUC、DCNv2 等），内存密集但计算强度低，拖累整体 MFU |
| **深层梯度不足（Insufficient Gradient Updates）** | RankMixer 通常仅堆叠 2 层，深层架构下训练稳定性和性能增益均难以保证 |
| **MoE 稀疏化不彻底（Inadequate MoE Sparsification）** | "Dense Train, Sparse Infer" 范式无法降低训练成本；ReLU-MoE 激活数量动态不可预测，推理不友好 |
| **规模探索受限（Limited Scaling）** | 框架和训练效率限制，RankMixer 参数量仅达约 1B |

### 核心目标

系统性解决上述问题，将推荐排序模型参数量扩展至 **7B/15B**，同时保证训练/推理效率。

## 核心方法：TokenMixer-Large 架构

### 总体结构

TokenMixer-Large 由三大模块组成：
1. **Tokenization**：稀疏特征 → 稠密 Embedding → 语义对齐 Token
2. **TokenMixer-Large Block**：Mixing & Reverting + Pertoken SwiGLU + 残差 & 归一化
3. **Sparse-Pertoken MoE**：稀疏训练 + 稀疏推理的统一 MoE 扩展

---

### 一、Tokenization（特征 Token 化）

#### Semantic Group-wise Tokenizer

工业推荐系统包含用户特征、物品特征、行为序列特征（DIN/SIM/LONGER）和交叉特征。所有特征首先经过 Embedding 层投影：

```math
e_i = \text{Embedding}(F_i, d_i) \in \mathbb{R}^{d_i}
```

按语义分组后，同组特征拼接并通过独立 MLP 对齐到统一维度：

```math
X_i = \text{MLP}_i(\text{concat}[e_l, \dots, e_m]), \quad e_l, \dots, e_m \in G_i
```

#### Global Token（全局 Token）

受 BERT `[CLS]` 启发，引入全局 Token 聚合所有组的信息：

```math
X_G = \text{MLP}_g(\text{concat}[G_1, \dots, G_{T-1}])
```

最终输入为全局 Token 与语义 Token 的拼接：

```math
\mathbf{X} = \text{concat}[X_G, X_0, \dots, X_{T-1}] \in \mathbb{R}^{T \times D}
```

---

### 二、Mixing & Reverting（核心创新）

#### RankMixer 的残差问题

RankMixer 的 Mixing 操作将原始 $`T`$ 个 token 重排为 $`H`$ 个 token，若 $`T \neq H`$ 则无法建立标准残差；即使 $`T = H`$，直接相加也存在语义错位（Mixing 前后 token 语义不一致）。

三个关键设计属性定义：
- **SR（Standard Residual）**：块间是否存在标准残差连接
- **OTR（Original Token Residual）**：原始 token 语义是否能传播到后续层
- **TSA（Token Semantic Alignment）**：残差操作 $`F(x') + x`$ 中 token 语义是否对齐

**RankMixer 三者均不满足，TokenMixer-Large 三者全满足。**

#### Mixing & Reverting 操作

TokenMixer-Large Block 采用对称的"混合–还原"双层结构：

**第一步：Mixing（跨 token 信息混合）**

```math
[\dots, [x_t^{(1)}, \dots, x_t^{(H)}], \dots] = \text{split}(\mathbf{X}) \in \mathbb{R}^{T \times H \times (D/H)}
```

```math
H_h = \text{concat}[x_1^{(h)}, \dots, x_T^{(h)}] \in \mathbb{R}^{T \times D/H}
```

```math
\mathbf{H} = \text{concat}[H_1, \dots, H_H] \in \mathbb{R}^{H \times (T \cdot D/H)}
```

```math
\mathbf{H}^{\text{next}} = \text{Norm}(\text{pSwiGLU}(\mathbf{H}) + \mathbf{H}) \in \mathbb{R}^{H \times (T \cdot D/H)}
```

**第二步：Reverting（还原到原始 token 维度）**

```math
[\dots, [x_1^{\prime(h)}, \dots, x_T^{\prime(h)}], \dots] = \text{split}(\mathbf{H}^{\text{next}}) \in \mathbb{R}^{T \times H \times (D/H)}
```

```math
X_t^{\text{revert}} = \text{concat}[x_t^{\prime(1)}, \dots, x_t^{\prime(H)}] \in \mathbb{R}^D
```

```math
\mathbf{X}^{\text{revert}} = \text{concat}[X_1^{\text{revert}}, \dots, X_T^{\text{revert}}] \in \mathbb{R}^{T \times D}
```

```math
\mathbf{X}^{\text{next}} = \text{Norm}(\text{pSwiGLU}(\mathbf{X}^{\text{revert}}) + \mathbf{X}) \in \mathbb{R}^{T \times D}
```

**关键**：Reverting 步骤将混合后的 token 还原为原始 $`T \times D`$ 形状，使输入输出维度一致，从而建立连续残差信号通路。

---

### 三、Pertoken SwiGLU

将 RankMixer 的 Pertoken FFN 升级为 Pertoken SwiGLU（每个 token 有独立参数）：

```math
\text{pSwiGLU}(\cdot) = \text{FC}_{\text{down}}(\text{Swish}(\text{FC}_{\text{gate}}(\cdot)) \odot \text{FC}_{\text{up}}(\cdot))
```

其中每个 FC 层均为 per-token 独立参数：

```math
\text{FC}_i(\mathbf{x}) = W_i^t \mathbf{x}_t + b_i^t, \quad i \in \{\text{up}, \text{gate}, \text{down}\}
```

参数维度：$`\{W_{\text{up}}^t, W_{\text{gate}}^t\} \in \mathbb{R}^{D \times nD}`$，$`W_{\text{down}}^t \in \mathbb{R}^{nD \times D}`$，$`n`$ 为隐层扩展比。

---

### 四、残差 & 归一化

- **Pre-Norm RMSNorm**（替代 RankMixer 的 Post-Norm LayerNorm）：
  - 移除 LayerNorm 的均值中心化步骤，保留归一化核心功能
  - 移除所有 bias kernel 矩阵（归一化后 bias 数学上冗余）
  - 端到端吞吐量提升 **8.4%**，训练稳定性显著改善
- Post-Norm 性能更好但容易梯度爆炸（实验中出现 NaN）

---

### 五、Inter-Residual & Auxiliary Loss（深层模型优化）

**层间残差（Inter-Residual）**：每隔 2~3 层添加跨层残差连接，强化低层特征向高层的传播，缓解深层网络梯度消失。注意：**最后一层不建议添加**，以免低层原始信息干扰最终高层抽象。

**辅助损失（Auxiliary Loss）**：将低层输出的 logits 与高层输出联合计算损失，使低层网络学习"估计高层特征偏差"的能力，防止深层模型中低层参数训练不足。

---

### 六、Sparse-Pertoken MoE（稀疏扩展）

#### 设计理念："先扩大，再稀疏"

与传统 MoE 直接增加 Expert 数量不同，TokenMixer-Large 先设计最优稠密模型，再将 Pertoken SwiGLU 细粒度拆分并稀疏激活，实现 **"Sparse Train, Sparse Infer"**。

**S-P MoE 公式**：

```math
\text{S-P MoE}(\cdot) = \alpha \cdot \sum_{i=1}^{k-1} g_i(\cdot) \cdot \text{Expert}_i(\cdot) + \text{SharedExpert}(\cdot)
```

其中每个 Expert 为独立 SwiGLU：

```math
\text{Expert}_j(\cdot) = \text{FC}_{\text{down},j}(\text{Swish}(\text{FC}_{\text{gate},j}(\cdot)) \odot \text{FC}_{\text{up},j}(\cdot))
```

#### 三大关键设计

| 设计 | 说明 |
|------|------|
| **Shared Expert** | 每个 token 有一个始终激活的共享专家（per-token 而非全局共享），提升训练稳定性 |
| **Gate Value Scaling（$`\alpha`$）** | 放大路由输出以增强被选专家的梯度更新；$`\alpha`$ 与稀疏比**成反比**（稀疏 1:2 时 $`\alpha=2`$，稀疏 1:4 时 $`\alpha=4`$） |
| **Down-Matrix Small Init** | 将 SwiGLU 中 $`\text{FC}_{\text{down}}`$ 的初始化 scale 从 1 降至 0.01（Xavier Uniform），使早期训练近似恒等映射，改善深层收敛 |

**稀疏比选择**：当前线上部署选择 **1:2 稀疏**（激活参数/总参数），可实现与稠密模型零损失的离线/在线性能，且无需额外负载均衡损失。

### 七、训练/推理工程优化

#### Token Parallel（Token 并行策略）

专为 TokenMixer-Large 设计的模型并行策略，将 Pertoken 操作权重和计算分布到多设备：

```python
# 核心伪代码
# Mixing: 按 Batch 分片 → all2all → 按 Head 分片
X_chunk = split(X_prev, num_splits=N, axis=2)   # Shard(Batch)
X_head = concat(X_chunk, axis=0)
X_hidden = all2all(X_head)                       # Shard(Head)
h_hat = pertokenOperation(X_hidden)

# Reverting: 按 Head 分片 → all2all → 按 Token 分片
chunks_t = split(h_hat, num_splits=T, axis=2)
x_revert = all2all(payload_t)                    # Shard(Token)
x_hat = pertokenOperation(x_revert)
```

**效果**：
- 4 路 Token 并行（batch=320）：吞吐提升 **29.2%**
- 叠加计算通信重叠（micro-batch 调度）：吞吐提升 **96.6%**
- 通信开销从 $`4L`$ 次降至 $`2L+1`$ 次（$`L`$ 层）

#### FP8 量化推理

- 训练：BF16 精度
- 推理：FP8 E4M3 后训练量化
- 效果：推理速度 **1.7×** 加速，模型精度零损失
- 流程：MoEPermute 输入量化 → MoEGroupedSwiglu 融合量化 → MoEGroupedGemm FP8 计算

#### 纯模型设计（Pure Model Design）

随着参数量增加，碎片化算子（DCN、LHUC、DHEN 等）的增益被 TokenMixer-Large 自身完全吸收：

| 参数量 | DCN 额外增益 |
|--------|------------|
| 150M | +0.09% |
| 500M | +0.04% |
| 700M | **+0.00%** |

移除这些 IO 密集型算子后，TokenMixer-Large Block 的 MFU 可达 **60%**。

### Scaling Law 分析

**不同场景 Scaling 结果**：

| 场景 | 离线最大规模 | 在线部署规模 |
|------|------------|------------|
| Feed 广告 | **15B** | 7B |
| 电商 | **7B** | 4B |
| 直播 | **4B** | 2B |

**两大 Scaling 规律**：
1. **均衡扩展收益更大**：超过 1B 参数后，单方向扩展（仅宽度/深度/FFN 比例）会遇到瓶颈，需多维度均衡扩展
2. **更大模型需要更多数据**：30M→90M 只需 14 天数据收敛；500M→2.3B 需要 60 天数据才能充分收敛

**直播场景收敛数据**（每行以上一行为基准）：

## 核心贡献总结

1. **Mixing & Reverting**：对称双层结构彻底解决 RankMixer 的三大残差缺陷（SR/OTR/TSA），在同等参数和 FLOPs 下 AUC 提升 +0.13%

2. **深层稳定训练三件套**：
   - Pre-Norm RMSNorm（吞吐 +8.4%，消除 NaN）
   - Inter-Residual（跨层残差缓解梯度消失）
   - Auxiliary Loss + Down-Matrix Small Init（低层参数充分训练 + 深层收敛改善）

3. **Sparse-Pertoken MoE**："先扩大再稀疏" + 共享专家 + Gate Value Scaling + 小初始化，实现 1:2 稀疏时零性能损失，真正的 Sparse Train & Sparse Infer

4. **Token Parallel + FP8**：专为 TokenMixer 架构设计的模型并行策略（通信次数 $`4L \to 2L+1`$），叠加计算通信重叠后吞吐提升 **96.6%**；FP8 推理加速 **1.7×**

5. **纯模型哲学**：验证了随参数规模增长，碎片化算子（DCN/LHUC/DHEN）的增益被主干网络完全吸收，移除后 MFU 达 **60%**

6. **极限 Scaling 验证**：在三个业务场景分别扩展至 15B/7B/4B，验证 TokenMixer-Large 具有比 RankMixer 更陡的 log-linear Scaling 曲线

## 与 RankMixer 的关键设计差异速查

| 维度 | RankMixer | TokenMixer-Large |
|------|-----------|-----------------|
| 跨 Token 交互 | Reshape（固定 H，可能 ≠ T） | Mixing & Reverting（H 可 ≠ T，但最终还原为 T） |
| 残差语义对齐 | ✗（Mixing 前后语义不一致） | ✓（Reverting 保证语义对齐） |
| FFN 激活函数 | GELU（Pertoken FFN） | SwiGLU（Pertoken SwiGLU） |
| 归一化 | Post-Norm LayerNorm | Pre-Norm RMSNorm |
| MoE 范式 | Dense Train, Sparse Infer（ReLU-MoE） | Sparse Train, Sparse Infer（S-P MoE） |
| 深层支持 | 通常 2 层 | 支持任意深度（Inter-Residual + AuxLoss） |
| 最大参数量（在线） | ~1B | **7B（电商）/ 7B（广告）** |
| 广告 MFU | ~低 | **60%** |
